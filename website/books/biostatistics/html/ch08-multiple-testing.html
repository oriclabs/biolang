Biostatistics in Practice

The Multiple Testing Problem

“If you look at enough things, you’re guaranteed to find something interesting — whether or not anything truly interesting exists.” — John Tukey

This is the most important chapter in the book. If you take away one lesson from studying biostatistics, let it be this: when you test thousands of hypotheses simultaneously, your p-values lie. A p-value of 0.01 does not mean what you think it means when you have computed 20,000 of them.

The multiple testing problem is not a theoretical curiosity. It is the central statistical challenge of genomics. Every differential expression analysis, every GWAS, every eQTL study, every enrichment analysis faces this problem. Getting it wrong means publishing false discoveries. Getting it right means understanding the difference between a p-value and a q-value, between FWER and FDR, between Bonferroni and Benjamini-Hochberg.

The Problem

A researcher has RNA-seq expression data from breast cancer (tumor) vs adjacent normal tissue. She runs a t-test on each of 20,000 genes and declares any gene with p < 0.05 as “differentially expressed.” Her results: 3,200 significant genes.

Is this a remarkable finding? Or is it exactly what you would expect from testing 20,000 genes even if NONE of them were truly different?

At alpha = 0.05, you expect 5% of your true null hypotheses to produce a “significant” result by chance. If 15,000 of the 20,000 genes are truly not different, you expect 15,000 * 0.05 = 750 false positives. Her 3,200 hits might include 750 genes that have nothing to do with cancer.

That is the multiple testing problem.

For a GWAS with 1 million SNPs, you expect 50,000 false positives at p < 0.05. For eQTL mapping (testing every gene against every nearby variant), you expect 10 million false positives. Without correction, your results are meaningless.

The Jelly Bean Analogy

Imagine testing 20 colors of jelly beans for a link to acne. At alpha = 0.05, you expect 1 color to show p < 0.05 by chance. You publish: “Green jelly beans cause acne!” This is exactly what happens in genomics, but with 20,000 tests instead of 20.

The Data

We return to the GSE183947 breast cancer expression dataset from Chapter 5. This time, our focus is not on individual gene comparisons but on what happens when we test ALL genes simultaneously.

Loading and Preparing

# Download and load the breast cancer expression data
let url = "https://ftp.ncbi.nlm.nih.gov/geo/series/GSE183nnn/GSE183947/suppl/GSE183947_fpkm.csv.gz"
let expr = download(url, "GSE183947_fpkm.csv.gz") |> read_csv()

println("Genes: " + to_string(len(expr)))
println("Samples: " + to_string(ncols(expr)))

# Identify tumor and normal samples
let all_cols = columns(expr)
let tumor_cols = all_cols |> filter(|c| contains(c, "tumor"))
let normal_cols = all_cols |> filter(|c| contains(c, "normal"))
let gene_col = all_cols[0]

println("Tumor samples: " + to_string(len(tumor_cols)))
println("Normal samples: " + to_string(len(normal_cols)))

Running Genome-Wide t-Tests

# Test every gene: tumor vs normal
let genes = col_values(expr, gene_col)

let results = genes |> map(|gene| {
  let row = expr |> filter_by(gene_col, |g| g == gene)

  let tumor_vals = tumor_cols |> map(|c| col_values(row, c)) |> flatten()
  let normal_vals = normal_cols |> map(|c| col_values(row, c)) |> flatten()

  # Log2 transform
  let lt = tumor_vals |> map(|x| log2(x + 1.0))
  let ln = normal_vals |> map(|x| log2(x + 1.0))

  let log2fc = mean(lt) - mean(ln)
  let t_res = ttest(lt, ln)

  record(
    "gene", gene,
    "log2fc", round(log2fc, 4),
    "p_value", t_res.p_value,
    "mean_tumor", round(mean(lt), 4),
    "mean_normal", round(mean(ln), 4)
  )
}) |> to_list() |> table()

println("Total genes tested: " + to_string(len(results)))

Exploration

The Histogram of P-Values

This is the single most important diagnostic plot in genomics. Before applying any correction, look at the histogram of your raw p-values.

# Histogram of raw p-values
let pvals = col_values(results, "p_value")
histogram(pvals, "Distribution of Raw P-values (20,000 genes)")
  |> save_plot("pvalue_histogram.svg")

How to read a p-value histogram:

Under the null hypothesis (no genes are truly different), p-values should follow a uniform distribution — a flat histogram from 0 to 1. Every bin should have approximately the same height.

In reality, you see:

  • A spike near 0: these are the true positives (genes that are actually differentially expressed). The spike is your signal.
  • A flat region from about 0.3 to 1.0: these are the true nulls. The height of this flat region tells you what fraction of genes are truly null.
  • A dip or bump in the middle: this can indicate problems (confounding, batch effects, wrong test).
# Count p-values in bins to see the shape
let bins = [0.0, 0.01, 0.05, 0.10, 0.20, 0.50, 1.0]
let bin_names = ["0-0.01", "0.01-0.05", "0.05-0.10", "0.10-0.20", "0.20-0.50", "0.50-1.0"]

bins |> enumerate() |> each(|pair| {
  let i = pair[0]
  if i < len(bins) - 1 then {
    let lower = bins[i]
    let upper = bins[i + 1]
    let count = pvals |> filter(|p| p >= lower and p < upper) |> len()
    let pct = round(to_float(count) / to_float(len(pvals)) * 100.0, 1)
    println(bin_names[i] + ": " + to_string(count) + " genes (" + to_string(pct) + "%)")
  }
})

The Analysis

Step 1: Family-Wise Error Rate (FWER) — Bonferroni

The FWER is the probability of making at least one false positive. Bonferroni controls FWER by dividing alpha by the number of tests.

# Bonferroni correction
# Adjusted p-value = raw p-value * number of tests
# (capped at 1.0)
let bonferroni_pvals = p_adjust(pvals, "bonferroni")

let n_bonf_sig = bonferroni_pvals |> filter(|p| p < 0.05) |> len()
println("=== Bonferroni Correction ===")
println("Effective alpha per test: " + to_string(0.05 / to_float(n_genes)))
println("Significant at FWER < 0.05: " + to_string(n_bonf_sig) + " genes")

# Add Bonferroni-adjusted p-values to results
let results_bonf = results
  |> mutate("p_bonferroni", |row| {
    let idx = col_values(results, "gene")
      |> enumerate()
      |> filter(|pair| pair[1] == row.gene)
    if len(idx) > 0 then bonferroni_pvals[idx[0][0]] else 1.0
  })

# Top hits by Bonferroni
let bonf_hits = results_bonf
  |> filter_by("p_bonferroni", |p| p < 0.05)
  |> arrange("p_bonferroni")

println("Top Bonferroni-significant genes:")
take(bonf_hits, 20)

Why Bonferroni is too conservative for genomics:

Bonferroni guarantees that the probability of even ONE false positive is less than alpha. This is extremely strict. With 20,000 tests, you are dividing alpha by 20,000 — requiring p < 0.0000025 for significance. Many truly differentially expressed genes with moderate effects will be missed.

# Demonstrate Bonferroni's conservatism
println("=== Bonferroni Threshold ===")
println("Original alpha: 0.05")
println("Bonferroni alpha: " + to_string(0.05 / to_float(n_genes)))
println("That is " + to_string(round(to_float(n_genes), 0)) + "x more stringent!")
println("")
println("With " + to_string(n_genes) + " tests at raw p<0.05: "
  + to_string(pvals |> filter(|p| p < 0.05) |> len()) + " significant")
println("With Bonferroni at FWER<0.05: "
  + to_string(n_bonf_sig) + " significant")
println("Bonferroni throws away " + to_string(
  (pvals |> filter(|p| p < 0.05) |> len()) - n_bonf_sig
) + " potentially real findings")

Step 2: Holm’s Step-Down Procedure

Holm’s method is a step-down version of Bonferroni that is uniformly more powerful (it always rejects at least as many hypotheses as Bonferroni) while still controlling FWER.

# Holm correction
let holm_pvals = p_adjust(pvals, "holm")

let n_holm_sig = holm_pvals |> filter(|p| p < 0.05) |> len()
println("=== Holm's Step-Down Correction ===")
println("Significant at FWER < 0.05: " + to_string(n_holm_sig) + " genes")
println("Bonferroni found: " + to_string(n_bonf_sig))
println("Holm is never worse than Bonferroni")

How Holm works:

  1. Sort all p-values from smallest to largest
  2. For the smallest p-value, compare to alpha/m (same as Bonferroni)
  3. For the second smallest, compare to alpha/(m-1)
  4. Continue until a p-value is not rejected — stop, reject nothing further

This is more powerful because later tests use less stringent thresholds.

Step 3: False Discovery Rate (FDR) — Benjamini-Hochberg

FDR is the expected proportion of false positives among all rejected hypotheses. Instead of controlling the probability of ANY false positive (FWER), FDR controls the RATE of false positives among your discoveries.

This is a fundamentally different question:

  • FWER: “What is the probability I made even one mistake?”
  • FDR: “Among the genes I call significant, what fraction are mistakes?”

For genomics, FDR is almost always more appropriate. If you call 500 genes significant at FDR = 0.05, you expect about 25 of them (5%) to be false positives. That is a tolerable error rate for most applications.

# Benjamini-Hochberg (BH) correction
let bh_pvals = p_adjust(pvals, "bh")

let n_bh_sig = bh_pvals |> filter(|p| p < 0.05) |> len()
println("=== Benjamini-Hochberg (FDR) Correction ===")
println("Significant at FDR < 0.05: " + to_string(n_bh_sig) + " genes")
println("Significant at FDR < 0.01: " + to_string(
  bh_pvals |> filter(|p| p < 0.01) |> len()
) + " genes")
println("Significant at FDR < 0.10: " + to_string(
  bh_pvals |> filter(|p| p < 0.10) |> len()
) + " genes")

# Compare all methods side by side
println("=== Comparison of Correction Methods ===")
println("Method          | Significant at 0.05 | Controls")
println("----------------|--------------------|---------")
println("None (raw)      | " + to_string(pvals |> filter(|p| p < 0.05) |> len())
  + "               | Nothing")
println("Bonferroni      | " + to_string(n_bonf_sig)
  + "               | FWER")
println("Holm            | " + to_string(n_holm_sig)
  + "               | FWER")
println("BH (FDR)        | " + to_string(n_bh_sig)
  + "               | FDR")

How BH works:

  1. Sort all m p-values from smallest to largest: p(1), p(2), …, p(m)
  2. For each p(i), compute the BH threshold: i/m * alpha
  3. Find the largest i where p(i) <= i/m * alpha
  4. Reject all hypotheses with p-values <= p(i)
# Demonstrate the BH procedure step by step
let sorted_pvals = pvals |> sort()
let m = len(sorted_pvals)
let alpha = 0.05

# Find the BH threshold
let bh_thresholds = sorted_pvals |> enumerate() |> map(|pair| {
  let i = pair[0] + 1  # 1-indexed rank
  let p = pair[1]
  let threshold = to_float(i) / to_float(m) * alpha
  record("rank", i, "p_value", p, "threshold", threshold,
         "reject", p <= threshold)
})

# Show the first 20 and last few before the cutoff
println("=== BH Procedure (first 20 genes) ===")
println("Rank | P-value      | Threshold    | Reject?")
bh_thresholds |> take(20) |> each(|row| {
  println(to_string(row.rank) + "    | "
    + to_string(row.p_value) + " | "
    + to_string(round(row.threshold, 8)) + " | "
    + to_string(row.reject))
})

Step 4: BY Correction for Correlated Tests

The BH procedure assumes independence between tests. But genes are correlated — co-regulated genes, pathway members, and genomic neighbors often have correlated expression. The Benjamini-Yekutieli (BY) correction handles arbitrary dependence structure.

# BY correction (for correlated tests)
let by_pvals = p_adjust(pvals, "by")

let n_by_sig = by_pvals |> filter(|p| p < 0.05) |> len()
println("=== Benjamini-Yekutieli (BY) Correction ===")
println("Significant at FDR < 0.05: " + to_string(n_by_sig) + " genes")
println("")
println("Comparison:")
println("  BH (assumes independence): " + to_string(n_bh_sig) + " genes")
println("  BY (allows correlation):   " + to_string(n_by_sig) + " genes")
println("  BY is more conservative than BH")

In practice, BH works well even with moderate correlation. BY is overly conservative for most genomics applications. Use BY only when you have strong prior reason to believe tests are heavily correlated.

Step 5: Independent Filtering — Improving Power

Here is a counterintuitive fact: you can INCREASE your power to detect true differences by REMOVING genes BEFORE testing. Specifically, removing genes with very low expression (low mean, low variance) removes true nulls that add noise to the multiple testing correction without contributing any signal.

# Independent filtering: remove low-expression genes before testing

# Step 1: compute mean expression across all samples
let mean_expr = genes |> map(|gene| {
  let row = expr |> filter_by(gene_col, |g| g == gene)
  let all_vals = concat(tumor_cols, normal_cols) |> map(|c| col_values(row, c)) |> flatten()
  let log_mean = mean(all_vals |> map(|x| log2(x + 1.0)))
  record("gene", gene, "mean_expr", log_mean)
}) |> to_list() |> table()

# Step 2: filter out lowly expressed genes (mean log2 FPKM < 1)
let expressed = mean_expr
  |> filter_by("mean_expr", |m| m >= 1.0)

let n_expressed = len(expressed)
let n_filtered = len(mean_expr) - n_expressed

println("=== Independent Filtering ===")
println("Total genes: " + to_string(len(mean_expr)))
println("Expressed (mean log2 >= 1): " + to_string(n_expressed))
println("Filtered out: " + to_string(n_filtered))

# Re-run BH on only expressed genes
let expressed_genes = col_values(expressed, "gene")
let expressed_results = results
  |> filter_by("gene", |g| expressed_genes |> contains(g))

let expressed_pvals = col_values(expressed_results, "p_value")
let expressed_bh = p_adjust(expressed_pvals, "bh")

let n_expressed_sig = expressed_bh |> filter(|p| p < 0.05) |> len()

println("=== Effect of Independent Filtering on BH ===")
println("BH on ALL genes:       " + to_string(n_bh_sig) + " significant")
println("BH on expressed genes: " + to_string(n_expressed_sig) + " significant")
println("Filtering INCREASED discoveries by: " + to_string(n_expressed_sig - n_bh_sig))
println("")
println("This works because:")
println("  - Low-expression genes are rarely truly differential")
println("  - Removing them reduces the number of tests m")
println("  - The BH threshold i/m * alpha becomes larger (less strict)")
println("  - True positives that were borderline now pass the threshold")

Important: The filtering criterion must be INDEPENDENT of the test statistic. You cannot filter by p-value first and then re-correct — that is circular. Mean expression is independent of the difference between groups, so it is a valid filter.

Step 7: The Pi-Zero Concept

Pi-zero (pi_0) is the estimated proportion of true null hypotheses. If you test 20,000 genes and 15,000 are truly not different (pi_0 = 0.75), then 5,000 are truly different. Estimating pi_0 tells you how much signal is in your data.

# Estimating pi_0 from the p-value histogram
# The flat region of the p-value histogram estimates pi_0

# Under the null, p-values are uniform
# Under the alternative, p-values are concentrated near 0
# The height of the flat region (say p > 0.5) estimates pi_0

let n_total = len(pvals)
let n_above_05 = pvals |> filter(|p| p > 0.5) |> len()

# If p-values were uniform, half would be above 0.5
# The fraction above 0.5 estimates pi_0
let pi0_estimate = to_float(n_above_05) / to_float(n_total) * 2.0

println("=== Estimating pi_0 (proportion of true nulls) ===")
println("Total genes: " + to_string(n_total))
println("P-values > 0.5: " + to_string(n_above_05))
println("Estimated pi_0: " + to_string(round(pi0_estimate, 3)))
println("Estimated true positives: " + to_string(round(to_float(n_total) * (1.0 - pi0_estimate), 0)))
println("Estimated true nulls: " + to_string(round(to_float(n_total) * pi0_estimate, 0)))

# Use different cutoffs to verify pi_0 estimate stability
let cutoffs = [0.3, 0.4, 0.5, 0.6, 0.7, 0.8, 0.9]
println("Pi_0 estimates at different cutoffs:")
cutoffs |> each(|lambda| {
  let n_above = pvals |> filter(|p| p > lambda) |> len()
  let pi0 = to_float(n_above) / (to_float(n_total) * (1.0 - lambda))
  println("  lambda = " + to_string(lambda) + ": pi_0 = " + to_string(round(pi0, 3)))
})

Step 8: Understanding What FDR = 0.05 Really Means

This is the most commonly misunderstood concept in genomics. When you set FDR = 0.05 and find 500 significant genes:

  • It does NOT mean each gene has a 5% probability of being a false positive
  • It DOES mean that among your 500 genes, approximately 25 (5%) are expected to be false positives
  • It does NOT guarantee exactly 25 false positives — it is an expected value
# What FDR = 0.05 means in practice

let fdr_thresholds = [0.01, 0.05, 0.10, 0.20]
println("=== FDR Interpretation ===")
fdr_thresholds |> each(|fdr| {
  let n_sig = bh_pvals |> filter(|p| p < fdr) |> len()
  let expected_fp = round(to_float(n_sig) * fdr, 1)
  let expected_tp = round(to_float(n_sig) * (1.0 - fdr), 1)

  println("FDR < " + to_string(fdr) + ":")
  println("  Significant genes: " + to_string(n_sig))
  println("  Expected false positives: " + to_string(expected_fp)
    + " (" + to_string(round(fdr * 100.0, 0)) + "% of discoveries)")
  println("  Expected true positives: " + to_string(expected_tp))
  println("")
})

# The q-value: gene-specific FDR
# The BH-adjusted p-value for a gene IS its q-value
# q = 0.03 means: if you call all genes with q <= 0.03 significant,
# 3% of those calls are expected to be false

# Example genes
let example_genes = ["TP53", "BRCA1", "GAPDH", "ERBB2", "ESR1",
                     "MKI67", "CDH1", "PGR"]

println("=== Q-values for Known Genes ===")
example_genes |> each(|gene| {
  let row = results_with_bh
    |> filter_by("gene", |g| contains(g, gene))
  if len(row) > 0 then {
    let raw_p = col_values(row, "p_value")[0]
    let bh_q = col_values(row, "bh_p")[0]
    let fc = col_values(row, "log2fc")[0]
    println(gene + ": raw p = " + to_string(raw_p)
      + ", q-value = " + to_string(round(bh_q, 6))
      + ", log2FC = " + to_string(fc))
  }
})

    all_pvals |> filter(|p| p < 0.01) |> len(),

    holm |> filter(|p| p < 0.01) |> len(),
    bh |> filter(|p| p < 0.01) |> len(),
    by |> filter(|p| p < 0.01) |> len()
  ]
)

println("=== Correction Method Comparison ===")
summary

# Write complete results to file
let final_results = results
  |> mutate("p_bonferroni", |row| {
    let idx = genes |> enumerate() |> filter(|p| p[1] == row.gene)
    if len(idx) > 0 then bonf[idx[0][0]] else 1.0
  })
  |> mutate("p_holm", |row| {
    let idx = genes |> enumerate() |> filter(|p| p[1] == row.gene)
    if len(idx) > 0 then holm[idx[0][0]] else 1.0
  })
  |> mutate("p_bh", |row| {
    let idx = genes |> enumerate() |> filter(|p| p[1] == row.gene)
    if len(idx) > 0 then bh[idx[0][0]] else 1.0
  })
  |> mutate("p_by", |row| {
    let idx = genes |> enumerate() |> filter(|p| p[1] == row.gene)
    if len(idx) > 0 then by[idx[0][0]] else 1.0
  })

# Sort by BH-adjusted p-value
let final_sorted = final_results |> arrange("p_bh")

write_tsv(final_sorted, "differential_expression_results.csv")
println("Results written to differential_expression_results.csv")

The GWAS Threshold

In genome-wide association studies, the convention is p < 5e-8, which corresponds to a Bonferroni correction for approximately 1 million independent tests (the effective number of independent SNPs in the genome after accounting for linkage disequilibrium).

Decision

The Complete Multiple Testing Pipeline

# ============================================================
# Complete Multiple Testing Correction Pipeline
# ============================================================

# 1. Load and prepare data
let url = "https://ftp.ncbi.nlm.nih.gov/geo/series/GSE183nnn/GSE183947/suppl/GSE183947_fpkm.csv.gz"
let expr = download(url, "GSE183947_fpkm.csv.gz") |> read_csv()

let all_cols = columns(expr)
let tumor_cols = all_cols |> filter(|c| contains(c, "tumor"))
let normal_cols = all_cols |> filter(|c| contains(c, "normal"))
let gene_col = all_cols[0]
let genes = col_values(expr, gene_col)

# 2. Pre-filter: remove lowly expressed genes (INDEPENDENT of test)
let keep_genes = genes |> filter(|gene| {
  let row = expr |> filter_by(gene_col, |g| g == gene)
  let all_vals = concat(tumor_cols, normal_cols)
    |> map(|c| col_values(row, c)) |> flatten()
  let log_mean = mean(all_vals |> map(|x| log2(x + 1.0)))
  log_mean >= 1.0
})

println("Genes before filter: " + to_string(len(genes)))
println("Genes after filter:  " + to_string(len(keep_genes)))

# 3. Test filtered genes
let filtered_results = keep_genes |> map(|gene| {
  let row = expr |> filter_by(gene_col, |g| g == gene)
  let lt = tumor_cols |> map(|c| col_values(row, c)) |> flatten()
    |> map(|x| log2(x + 1.0))
  let ln = normal_cols |> map(|c| col_values(row, c)) |> flatten()
    |> map(|x| log2(x + 1.0))
  let fc = mean(lt) - mean(ln)
  let t = ttest(lt, ln)
  record("gene", gene, "log2fc", round(fc, 4), "p_value", t.p_value)
}) |> to_list() |> table()

# 4. Correct for multiple testing
let raw_p = col_values(filtered_results, "p_value")
let adj_p = p_adjust(raw_p, "bh")

# 5. Inspect p-value histogram
histogram(raw_p, "P-value Distribution (filtered genes)")
  |> save_plot("pval_hist_filtered.svg")

# 6. Apply threshold and report
let n_sig = adj_p |> filter(|p| p < 0.05) |> len()
let expected_fp = round(to_float(n_sig) * 0.05, 0)

println("=== Final Results ===")
println("Genes tested: " + to_string(len(keep_genes)))
println("Significant (FDR < 0.05): " + to_string(n_sig))
println("Expected false discoveries: ~" + to_string(expected_fp))

# 7. Final volcano plot
let final_data = filtered_results
  |> mutate("neg_log10_fdr", |row| {
    let idx = col_values(filtered_results, "gene") |> enumerate()
      |> filter(|p| p[1] == row.gene)
    if len(idx) > 0 then -log10(adj_p[idx[0][0]] + 1e-300) else 0.0
  })

volcano(final_data, "log2fc", "neg_log10_fdr")
  |> save_plot("final_volcano.svg")

Pitfalls

Pitfall 1: Not Correcting at All

# The most common mistake: reporting raw p-values from genome-wide tests
println("If you test 20,000 genes and report raw p < 0.05:")
println("  You WILL have hundreds of false positives")
println("  Reviewers WILL reject your paper")
println("  The results CANNOT be trusted")
println("")
println("ALWAYS correct for multiple testing in genome-wide analyses")

Pitfall 2: Applying Bonferroni When BH Would Be Better

# Bonferroni is appropriate when:
# - You have few tests (< 50)
# - False positives are very costly (drug targets, clinical decisions)
# - You need a simple, universally accepted method

# Bonferroni is NOT appropriate when:
# - You have thousands of tests (genomics)
# - You can tolerate some false positives
# - You want to maximize discoveries for follow-up

println("With 20,000 genes:")
println("  Bonferroni threshold: p < " + to_string(0.05 / 20000.0))
println("  BH can identify 10x more genes at the same overall error rate")
println("  Use BH for discovery, Bonferroni for validation")

Pitfall 3: Filtering AFTER Testing

# WRONG: test all genes, then filter by expression level
# This biases the correction because you counted tests you then excluded

# RIGHT: filter by expression BEFORE testing
# The filter must be INDEPENDENT of the test statistic

# Common independent filters:
println("Valid independent filters:")
println("  - Mean expression across all samples")
println("  - Variance across all samples")
println("  - Gene length")
println("  - GC content")
println("")
println("INVALID filters (dependent on test):")
println("  - P-value (circular!)")
println("  - Fold change (correlated with test statistic)")
println("  - Expression in only one group")

Pitfall 4: Confusing FDR with Per-Gene Error Probability

# A common misinterpretation:
# "This gene has FDR = 0.03, so there's a 3% chance it's a false positive"
# WRONG!

# The correct interpretation:
# "If I call all genes with FDR <= 0.03 significant,
#  3% of those genes are expected to be false positives"

# FDR is a PROPERTY OF THE LIST, not of individual genes
println("FDR = 0.05 means:")
println("  Among ALL genes you call significant at this threshold,")
println("  5% are expected to be false discoveries.")
println("")
println("It does NOT mean:")
println("  Each individual gene has a 5% chance of being false.")

Pitfall 5: Multiple Testing Across Multiple Analyses

# You run DE analysis, then pathway analysis, then GO enrichment,
# then network analysis on the same dataset.
# Each analysis has its own multiple testing correction.
# But you are also testing multiple ANALYSES!

println("The meta-analysis problem:")
println("  1. Differential expression: 20,000 tests (BH corrected)")
println("  2. Pathway enrichment: 300 pathways (BH corrected)")
println("  3. GO enrichment: 5,000 terms (BH corrected)")
println("  4. Motif enrichment: 800 motifs (BH corrected)")
println("")
println("Each is individually corrected, but you ran 4 analyses.")
println("If each has a 5% FDR, the combined FDR is higher.")
println("")
println("In practice, this is usually acknowledged in the discussion")
println("rather than formally corrected. Pre-registration helps.")

Pitfall 6: The Replication Trap

# "We found 500 genes at FDR < 0.05 in dataset 1.
#  We then tested just these 500 genes in dataset 2.
#  We corrected for 500 tests (not 20,000)."
#
# This is actually CORRECT and is the standard approach.
# The replication study tests a pre-specified set of hypotheses.
# The multiple testing burden is only for the number of hypotheses tested.

println("Replication analysis:")
println("  Discovery: test 20,000 genes, correct for 20,000")
println("  Replication: test only the significant ones, correct for THOSE")
println("  This is valid because the hypotheses were pre-specified")

Pitfall 7: Thresholding-Induced Artifacts

# Using a hard FDR threshold creates artificial boundaries
# Gene A with FDR = 0.049 is "significant"
# Gene B with FDR = 0.051 is "not significant"
# But they are essentially identical

# Better approaches:
println("Instead of binary significant/not-significant:")
println("  1. Report the full ranked list with FDR values")
println("  2. Use gene set enrichment (which avoids thresholds)")
println("  3. Report results at multiple FDR thresholds")
println("  4. Focus on effect size (fold change) alongside FDR")
println("  5. Use volcano plots that show the continuous relationship")

Key Takeaways

  1. Testing 20,000 genes at p < 0.05 gives ~1,000 false positives by chance
  2. FWER controls the probability of ANY false positive (Bonferroni, Holm)
  3. FDR controls the RATE of false positives among discoveries (BH, BY)
  4. Bonferroni is too conservative for genomics — use BH instead
  5. BH is the workhorse of genomics multiple testing correction
  6. Independent filtering improves power without inflating false positives
  7. The p-value histogram is the key diagnostic — always plot it
  8. Pi-zero estimates the fraction of true nulls in your data
  9. FDR is a property of the list, not of individual genes
  10. Report both raw and adjusted p-values alongside effect sizes
