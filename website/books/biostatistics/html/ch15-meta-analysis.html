Biostatistics in Practice

Chapter 15: Meta-Analysis

The Problem

Three genome-wide association studies have examined the same variant for its association with type 2 diabetes. Study A (n=5,000) reports an odds ratio of 1.35 with p=0.02. Study B (n=3,200) reports OR=1.18 with p=0.15. Study C (n=8,500) reports OR=1.28 with p=0.005. Is this variant truly associated with the disease?

Study A says yes. Study B says no. Study C says yes emphatically. A naive reading would conclude “two out of three studies agree.” But that ignores sample size, effect size magnitude, confidence intervals, and the possibility that the studies are measuring slightly different things.

Meta-analysis is the formal statistical framework for combining results across studies. It produces a single pooled estimate with proper uncertainty quantification, tests for consistency across studies, and can detect when studies disagree for systematic reasons.

Why It Matters

Individual genomics studies are almost always underpowered. A typical GWAS has 80% power to detect an odds ratio of 1.3 at genome-wide significance (p < 5e-8) only when the sample size exceeds 10,000 cases and 10,000 controls. Most studies are smaller. This means:

  • True associations are missed (false negatives)
  • Detected associations have inflated effect sizes (winner’s curse)
  • Results vary dramatically between studies by chance

Meta-analysis solves these problems by combining evidence:

The GWAS Catalog at EBI (https://www.ebi.ac.uk/gwas/) contains over 500,000 variant-trait associations from thousands of studies. Many of the most important GWAS discoveries came from meta-analyses — the GIANT consortium for height (>5 million individuals), the PGC for psychiatric disorders, the DIAGRAM consortium for diabetes.

The GTEx consortium performs meta-analysis across tissues to identify expression quantitative trait loci (eQTLs) — genetic variants that affect gene expression. Their multi-tissue meta-analysis (METASOFT) identifies eQTLs that are shared across tissues vs tissue-specific.

Drug target validation increasingly relies on Mendelian randomization, which is itself a form of meta-analysis — combining effect sizes from multiple genetic instruments to estimate causal effects.

Exploration

Understanding Effect Sizes and Their Uncertainties

Before combining studies, we need to understand what each study reports. The key quantities are:

  • Effect size: odds ratio (OR) for case-control studies, beta coefficient for quantitative traits
  • Standard error (SE): the uncertainty in the effect size
  • Confidence interval: typically 95% CI = effect +/- 1.96 * SE
  • P-value: significance of the effect

All meta-analysis methods work with effect sizes and their standard errors. If a study only reports a p-value (common in older GWAS), the effect size cannot be directly meta-analyzed — you must use p-value combination methods instead.

# Extract effect sizes for a specific SNP across studies
# Example: rs7903146 in TCF7L2 — the strongest T2D risk variant

let target_snp = "rs7903146"
let snp_studies = t2d_studies |> filter(|row|
  row.SNPS == target_snp or row.SNPS == target_snp
)

println("Studies for " + target_snp + ":")
println("  Number of studies: " + str(len(snp_studies)))

# Extract effect sizes and sample sizes
let study_data = snp_studies |> mutate(|row| {
  study: row.STUDY,
  first_author: row.FIRST_AUTHOR,
  year: row.DATE,
  or_beta: row.OR_BETA,
  ci_95: row.CI_95,
  pvalue: row.P_VALUE,
  sample_size: row.SAMPLE_SIZE,
  journal: row.JOURNAL
})

println("")

Converting Between Effect Size Scales

Different studies may report effect sizes on different scales. Binary outcomes use odds ratios (OR), while quantitative traits use beta coefficients. For meta-analysis, all studies must be on the same scale.

# For odds ratios, meta-analysis is done on the log(OR) scale
# because log(OR) is approximately normally distributed

# Example: three studies report ORs for a diabetes variant
let studies = [
  { name: "Study A", or: 1.35, se_log_or: 0.12, n: 5000 },
  { name: "Study B", or: 1.18, se_log_or: 0.15, n: 3200 },
  { name: "Study C", or: 1.28, se_log_or: 0.08, n: 8500 }
]

# Convert to log(OR) scale
let log_or_studies = studies |> map(|s| {
  name: s.name,
  effect: log(s.or),              # log(OR)
  se: s.se_log_or,                # SE of log(OR)
  variance: s.se_log_or * s.se_log_or,
  weight: 1.0 / (s.se_log_or * s.se_log_or),  # inverse-variance weight
  n: s.n,
  or: s.or
})

println("Studies on log(OR) scale:")
log_or_studies |> each(|s|
  let ci_lo = exp(s.effect - 1.96 * s.se)
  let ci_hi = exp(s.effect + 1.96 * s.se)
  println("  " + s.name + ": log(OR)=" + str(round(s.effect, 4)) +
          " SE=" + str(round(s.se, 4)) +
          " OR=" + str(round(s.or, 3)) +
          " 95%CI=[" + str(round(ci_lo, 3)) + "," + str(round(ci_hi, 3)) + "]")
)

The Analysis

Fixed-Effects Meta-Analysis (Inverse-Variance Weighting)

The simplest meta-analysis model is the fixed-effects model. It assumes that all studies are estimating the same true effect, and differences between studies are due to sampling variation alone.

The pooled estimate is the weighted average of the study effects, where each study is weighted by the inverse of its variance (i.e., more precise studies get more weight):

Pooled effect = sum(w_i * theta_i) / sum(w_i) SE of pooled = 1 / sqrt(sum(w_i))

where w_i = 1 / SE_i^2 and theta_i is the effect estimate from study i.

# Fixed-effects meta-analysis
let total_weight = log_or_studies |> map(|s| s.weight) |> sum()
let weighted_sum = log_or_studies |> map(|s| s.weight * s.effect) |> sum()

let pooled_effect = weighted_sum / total_weight
let pooled_se = 1.0 / sqrt(total_weight)
let pooled_or = exp(pooled_effect)
let pooled_ci_lo = exp(pooled_effect - 1.96 * pooled_se)
let pooled_ci_hi = exp(pooled_effect + 1.96 * pooled_se)
let pooled_z = pooled_effect / pooled_se
let pooled_p = 2.0 * (1.0 - pnorm(abs(pooled_z)))

println("=== Fixed-Effects Meta-Analysis ===")
println("Pooled log(OR): " + str(round(pooled_effect, 4)))
println("Pooled SE: " + str(round(pooled_se, 4)))
println("Pooled OR: " + str(round(pooled_or, 3)))
println("95% CI: [" + str(round(pooled_ci_lo, 3)) + ", " +
        str(round(pooled_ci_hi, 3)) + "]")
println("Z-score: " + str(round(pooled_z, 2)))
println("P-value: " + str(round(pooled_p, 8)))
println("")

# Weight contribution from each study
log_or_studies |> each(|s|
  let pct_weight = 100.0 * s.weight / total_weight
  println("  " + s.name + " contributes " + str(round(pct_weight, 1)) +
          "% of the weight (n=" + str(s.n) + ")")
)

Testing for Heterogeneity: The Q Statistic

The fixed-effects model assumes all studies estimate the same true effect. The Q statistic tests this assumption. Q is the weighted sum of squared deviations from the pooled effect:

Q = sum(w_i * (theta_i - pooled)^2)

Under the null hypothesis of no heterogeneity, Q follows a chi-squared distribution with k-1 degrees of freedom (where k is the number of studies).

# Cochran's Q test for heterogeneity
let q_stat = log_or_studies |> map(|s|
  let dev = s.effect - pooled_effect
  s.weight * dev * dev
) |> sum()

let k = len(log_or_studies)
let df_q = k - 1

# P-value from chi-squared distribution
# Using the approximation: for df=2, critical value at 0.05 is 5.991
let q_pvalue = 1.0 - pnorm(sqrt(2.0 * q_stat) - sqrt(2.0 * df_q - 1.0))

println("=== Heterogeneity Test ===")
println("Cochran's Q: " + str(round(q_stat, 3)))
println("Degrees of freedom: " + str(df_q))
println("P-value: " + str(round(q_pvalue, 4)))

if q_pvalue < 0.10 then
  println("-> Significant heterogeneity detected (p < 0.10)")
  println("   Consider random-effects model")
else
  println("-> No significant heterogeneity (p >= 0.10)")
  println("   Fixed-effects model is appropriate")
end

The I-Squared Statistic

While Q tests whether heterogeneity exists, I-squared quantifies how much of the variability in effect sizes is due to true heterogeneity (as opposed to sampling variation).

I^2 = max(0, (Q - df) / Q) * 100%

Interpretation:

  • I^2 = 0%: No heterogeneity; all variability is sampling error
  • I^2 = 25%: Low heterogeneity
  • I^2 = 50%: Moderate heterogeneity
  • I^2 = 75%: High heterogeneity
# I-squared statistic
let i_squared = max(0, (q_stat - df_q) / q_stat) * 100.0

println("I-squared: " + str(round(i_squared, 1)) + "%")

if i_squared < 25 then
  println("-> Low heterogeneity: studies are consistent")
  println("   Fixed-effects meta-analysis is appropriate")
end
if i_squared >= 25 and i_squared < 50 then
  println("-> Low-to-moderate heterogeneity")
  println("   Fixed-effects is likely still appropriate")
end
if i_squared >= 50 and i_squared < 75 then
  println("-> Moderate heterogeneity")
  println("   Random-effects model recommended")
  println("   Investigate sources of heterogeneity")
end
if i_squared >= 75 then
  println("-> High heterogeneity")
  println("   Random-effects model essential")
  println("   Pool estimate may not be meaningful without understanding why studies differ")
end

Random-Effects Meta-Analysis (DerSimonian-Laird)

When heterogeneity is present, the fixed-effects model is too aggressive — it gives confidence intervals that are too narrow because it ignores the between-study variation. The random-effects model adds a between-study variance component (tau-squared) to account for this.

The DerSimonian-Laird method estimates tau-squared from the data:

tau^2 = max(0, (Q - df) / (sum(w_i) - sum(w_i^2)/sum(w_i)))

Then the random-effects weights become:

w_i* = 1 / (SE_i^2 + tau^2)

# DerSimonian-Laird random-effects meta-analysis

# Estimate tau-squared (between-study variance)
let sum_w = log_or_studies |> map(|s| s.weight) |> sum()
let sum_w2 = log_or_studies |> map(|s| s.weight * s.weight) |> sum()
let c = sum_w - sum_w2 / sum_w

let tau_sq = max(0, (q_stat - df_q) / c)

println("=== Random-Effects Meta-Analysis (DerSimonian-Laird) ===")
println("Tau-squared (between-study variance): " + str(round(tau_sq, 6)))
println("Tau (between-study SD): " + str(round(sqrt(tau_sq), 4)))

# Compute random-effects weights
let re_studies = log_or_studies |> map(|s| {
  name: s.name,
  effect: s.effect,
  se: s.se,
  re_variance: s.variance + tau_sq,
  re_weight: 1.0 / (s.variance + tau_sq),
  n: s.n
})

let re_total_weight = re_studies |> map(|s| s.re_weight) |> sum()
let re_weighted_sum = re_studies |> map(|s| s.re_weight * s.effect) |> sum()

let re_pooled_effect = re_weighted_sum / re_total_weight
let re_pooled_se = 1.0 / sqrt(re_total_weight)
let re_pooled_or = exp(re_pooled_effect)
let re_pooled_ci_lo = exp(re_pooled_effect - 1.96 * re_pooled_se)
let re_pooled_ci_hi = exp(re_pooled_effect + 1.96 * re_pooled_se)
let re_z = re_pooled_effect / re_pooled_se
let re_p = 2.0 * (1.0 - pnorm(abs(re_z)))

println("")
println("Pooled log(OR): " + str(round(re_pooled_effect, 4)))
println("Pooled SE: " + str(round(re_pooled_se, 4)))
println("Pooled OR: " + str(round(re_pooled_or, 3)))
println("95% CI: [" + str(round(re_pooled_ci_lo, 3)) + ", " +
        str(round(re_pooled_ci_hi, 3)) + "]")
println("P-value: " + str(round(re_p, 8)))

# Compare fixed vs random effects
println("")
println("=== Comparison ===")
println("Fixed-effects OR: " + str(round(pooled_or, 3)) +
        " [" + str(round(pooled_ci_lo, 3)) + ", " +
        str(round(pooled_ci_hi, 3)) + "]")
println("Random-effects OR: " + str(round(re_pooled_or, 3)) +
        " [" + str(round(re_pooled_ci_lo, 3)) + ", " +
        str(round(re_pooled_ci_hi, 3)) + "]")
println("")
println("Note: Random-effects CI is always wider than or equal to fixed-effects CI")
println("The random-effects estimate is more conservative — it accounts for")
println("the possibility that the true effect varies between studies")

# Weight redistribution
println("")
println("Weight redistribution:")
re_studies |> each(|s|
  let fe_pct = 100.0 * (1.0 / s.se / s.se) / total_weight
  let re_pct = 100.0 * s.re_weight / re_total_weight
  println("  " + s.name + ": FE=" + str(round(fe_pct, 1)) +
          "% -> RE=" + str(round(re_pct, 1)) + "%")
)
println("Random-effects gives relatively more weight to small studies")

Forest Plot: The Essential Visualization

The forest plot is the standard visualization for meta-analysis. It shows each study’s estimate with its confidence interval, the pooled estimate, and the relative weight of each study.

# Create the forest plot data

let fp_data = log_or_studies |> map(|s| {
  study: s.name + " (n=" + str(s.n) + ")",
  estimate: s.or,
  ci_lower: exp(s.effect - 1.96 * s.se),
  ci_upper: exp(s.effect + 1.96 * s.se),
  weight: round(100.0 * s.weight / total_weight, 1)
})

# Add the pooled estimates
let fp_pooled_fe = {
  study: "Fixed Effects (pooled)",
  estimate: pooled_or,
  ci_lower: pooled_ci_lo,
  ci_upper: pooled_ci_hi,
  weight: 100.0
}

let fp_pooled_re = {
  study: "Random Effects (pooled)",
  estimate: re_pooled_or,
  ci_lower: re_pooled_ci_lo,
  ci_upper: re_pooled_ci_hi,
  weight: 100.0
}

let all_fp_data = concat(fp_data, [fp_pooled_fe, fp_pooled_re])
let fp_table = table(all_fp_data)

forest_plot(fp_table, "study", "estimate", "ci_lower", "ci_upper",

  x_label: "Odds Ratio",
  null_value: 1.0)
  |> save_plot("meta_analysis_forest.svg")

println("Forest plot saved to meta_analysis_forest.svg")
println("The diamond represents the pooled estimate")
println("The vertical line at OR=1.0 represents the null (no effect)")

Combining P-Values: Fisher’s Method

When effect sizes are not available (common with older studies), you can still combine p-values. Fisher’s method uses the fact that under the null hypothesis, -2 * sum(log(p_i)) follows a chi-squared distribution with 2k degrees of freedom.

# Fisher's method for combining p-values

let pvalues = [0.02, 0.15, 0.005]

let fisher_stat = pvalues |> map(|p| -2.0 * log(p)) |> sum()
let fisher_df = 2 * len(pvalues)

# Chi-squared p-value approximation
# For large df, chi-squared is approximately normal
let fisher_z = (fisher_stat - fisher_df) / sqrt(2.0 * fisher_df)
let fisher_p = 1.0 - pnorm(fisher_z)

println("=== Fisher's Method for Combining P-Values ===")
println("Individual p-values: " + str(pvalues))
println("Test statistic: " + str(round(fisher_stat, 3)))
println("Degrees of freedom: " + str(fisher_df))
println("Combined p-value: " + str(round(fisher_p, 8)))
println("")

# Note: Fisher's method is very sensitive to a single strong result
# One study with p=0.001 can make the combined p-value significant
# even if other studies show nothing

Stouffer’s Method: Weighted P-Value Combination

Stouffer’s method converts p-values to z-scores, weights them (typically by the square root of the sample size), and combines:

Z_pooled = sum(w_i * Z_i) / sqrt(sum(w_i^2))

# Stouffer's weighted Z method

let study_ps = [
  { pvalue: 0.02,  n: 5000 },
  { pvalue: 0.15,  n: 3200 },
  { pvalue: 0.005, n: 8500 }
]

# Convert p-values to z-scores (one-sided)
let z_scores = study_ps |> map(|s| {
  z: qnorm(1.0 - s.pvalue / 2.0),   # two-sided p to z
  weight: sqrt(s.n),                  # weight by sqrt(n)
  n: s.n
})

let weighted_z_sum = z_scores |> map(|s| s.weight * s.z) |> sum()
let weight_ss = z_scores |> map(|s| s.weight * s.weight) |> sum()
let pooled_z_stouffer = weighted_z_sum / sqrt(weight_ss)
let stouffer_p = 2.0 * (1.0 - pnorm(abs(pooled_z_stouffer)))

println("=== Stouffer's Weighted Z Method ===")
z_scores |> each(|s|
  println("  Z=" + str(round(s.z, 3)) + " weight=" +
          str(round(s.weight, 1)) + " (n=" + str(s.n) + ")")
)
println("Pooled Z: " + str(round(pooled_z_stouffer, 3)))
println("Combined p-value: " + str(round(stouffer_p, 8)))
println("")
println("Stouffer's method is preferred over Fisher's when sample sizes vary")
println("because it gives more weight to larger studies")

Publication Bias: The Funnel Plot

Publication bias is the tendency for studies with statistically significant results to be published, while studies with null results remain in file drawers. This biases meta-analyses toward larger effect sizes.

The funnel plot visualizes this: it plots each study’s effect size against its precision (1/SE or sample size). In the absence of publication bias, the plot should be symmetric around the pooled estimate. Asymmetry — especially missing small-study null results in the lower left — suggests publication bias.

# Create a funnel plot

let funnel_data = log_or_studies |> map(|s| {
  effect: s.effect,          # log(OR) on x-axis
  precision: 1.0 / s.se,    # 1/SE on y-axis (more precise studies at top)
  study: s.name
})

let funnel_table = table(funnel_data)

scatter(funnel_table, "effect", "precision",
  title: "Funnel Plot: " + target_snp,
  x_label: "Effect Size (log OR)",
  y_label: "Precision (1/SE)")
  |> save_plot("funnel_plot.svg")

# Add reference lines
println("Funnel plot interpretation:")
println("  Vertical line at pooled effect = " + str(round(pooled_effect, 4)))
println("  Symmetric distribution: no publication bias")
println("  Missing lower-left: small negative studies unpublished")
println("  Missing lower-right: small positive studies unpublished")

# Egger's test for funnel plot asymmetry
# Regress the standard normal deviate (effect/SE) against precision (1/SE)
# Intercept significantly different from zero suggests bias

let egger_x = log_or_studies |> map(|s| 1.0 / s.se)
let egger_y = log_or_studies |> map(|s| s.effect / s.se)

let egger_result = lm(egger_y, egger_x)
println("")
println("Egger's test for publication bias:")
println("  Intercept: " + str(round(egger_result.intercept, 4)))
println("  P-value for intercept: " + str(round(egger_result.pvalue, 4)))

if egger_result.pvalue < 0.10 then
  println("  -> Evidence of funnel plot asymmetry (potential publication bias)")
else
  println("  -> No significant asymmetry detected")
end

Subgroup Analysis: Exploring Heterogeneity

When heterogeneity is detected, subgroup analysis investigates whether study-level characteristics explain the differences. Common subgroup variables in genomics:

  • Ancestry: European vs East Asian vs African
  • Study design: Case-control vs cohort
  • Genotyping platform: Array vs sequencing
  • Phenotype definition: Strict vs broad diagnostic criteria
# Subgroup analysis by ancestry
# Suppose we have ancestry information for each study

let studies_with_ancestry = [
  { name: "Study A", effect: log(1.35), se: 0.12, ancestry: "European", n: 5000 },
  { name: "Study B", effect: log(1.18), se: 0.15, ancestry: "East Asian", n: 3200 },
  { name: "Study C", effect: log(1.28), se: 0.08, ancestry: "European", n: 8500 },
  { name: "Study D", effect: log(1.42), se: 0.10, ancestry: "European", n: 6200 },
  { name: "Study E", effect: log(1.22), se: 0.18, ancestry: "East Asian", n: 2100 },
  { name: "Study F", effect: log(1.15), se: 0.20, ancestry: "African", n: 1800 }
]

let ancestries = unique(studies_with_ancestry |> map(|s| s.ancestry))

println("=== Subgroup Analysis by Ancestry ===")
ancestries |> each(|anc|
  let subgroup = studies_with_ancestry |> filter(|s| s.ancestry == anc)

  # Fixed-effects meta-analysis within subgroup
  let sg_weights = subgroup |> map(|s| 1.0 / (s.se * s.se))
  let sg_total_w = sg_weights |> sum()
  let sg_pooled = subgroup |> map(|s|
    (1.0 / (s.se * s.se)) * s.effect
  ) |> sum() / sg_total_w
  let sg_se = 1.0 / sqrt(sg_total_w)
  let sg_or = exp(sg_pooled)
  let sg_ci_lo = exp(sg_pooled - 1.96 * sg_se)
  let sg_ci_hi = exp(sg_pooled + 1.96 * sg_se)

          " [" + str(round(sg_ci_lo, 3)) + ", " +
          str(round(sg_ci_hi, 3)) + "]")
  println("")
)

The Winner’s Curse

The winner’s curse is a form of selection bias that inflates effect sizes in the discovery cohort. When you test thousands of variants and select the “winners” (most significant), you are preferentially selecting those where random noise happened to increase the apparent effect.

# Demonstrate the winner's curse with simulation

# True effect: OR = 1.2 (log OR = 0.182)
let true_log_or = log(1.2)
let true_or = 1.2

# Simulate 100 studies, each with different SE
let simulated = rnorm(100) |> map(|noise|
  let se = 0.05 + abs(rnorm(1)[0]) * 0.15  # random SE between 0.05 and ~0.20
  let observed_effect = true_log_or + noise * se  # noisy observation
  let z = observed_effect / se
  let p = 2.0 * (1.0 - pnorm(abs(z)))
  {
    effect: observed_effect,
    se: se,
    or: exp(observed_effect),
    pvalue: p,
    significant: p < 0.05
  }
)

# Compare significant vs non-significant studies
let sig_studies = simulated |> filter(|s| s.significant)
let nonsig_studies = simulated |> filter(|s| !s.significant)

let sig_mean_or = mean(sig_studies |> map(|s| s.or))
let nonsig_mean_or = mean(nonsig_studies |> map(|s| s.or))
let all_mean_or = mean(simulated |> map(|s| s.or))

println("=== Winner's Curse Demonstration ===")
println("True OR: " + str(true_or))
println("Mean OR across all 100 studies: " + str(round(all_mean_or, 3)))
println("Mean OR in significant studies only: " + str(round(sig_mean_or, 3)))
println("Mean OR in non-significant studies: " + str(round(nonsig_mean_or, 3)))
println("")
println("The significant studies overestimate the true effect by " +
        str(round(100 * (sig_mean_or - true_or) / true_or, 1)) + "%")
println("This is the winner's curse")
println("")
println("Implication: the first GWAS to discover an association will report")
println("an inflated effect size. Replication studies typically show smaller effects.")
println("Meta-analysis of all studies (including replications) corrects this bias.")

Multi-Trait Meta-Analysis

In genomics, we often want to combine evidence across related traits. For example, a variant may affect both fasting glucose and HbA1c — combining evidence across these correlated traits increases power.

# Multi-trait analysis: combine evidence across related phenotypes

let multi_trait = [
  { trait: "Type 2 diabetes", effect: log(1.30), se: 0.06, n: 15000 },
  { trait: "Fasting glucose", effect: 0.08, se: 0.02, n: 45000 },   # beta in mmol/L
  { trait: "HbA1c", effect: 0.05, se: 0.015, n: 50000 },            # beta in %
  { trait: "Fasting insulin", effect: 0.03, se: 0.025, n: 25000 }   # beta in pmol/L
]

# These are on different scales — cannot directly meta-analyze with IVW
# Convert to Z-scores for combination

let trait_zscores = multi_trait |> map(|t| {
  trait: t.trait,
  z: t.effect / t.se,
  pvalue: 2.0 * (1.0 - pnorm(abs(t.effect / t.se))),
  n: t.n
})

println("=== Multi-Trait Evidence ===")
trait_zscores |> each(|t|
  println("  " + t.trait + ": Z=" + str(round(t.z, 2)) +
          " p=" + str(round(t.pvalue, 8)))
)

# Combine using Stouffer's method (weighted by sqrt(n))
let multi_w = trait_zscores |> map(|t| sqrt(t.n))
let multi_wz = trait_zscores |> map(|t| sqrt(t.n) * t.z) |> sum()
let multi_wss = multi_w |> map(|w| w * w) |> sum()
let multi_z_pooled = multi_wz / sqrt(multi_wss)
let multi_p = 2.0 * (1.0 - pnorm(abs(multi_z_pooled)))

println("")
println("Multi-trait combined Z: " + str(round(multi_z_pooled, 2)))
println("Multi-trait combined p: " + str(round(multi_p, 12)))
println("")
println("Caution: traits are correlated, so this overstates significance")
println("Proper multi-trait methods (MTAG, ASSET) account for trait correlations")

Sensitivity Analysis

Robust meta-analyses include sensitivity analyses to check whether the results depend on any single study:

# Leave-one-out sensitivity analysis
println("=== Leave-One-Out Analysis ===")
println("Removing each study in turn to check robustness:")
println("")

  let remaining = log_or_studies |> filter(|s| s.name != excluded.name)

)

Building a Complete Meta-Analysis Pipeline

# Complete meta-analysis pipeline function

# Step 1: Collect study-level data
let all_studies = [
  { name: "Grant 2006", effect: log(1.45), se: 0.10, n: 3500, ancestry: "European" },
  { name: "Scott 2007", effect: log(1.37), se: 0.08, n: 5500, ancestry: "European" },
  { name: "Sladek 2007", effect: log(1.40), se: 0.12, n: 2800, ancestry: "European" },
  { name: "DIAGRAM 2008", effect: log(1.36), se: 0.05, n: 12000, ancestry: "European" },
  { name: "Takeuchi 2009", effect: log(1.25), se: 0.14, n: 3000, ancestry: "East Asian" },
  { name: "Unoki 2008", effect: log(1.20), se: 0.11, n: 4200, ancestry: "East Asian" },
  { name: "Waters 2010", effect: log(1.30), se: 0.18, n: 1500, ancestry: "African" },
  { name: "DIAGRAM 2012", effect: log(1.32), se: 0.04, n: 22000, ancestry: "European" },
  { name: "Mahajan 2018", effect: log(1.29), se: 0.03, n: 74000, ancestry: "Multi-ethnic" }
]

# Step 2: Fixed-effects analysis
let fe_w = all_studies |> map(|s| 1.0 / (s.se * s.se))
let fe_total = fe_w |> sum()
let fe_pooled = all_studies |> map(|s| (1.0 / (s.se * s.se)) * s.effect) |> sum() / fe_total
let fe_se = 1.0 / sqrt(fe_total)

# Step 3: Heterogeneity
let q = all_studies |> map(|s|
  let w = 1.0 / (s.se * s.se)
  w * pow(s.effect - fe_pooled, 2)
) |> sum()
let df = len(all_studies) - 1
let i2 = max(0, (q - df) / q) * 100.0

# Step 4: Random-effects if needed
let c_val = fe_total - (fe_w |> map(|w| w * w) |> sum()) / fe_total
let tau2 = max(0, (q - df) / c_val)
let re_w = all_studies |> map(|s| 1.0 / (s.se * s.se + tau2))
let re_total = re_w |> sum()
let re_pooled = all_studies |> map(|s|
  (1.0 / (s.se * s.se + tau2)) * s.effect
) |> sum() / re_total
let re_se_val = 1.0 / sqrt(re_total)

# Step 5: Report
println("=== Complete Meta-Analysis Report ===")
println("Variant: rs7903146 (TCF7L2)")
println("Trait: Type 2 Diabetes")
println("Studies: " + str(len(all_studies)))
println("Total sample: " + str(all_studies |> map(|s| s.n) |> sum()))
println("")
println("Fixed-effects: OR=" + str(round(exp(fe_pooled), 3)) +
        " [" + str(round(exp(fe_pooled - 1.96 * fe_se), 3)) + ", " +
        str(round(exp(fe_pooled + 1.96 * fe_se), 3)) + "]")
println("Random-effects: OR=" + str(round(exp(re_pooled), 3)) +
        " [" + str(round(exp(re_pooled - 1.96 * re_se_val), 3)) + ", " +
        str(round(exp(re_pooled + 1.96 * re_se_val), 3)) + "]")
println("Heterogeneity: Q=" + str(round(q, 2)) + " I2=" +
        str(round(i2, 1)) + "%")
println("Tau2=" + str(round(tau2, 6)))

# Step 6: Forest plot
let final_fp = all_studies |> map(|s| {
  study: s.name + " (" + s.ancestry + ")",
  estimate: exp(s.effect),
  ci_lower: exp(s.effect - 1.96 * s.se),
  ci_upper: exp(s.effect + 1.96 * s.se),
  weight: round(100.0 * (1.0 / (s.se * s.se)) / fe_total, 1)
})

let final_fp = concat(final_fp, [{
  study: "RE Pooled",
  estimate: exp(re_pooled),
  ci_lower: exp(re_pooled - 1.96 * re_se_val),
  ci_upper: exp(re_pooled + 1.96 * re_se_val),
  weight: 100.0
}])

forest_plot(table(final_fp), "study", "estimate", "ci_lower", "ci_upper",

Interpretation

Reading a Forest Plot

The forest plot tells you everything you need to know at a glance:

  1. Individual study squares: Size proportional to weight. Position shows the point estimate. Horizontal lines are 95% CIs.
  2. The diamond: The pooled estimate. Width represents the pooled CI.
  3. The null line: Vertical line at OR=1 (or beta=0). If a CI crosses this line, that study’s result is not significant.
  4. Overlap: If all CIs overlap substantially, there is little heterogeneity. If some CIs are completely separated from others, the studies genuinely disagree.

When Studies Disagree

Heterogeneity is not always bad. It can reveal important biology:

  • Ancestry-specific effects: A variant may have OR=1.4 in Europeans but OR=1.1 in East Asians due to different linkage disequilibrium patterns or gene-environment interactions.
  • Phenotype heterogeneity: “Type 2 diabetes” diagnosed by fasting glucose vs oral glucose tolerance test may capture different patient populations.
  • Winner’s curse: Early studies with smaller samples report larger effects. This is expected and should decrease over time.
# Investigating heterogeneity sources

# Stratify by ancestry
let european = all_studies |> filter(|s| s.ancestry == "European")
let east_asian = all_studies |> filter(|s| s.ancestry == "East Asian")

let eur_w = european |> map(|s| 1.0 / (s.se * s.se))
let eur_total = eur_w |> sum()
let eur_pooled = european |> map(|s| (1.0 / (s.se * s.se)) * s.effect) |> sum() / eur_total

let ea_w = east_asian |> map(|s| 1.0 / (s.se * s.se))
let ea_total = ea_w |> sum()
let ea_pooled = east_asian |> map(|s| (1.0 / (s.se * s.se)) * s.effect) |> sum() / ea_total

println("Ancestry-stratified results:")
println("  European: OR=" + str(round(exp(eur_pooled), 3)))
println("  East Asian: OR=" + str(round(exp(ea_pooled), 3)))

# Test for difference between subgroups
let diff = eur_pooled - ea_pooled
let diff_se = sqrt(1.0 / eur_total + 1.0 / ea_total)
let diff_z = diff / diff_se
let diff_p = 2.0 * (1.0 - pnorm(abs(diff_z)))

println("  Difference test: Z=" + str(round(diff_z, 2)) +
        " p=" + str(round(diff_p, 4)))

if diff_p < 0.05 then
  println("  -> Effect size differs significantly between ancestries")
else
  println("  -> No significant difference between ancestries")
end

Pitfalls

Pitfall 1: Combining Incompatible Effect Sizes

You cannot meta-analyze odds ratios from one study with beta coefficients from another without proper conversion. You cannot combine standardized effect sizes (Cohen’s d) with unstandardized effect sizes. Always verify that all studies are on the same scale.

# Common scale conversions

# Odds ratio to log odds ratio (for meta-analysis)
let or_value = 1.35
let log_or = log(or_value)
println("OR " + str(or_value) + " -> log(OR) = " + str(round(log_or, 4)))

# Cohen's d to log odds ratio (approximate)
let d = 0.5
let log_or_from_d = d * 3.14159 / sqrt(3)
println("Cohen's d " + str(d) + " -> log(OR) ≈ " + str(round(log_or_from_d, 4)))

# Correlation r to Cohen's d (approximate)
let r = 0.3
let d_from_r = 2 * r / sqrt(1 - r * r)
println("r " + str(r) + " -> Cohen's d ≈ " + str(round(d_from_r, 4)))

Pitfall 2: Ignoring Study Overlap

If two meta-analyzed studies share participants (e.g., both used UK Biobank), treating them as independent inflates the effective sample size and produces falsely precise pooled estimates. Always check for sample overlap.

Pitfall 3: The File Drawer Problem

You can only meta-analyze published studies. If negative results remain unpublished, your meta-analysis overestimates the true effect. The funnel plot and Egger’s test help detect this, but they cannot fix it.

Pitfall 4: Confusing Statistical Significance with Clinical Significance

A meta-analysis of 100,000 participants may produce a p-value of 1e-30 for an odds ratio of 1.05. Statistically significant — but an OR of 1.05 means a 5% increase in risk. Is that clinically meaningful?

# Distinguish statistical from clinical significance

let examples = [
  { or: 1.05, p: 1e-30, n: 100000, interpretation: "Tiny effect, huge study" },
  { or: 1.50, p: 0.04,  n: 200,    interpretation: "Moderate effect, small study" },
  { or: 2.00, p: 1e-15, n: 5000,   interpretation: "Large effect, well-powered" }
]

println("Statistical vs clinical significance:")
examples |> each(|e|
  println("  OR=" + str(e.or) + " p=" + str(e.p) + " n=" + str(e.n))
  println("    " + e.interpretation)
  println("")
)

Pitfall 5: Cherry-Picking Studies

Including or excluding studies based on their results (rather than predefined criteria) is a form of p-hacking. The inclusion criteria must be defined before looking at the data. This is why systematic reviews require a registered protocol.

Where Next?

Mendelian randomization — A causal inference method that uses genetic variants as “instruments” to estimate causal effects. It is a specialized form of meta-analysis where you combine multiple genetic instruments, each with an exposure effect and an outcome effect, to estimate the causal relationship between exposure and outcome.

Network meta-analysis — When multiple treatments have been compared in different studies (Drug A vs placebo, Drug B vs placebo, Drug A vs Drug B), network meta-analysis combines both direct and indirect comparisons to rank all treatments simultaneously.

Mega-analysis vs meta-analysis — Instead of combining summary statistics from published studies, mega-analysis combines individual-level data from all studies. This is more powerful (you can adjust for individual-level covariates) but requires data sharing agreements. The PGC and UK Biobank increasingly enable mega-analysis.

Multi-ancestry meta-analysis — Methods like MANTRA and MR-MEGA account for ancestry-specific linkage disequilibrium patterns when combining GWAS across ancestries. These are critical for identifying causal variants (which are shared across ancestries) vs tag variants (which are ancestry-specific).

Prediction Intervals in Random-Effects Models

The confidence interval of a random-effects meta-analysis tells you the uncertainty in the average effect. But the prediction interval tells you the range in which a future study’s true effect is likely to fall. The prediction interval is always wider than the confidence interval because it includes both the uncertainty in the mean AND the between-study variance.

# Prediction interval for a random-effects meta-analysis
# PI = pooled +/- t(df=k-2) * sqrt(tau2 + SE_pooled^2)

let t_crit = qnorm(0.975)  # approximate for df > 5
let pi_se = sqrt(tau2 + re_se_val * re_se_val)
let pi_lo = exp(re_pooled - t_crit * pi_se)
let pi_hi = exp(re_pooled + t_crit * pi_se)

println("=== Prediction Interval ===")
println("95% CI for the average effect: [" +
        str(round(re_pooled_ci_lo, 3)) + ", " +
        str(round(re_pooled_ci_hi, 3)) + "]")
println("95% PI for a new study: [" +
        str(round(pi_lo, 3)) + ", " + str(round(pi_hi, 3)) + "]")
println("")
println("The CI tells you: where is the average effect?")
println("The PI tells you: what would a new study likely find?")
println("")
println("If the PI includes 1.0 (for OR), a new study might find no effect")
println("even if the pooled estimate is significant")

Trim-and-Fill Method for Publication Bias Adjustment

When the funnel plot is asymmetric (suggesting publication bias), the trim-and-fill method estimates how many studies are missing and imputes their results to produce a bias-corrected pooled estimate.

# Trim-and-fill concept

# 2. "Fill" the funnel with imputed mirror-image studies
# 3. Recompute the pooled estimate

# The adjusted estimate is always closer to the null (OR = 1.0)
# than the original estimate

println("  1. Identifying the most extreme studies on the asymmetric side")
println("  2. Removing ('trimming') them to estimate the true center")
println("  3. Replacing them and adding their mirror images ('filling')")
println("  4. Recomputing the pooled estimate with all studies (real + imputed)")

Effect Size Inflation and Correction

The winner’s curse can be formally corrected using conditional likelihood methods or bootstrap approaches:

# Simple winner's curse correction
# The conditional MLE adjusts the effect size for selection bias

# If a variant was selected because |z| > z_threshold:
let z_threshold = qnorm(1.0 - 5e-8 / 2)  # genome-wide significance
println("Genome-wide significance z-threshold: " + str(round(z_threshold, 2)))

# Observed z-score for a hit
let observed_z = 6.5
let observed_se = 0.05
let observed_beta = observed_z * observed_se

# Expected true effect (conditional MLE approximation)
# The true beta is approximately: observed_beta * (1 - threshold^2 / z^2)
let correction_factor = 1.0 - (z_threshold * z_threshold) / (observed_z * observed_z)
let corrected_beta = observed_beta * max(correction_factor, 0.0)

println("Observed beta: " + str(round(observed_beta, 4)))
println("Correction factor: " + str(round(correction_factor, 4)))
println("Corrected beta: " + str(round(corrected_beta, 4)))
println("Shrinkage: " + str(round(100 * (1 - correction_factor), 1)) + "%")
println("")
println("Variants barely passing the threshold are most inflated")
println("Variants far exceeding the threshold are least affected")

ConceptKey Takeaway

I-squaredQuantifies heterogeneity: 0%=none, 50%=moderate, 75%=high
