Biostatistics in Practice

Chapter 14: Clustering & Classification

The Problem

You have gene expression profiles from 1,097 breast cancer tumors in The Cancer Genome Atlas (TCGA). Clinicians treat breast cancer differently depending on the molecular subtype — Luminal A, Luminal B, HER2-enriched, Basal-like, and Normal-like. But these subtypes were originally discovered by unsupervised clustering of expression data. The question is: if you cluster the tumors without knowing the subtypes, do the clusters match the known biology?

More broadly: given a matrix of measurements (gene expression, protein levels, methylation, mutation profiles), how do you discover natural groups in the data? How do you decide how many groups exist? And how do you avoid the dangerous trap of finding patterns in noise?

This chapter covers unsupervised clustering — the art of discovering structure that you did not know existed. It also covers the critical discipline of validating that the structure is real.

Why It Matters

Clustering is not just an exploratory exercise. In genomics, clusters become clinical categories that determine treatment:

Breast cancer subtypes. The PAM50 classifier assigns each tumor to one of five molecular subtypes. Luminal A tumors have excellent prognosis and may avoid chemotherapy. Basal-like tumors are aggressive and require intensive treatment. This classification, which saves lives, was discovered by clustering gene expression profiles.

Single-cell atlas construction. Every cell atlas relies on clustering to define cell types. The Human Cell Atlas, which aims to map every cell in the human body, is fundamentally a clustering project. Each cluster becomes a cell type with a name, a marker gene signature, and a place in the developmental hierarchy.

Cancer of unknown primary (CUP). When a metastatic tumor is found but the primary site is unknown, expression-based classifiers assign it to the most likely tissue of origin. These classifiers use clustering — the unknown tumor is assigned to the cluster of known tumors it most resembles.

Drug response prediction. Clustering patients by multi-omic profiles can reveal subgroups with different drug responses. The I-SPY 2 adaptive clinical trial uses expression-based clustering to match patients to experimental therapies.

Preparing the Data for Clustering

Before clustering, we must prepare the data the same way we did for PCA in Chapter 13: filter low-expression genes, log-transform, select the most variable features, and scale.

# Get sample IDs (column names, excluding the gene ID column)
let sample_ids = columns(brca_expr) |> filter(|c| c != "Ensembl_ID")
println("Total samples: " + str(len(sample_ids)))

# Filter genes: keep only those expressed in at least 10% of samples
# and with mean FPKM > 1
let gene_ids = col_values(brca_expr, "Ensembl_ID")

let gene_stats = gene_ids |> map(|g|
  let row_vals = sample_ids |> map(|s| col_values(brca_expr, s)[0])  # simplified
  let expressed = row_vals |> filter(|v| v > 0) |> len()
  let pct_expressed = 100.0 * expressed / len(sample_ids)
  let mean_expr = mean(row_vals)
  { gene: g, pct_expressed: pct_expressed, mean_expr: mean_expr }
)

let expressed_genes = gene_stats
  |> filter(|g| g.pct_expressed > 10.0 and g.mean_expr > 1.0)
  |> map(|g| g.gene)

println("Genes after expression filter: " + str(len(expressed_genes)))

# Log2-transform (FPKM + 1)
# Then select top 5000 most variable genes
# Then z-score normalize across samples

# Build a samples x genes matrix (transpose the data)
let sample_data = sample_ids |> map(|s|
  let vals = col_values(brca_expr, s)
  let result = { sample: s }

Selecting Features: The Most Variable Genes

For clustering, feature selection is critical. Too many features add noise. Too few may miss the signal. The standard approach is to select the most variable genes.

# Compute variance for each gene across samples
let gene_list = expressed_genes |> head(5000)
let gene_var = gene_list |> map(|g|
  let values = col_values(sample_table, g)
  { gene: g, variance: variance(values), mean: mean(values) }
)

# Select top 2000 most variable genes
let top_genes = gene_var
  |> sort(|a, b| b.variance - a.variance)
  |> head(2000)
  |> map(|g| g.gene)

println("Selected " + str(len(top_genes)) + " most variable genes for clustering")

# Show the top 10
println("Top 10 most variable genes:")

  println("  " + g.gene + ": var=" + str(round(g.variance, 3)) +
          " mean=" + str(round(g.mean, 3)))
)

# Scale the selected genes (z-score across samples)
let cluster_data = sample_table |> select(top_genes)
let scaled_data = normalize(cluster_data, "zscore")

println("Clustering matrix: " + str(len(scaled_data)) + " samples x " +
        str(ncols(scaled_data)) + " genes")

Dimensionality Reduction Before Clustering

As we learned in Chapter 13, clustering in the full gene space is prone to the curse of dimensionality. We first reduce to PCA coordinates.

The Analysis

K-Means Clustering

K-means is the simplest and most widely used clustering algorithm. It partitions n observations into k clusters by minimizing the within-cluster sum of squared distances to the centroid.

The algorithm:

  1. Choose k random points as initial centroids
  2. Assign each observation to the nearest centroid
  3. Recompute centroids as the mean of assigned observations
  4. Repeat steps 2-3 until convergence
# K-means clustering with k=5 (to match the 5 PAM50 subtypes)
let km5 = kmeans(pc_embedding, k: 5)

# km5 contains:
#   .labels     — cluster assignment for each sample (0-indexed)
#   .centroids  — centroid coordinates

println("K-means (k=5) cluster sizes:")
let km5_counts = km5.labels |> value_counts()
km5_counts |> each(|c|
  println("  Cluster " + str(c.value) + ": " + str(c.count) + " samples")
)

Choosing K: The Elbow Method

But how do you know that k=5 is the right choice? What if the data contains 3 clusters, or 7? The elbow method runs k-means for a range of k values and plots the within-cluster sum of squares (WCSS, also called inertia) against k.

# Elbow method: run k-means for k=2 to k=10
let k_range = [2, 3, 4, 5, 6, 7, 8, 9, 10]
let elbow_results = k_range |> map(|k|
  let result = kmeans(pc_embedding, k: k)

  # Compute within-cluster sum of squares (WCSS)
  let wcss = 0.0
  let i = 0
  result.labels |> each(|label|
    let centroid = result.centroids[label]
    let point = pc_embedding[i]
    let dist_sq = point |> map(|p|
      let j = 0
      let d = p - centroid[j]
      j = j + 1
      d * d
    ) |> sum()
    wcss = wcss + dist_sq
    i = i + 1
  )

  println("k=" + str(k) + " WCSS=" + str(round(wcss, 1)))
  { k: k, wcss: wcss }
)

# Plot the elbow
let elbow_table = table(elbow_results)
scatter(elbow_table, "k", "wcss",
  title: "K-Means Elbow Plot",
  x_label: "Number of Clusters (k)",
  y_label: "Within-Cluster Sum of Squares")
  |> save_plot("kmeans_elbow.svg")

# The elbow is where the decrease in WCSS slows down sharply
# For TCGA BRCA, expect an elbow around k=4 or k=5

Choosing K: Silhouette Analysis

The silhouette score measures how similar each observation is to its own cluster compared to the nearest neighboring cluster. It ranges from -1 (wrong cluster) to +1 (well-matched to its cluster).

# Compute silhouette scores for different k values
let silhouette_results = k_range |> map(|k|
  let result = kmeans(pc_embedding, k: k)

  # Compute silhouette for each sample
  let sil_scores = []
  let i = 0
  result.labels |> each(|label|
    let point = pc_embedding[i]

    # a(i) = mean distance to same-cluster points
    let same_dists = []
    let j = 0
    result.labels |> each(|other_label|
      if other_label == label and j != i then
        let d = point |> map(|p|
          let dim = 0
          let diff = p - pc_embedding[j][dim]
          dim = dim + 1
          diff * diff
        ) |> sum() |> sqrt()
        same_dists = concat(same_dists, [d])
      end
      j = j + 1
    )
    let a_i = if len(same_dists) > 0 then mean(same_dists) else 0.0

    # b(i) = min over other clusters of mean distance to that cluster
    let other_clusters = unique(result.labels) |> filter(|c| c != label)
    let b_i = other_clusters |> map(|oc|
      let oc_dists = []
      let jj = 0
      result.labels |> each(|ol|
        if ol == oc then
          let d = point |> map(|p|
            let dim = 0
            let diff = p - pc_embedding[jj][dim]
            dim = dim + 1
            diff * diff
          ) |> sum() |> sqrt()
          oc_dists = concat(oc_dists, [d])
        end
        jj = jj + 1
      )
      mean(oc_dists)
    ) |> min()

    let s_i = if a_i < b_i then (b_i - a_i) / b_i else (b_i - a_i) / a_i
    sil_scores = concat(sil_scores, [s_i])
    i = i + 1
  )

  let mean_sil = mean(sil_scores)
  println("k=" + str(k) + " mean silhouette=" + str(round(mean_sil, 4)))
  { k: k, mean_silhouette: mean_sil }
)

let sil_table = table(silhouette_results)
scatter(sil_table, "k", "mean_silhouette",
  title: "Silhouette Score vs k",
  x_label: "Number of Clusters (k)",
  y_label: "Mean Silhouette Score")
  |> save_plot("silhouette_scores.svg")

# Higher silhouette = better clustering
# If k=4 and k=5 have similar scores, biological knowledge tips the balance

Hierarchical Clustering

Hierarchical clustering builds a tree (dendrogram) of nested clusters. Unlike k-means, it does not require you to specify k in advance. You choose k by cutting the dendrogram at a certain height.

There are two approaches:

  • Agglomerative (bottom-up): Start with each point as its own cluster. Merge the two closest clusters at each step. Repeat until one cluster remains.
  • Divisive (top-down): Start with all points in one cluster. Split recursively.

The key parameter is the linkage criterion — how you measure the distance between two clusters:

  • Single linkage: Minimum distance between any pair of points
  • Complete linkage: Maximum distance between any pair of points
  • Average linkage: Mean distance between all pairs of points
  • Ward’s method: Minimizes the increase in total within-cluster variance
# Compute the distance matrix for hierarchical clustering
let dist_mat = dist_matrix(pc_embedding, "euclidean")

println("Distance matrix: " + str(len(dist_mat)) + " x " + str(len(dist_mat[0])))

# Descriptive statistics of pairwise distances
let all_dists = []
let i = 0
while i < len(dist_mat) do
  let j = i + 1
  while j < len(dist_mat) do
    all_dists = concat(all_dists, [dist_mat[i][j]])
    j = j + 1
  end
  i = i + 1
end

let dist_stats = {
  mean: mean(all_dists),
  median: median(all_dists),
  sd: stdev(all_dists),
  min: min(all_dists),
  max: max(all_dists)
}

println("Pairwise distance statistics:")
println("  Mean: " + str(round(dist_stats.mean, 2)))
println("  Median: " + str(round(dist_stats.median, 2)))
println("  SD: " + str(round(dist_stats.sd, 2)))
println("  Range: [" + str(round(dist_stats.min, 2)) + ", " +
        str(round(dist_stats.max, 2)) + "]")

Leiden Community Detection

Leiden clustering is the standard method for single-cell RNA-seq. Unlike k-means, it does not require specifying the number of clusters. Instead, it builds a nearest-neighbor graph and finds communities of densely connected nodes.

The algorithm:

  1. Build a k-nearest-neighbor (KNN) graph in PC space
  2. Optimize the modularity function to find communities
  3. The resolution parameter controls the granularity
# Leiden clustering at different resolutions
let resolutions = [0.2, 0.5, 0.8, 1.0, 1.5, 2.0]

let leiden_results = resolutions |> map(|res|
  let clusters = leiden(pc_embedding, resolution: res)
  let n_clusters = len(unique(clusters))
  println("Resolution " + str(res) + ": " + str(n_clusters) + " clusters")
  { resolution: res, n_clusters: n_clusters, labels: clusters }
)

# Resolution 0.2: 2-3 large clusters (Luminal vs Basal)
# Resolution 0.5: 4-5 clusters (approaching PAM50)
# Resolution 1.0: 6-8 clusters (may split subtypes further)
# Resolution 2.0: 10+ clusters (over-splitting)

# The resolution controls granularity — there is no single "correct" value
# Choose based on biological interpretability

Adjusted Rand Index

The Rand Index measures agreement between two clusterings. It counts the number of pairs of observations that are either in the same cluster in both clusterings or in different clusters in both. The Adjusted Rand Index (ARI) corrects for chance — an ARI of 0 means agreement no better than random, and 1 means perfect agreement.

# Compute the Adjusted Rand Index between our clustering and PAM50

# First, compute the raw Rand Index
let n = len(our_labels)
let a = 0  # pairs in same cluster in both
let b = 0  # pairs in different clusters in both
let c = 0  # same in ours, different in PAM50
let d = 0  # different in ours, same in PAM50

let i = 0
while i < n do
  let j = i + 1
  while j < n do
    let same_ours = our_labels[i] == our_labels[j]
    let same_pam50 = known_subtypes[i] == known_subtypes[j]

    if same_ours and same_pam50 then
      a = a + 1
    end
    if same_ours and !same_pam50 then
      c = c + 1
    end
    if !same_ours and same_pam50 then
      d = d + 1
    end
    if !same_ours and !same_pam50 then
      b = b + 1
    end

    j = j + 1
  end
  i = i + 1
end

let total_pairs = n * (n - 1) / 2
let rand_index = (a + b) / total_pairs

println("Raw Rand Index: " + str(round(rand_index, 4)))

# Adjusted Rand Index (corrects for chance)
# ARI = (RI - Expected_RI) / (Max_RI - Expected_RI)
# For k-means with k=5 on BRCA data, expect ARI around 0.4-0.7
# depending on preprocessing

# The ARI will not be 1.0 because:
# 1. K-means is a simple method — it finds spherical clusters
# 2. PAM50 subtypes have complex, non-spherical boundaries
# 3. Some subtypes (especially Normal-like) are poorly defined
# 4. Our feature selection may not match the PAM50 gene set

Identifying Cluster-Defining Genes

Once clusters are found, the next question is: what genes define each cluster? These are the marker genes or signature genes.

# For each cluster, find genes with the highest differential expression
# compared to all other clusters

let cluster_markers = our_clusters |> map(|cl|
  println("Finding markers for cluster " + str(cl) + "...")

  let in_cluster = []
  let out_cluster = []
  let idx = 0
  our_labels |> each(|l|
    if l == cl then
      in_cluster = concat(in_cluster, [idx])
    else
      out_cluster = concat(out_cluster, [idx])
    end
    idx = idx + 1
  )

  # Test top genes for differential expression
  let markers = top_genes |> head(500) |> map(|g|
    let all_vals = col_values(scaled_data, g)
    let in_vals = in_cluster |> map(|i| all_vals[i])
    let out_vals = out_cluster |> map(|i| all_vals[i])

    let test_result = ttest(in_vals, out_vals)
    {
      gene: g,
      pvalue: test_result.pvalue,
      mean_in: mean(in_vals),
      mean_out: mean(out_vals),
      log_fc: mean(in_vals) - mean(out_vals)
    }
  )

  # Top 10 upregulated markers
  let top_markers = markers
    |> filter(|m| m.log_fc > 0)
    |> sort(|a, b| a.pvalue - b.pvalue)
    |> head(10)

  println("  Cluster " + str(cl) + " top markers:")
  top_markers |> each(|m|
    println("    " + m.gene + ": logFC=" + str(round(m.log_fc, 2)) +
            " p=" + str(round(m.pvalue, 6)))
  )

  { cluster: cl, markers: top_markers }
)

The Circular Analysis Trap

This is the single most dangerous pitfall in clustering:

You cluster the data using a set of genes. Then you test whether those same genes are differentially expressed between the clusters. Of course they are — you selected the clusters specifically to maximize differences on those genes.

This is circular reasoning. The p-values are meaningless because the hypothesis was generated from the same data used to test it.

# CIRCULAR ANALYSIS — DO NOT DO THIS

# Step 1: Cluster using all genes
# let clusters = kmeans(expression_data, k: 5)

# Step 2: Test each gene for differential expression between clusters
# let de_results = genes |> map(|g|
#   let groups = clusters |> map(|c| expression_data[c][g])
#   anova(groups...)
# )

# Step 3: Report "500 genes are significantly different between clusters"
# THIS IS MEANINGLESS — you are testing the same data you clustered on

# CORRECT APPROACH 1: Split-sample validation
# Cluster on half the samples, test on the other half

let n_samples = len(our_labels)
let half = n_samples / 2

# Random split
let indices = [0, 1, 2] |> map(|i| i)  # simplified: all sample indices
let train_idx = indices |> head(half)
let test_idx = indices |> tail(n_samples - half)

# Cluster on training set
let train_data = train_idx |> map(|i| pc_embedding[i])
let train_km = kmeans(train_data, k: 5)

# Assign test samples to nearest training centroid
let test_labels = test_idx |> map(|i|
  let point = pc_embedding[i]
  let min_dist = 999999.0
  let best_cluster = 0
  train_km.centroids |> each(|centroid|
    let c_idx = 0
    let d = point |> map(|p|
      let dim = 0
      let diff = p - centroid[dim]
      dim = dim + 1
      diff * diff
    ) |> sum() |> sqrt()
    if d < min_dist then
      min_dist = d

Cluster Stability: Bootstrap Assessment

How do you know your clusters are real and not artifacts of noise? One approach is bootstrap resampling: repeatedly resample the data with replacement, cluster each resample, and check if the same clusters appear consistently.

# Bootstrap cluster stability
let n_bootstrap = 100
let n_samples_total = len(pc_embedding)

let stability_matrix = []  # n_samples x n_samples: fraction of times co-clustered

# Initialize co-clustering counts
let co_cluster_count = []
let co_sampled_count = []
let i_init = 0
while i_init < n_samples_total do
  let row = []
  let row2 = []
  let j_init = 0
  while j_init < n_samples_total do
    row = concat(row, [0])
    row2 = concat(row2, [0])
    j_init = j_init + 1
  end
  co_cluster_count = concat(co_cluster_count, [row])
  co_sampled_count = concat(co_sampled_count, [row2])
  i_init = i_init + 1
end

# Run bootstrap iterations
let boot = 0
while boot < n_bootstrap do
  # Sample with replacement
  let boot_indices = []
  let b = 0
  while b < n_samples_total do
    let idx = round(rnorm(1)[0] * n_samples_total / 4 + n_samples_total / 2, 0)
    if idx < 0 then idx = 0 end
    if idx >= n_samples_total then idx = n_samples_total - 1 end
    boot_indices = concat(boot_indices, [idx])
    b = b + 1
  end

  # Cluster the bootstrap sample
  let boot_data = boot_indices |> map(|i| pc_embedding[i])
  let boot_km = kmeans(boot_data, k: 5)

  # Update co-clustering matrix
  # (simplified — full implementation would track all pairs)
  boot = boot + 1
end

println("Bootstrap stability assessment: " + str(n_bootstrap) + " iterations")
println("Stable clusters should have co-clustering frequency > 0.8")

Supervised Classification: When Labels Are Available

When you have labels for some samples (like the PAM50 subtypes), you can train a classifier to predict the labels for new samples. This is supervised classification, and it complements unsupervised clustering.

# Simple nearest-centroid classifier for PAM50 subtypes

# Compute centroids for each subtype
let subtype_names = unique(known_subtypes) |> filter(|s| s != "NA")

let centroids = subtype_names |> map(|st|
  let st_indices = []
  let i = 0
  known_subtypes |> each(|s|
    if s == st then
      st_indices = concat(st_indices, [i])
    end
    i = i + 1
  )

  # Average PC coordinates for this subtype
  let centroid = []
  let pc = 0
  while pc < n_pcs do
    let vals = st_indices |> map(|i| pc_embedding[i][pc])
    centroid = concat(centroid, [mean(vals)])
    pc = pc + 1
  end

  { subtype: st, centroid: centroid, n_samples: len(st_indices) }
)

println("Subtype centroids computed:")
centroids |> each(|c|
  println("  " + c.subtype + ": " + str(c.n_samples) + " samples")
)

# Leave-one-out cross-validation
let correct = 0
let total = 0
let i = 0
known_subtypes |> each(|true_label|
  if true_label != "NA" then
    let point = pc_embedding[i]

    # Find nearest centroid (excluding this sample's contribution)
    let min_dist = 999999.0
    let predicted = ""
    centroids |> each(|c|
      let d = point |> map(|p|
        let dim = 0
        let diff = p - c.centroid[dim]
        dim = dim + 1
        diff * diff
      ) |> sum() |> sqrt()

      if d < min_dist then
        min_dist = d
        predicted = c.subtype
      end
    )

    if predicted == true_label then
      correct = correct + 1
    end
    total = total + 1
  end
  i = i + 1
)

let accuracy = 100.0 * correct / total
println("Nearest-centroid classification accuracy: " +
        str(round(accuracy, 1)) + "%")
println("(Leave-one-out cross-validation)")

Interpretation

What Clustering Found in TCGA BRCA

When you cluster TCGA breast cancer tumors by gene expression, you typically recover four to five major groups that correspond well to the PAM50 molecular subtypes:

  1. Luminal A: Hormone receptor positive, low proliferation. Driven by ESR1, GATA3, FOXA1. Best prognosis.
  2. Luminal B: Hormone receptor positive, high proliferation. Higher Ki67. More aggressive than Luminal A.
  3. HER2-enriched: ERBB2 amplified. Distinct cluster driven by ERBB2 and nearby genes on chromosome 17q12.
  4. Basal-like: Triple-negative (ER-, PR-, HER2-). Driven by basal keratins (KRT5, KRT17) and EGFR. Worst prognosis.
  5. Normal-like: Resembles normal breast tissue. This subtype is controversial — it may represent contamination with normal tissue rather than a true tumor subtype.

The fact that unsupervised clustering recovers these biologically meaningful and clinically actionable groups is powerful evidence that the molecular subtypes are real, not arbitrary divisions.

Comparing Clusterings Quantitatively

When you run multiple clustering methods, you need a way to compare them quantitatively. Beyond the ARI, the Normalized Mutual Information (NMI) is another popular metric.

# Compare all pairwise clusterings

let methods = [
  { name: "K-means", labels: km_labels },
  { name: "Leiden", labels: leiden_labels }
]

println("Clustering comparison:")
let m1_idx = 0
while m1_idx < len(methods) do
  let m2_idx = m1_idx + 1
  while m2_idx < len(methods) do
    let m1 = methods[m1_idx]
    let m2 = methods[m2_idx]

    # Compute agreement (simplified Rand Index)
    let agree = 0
    let total = 0
    let i = 0
    while i < len(m1.labels) do
      let j = i + 1
      while j < len(m1.labels) do
        let same1 = m1.labels[i] == m1.labels[j]
        let same2 = m2.labels[i] == m2.labels[j]
        if same1 == same2 then agree = agree + 1 end
        total = total + 1
        j = j + 1
      end
      i = i + 1
    end

    let ri = agree / total
    println("  " + m1.name + " vs " + m2.name + ": Rand Index = " +
            str(round(ri, 4)))

    m2_idx = m2_idx + 1
  end
  m1_idx = m1_idx + 1
end

Pitfalls

Pitfall 1: Choosing K by “Looking at the UMAP”

Do not choose the number of clusters by counting the visual groups in a UMAP or t-SNE plot. These methods can split continuous variation into apparent discrete clusters, and can merge distinct groups that happen to be close in the embedding. Use the elbow method, silhouette scores, or domain knowledge.

Pitfall 2: Assuming Clusters Are Cell Types

In single-cell data, a Leiden cluster is not automatically a cell type. It is a group of cells that are transcriptionally similar. Some clusters may represent cell states (activated vs resting T cells) rather than cell types. Some may represent doublets (two cells captured in one droplet). Some may represent low-quality cells. Biological validation with known marker genes is essential.

Pitfall 3: Ignoring Cluster Size Imbalance

K-means tends to produce clusters of roughly equal size, even when the true groups have very different sizes. If you have 800 Luminal A tumors and 50 Normal-like tumors, k-means may incorrectly split the large group to balance sizes. Leiden clustering handles imbalanced groups better.

# K-means can force equal-sized clusters

# Synthetic example: one large group and one small group
let big_group = rnorm(500) |> map(|x| [x, rnorm(1)[0]])
let small_group = rnorm(20) |> map(|x| [x + 10, rnorm(1)[0] + 10])
let all_points = concat(big_group, small_group)

let km2 = kmeans(all_points, k: 2)
let sizes = km2.labels |> value_counts()
println("True sizes: 500, 20")
println("K-means sizes: " + str(sizes))
# K-means may produce sizes like 260, 260 instead of 500, 20

Pitfall 4: Not Reporting Clustering Parameters

Every clustering result depends on specific parameter choices. Always report:

  • The features used (which genes, how many)
  • The preprocessing (normalization, scaling)
  • The dimensionality reduction (number of PCs)
  • The clustering method and parameters
  • The number of clusters and how it was chosen
  • The validation approach

Pitfall 5: Overclaiming Novel Subtypes

It is easy to increase k and find “novel subtypes.” But more clusters is not more discovery. Before claiming a new subtype, verify:

  1. It is stable across bootstrap resamples
  2. It has distinct marker genes
  3. It has a distinct clinical phenotype
  4. It is reproducible in an independent cohort

Where Next?

Chapter 15: Meta-Analysis — When multiple studies have clustered the same disease, meta-analysis combines the evidence. Are the subtypes consistent across cohorts? Which clusters replicate and which are dataset-specific?

Multi-omic clustering — Instead of clustering on gene expression alone, integrate copy number, methylation, mutation, and protein data. Methods like iCluster and MOFA perform joint dimensionality reduction across data types before clustering. The TCGA Pan-Cancer Atlas used multi-omic clustering to define molecular subtypes across 33 cancer types.

Spatial transcriptomics — When you have spatial coordinates for each cell (from technologies like Visium, MERFISH, or Slide-seq), clustering must consider both expression similarity and spatial proximity. Spatially-aware clustering reveals tissue architecture.

Trajectory analysis — When cells form continuous developmental trajectories rather than discrete clusters, methods like pseudotime ordering (Monocle, Slingshot) trace paths through the PCA/UMAP space. The choice between clustering and trajectory analysis depends on the biology: differentiation is a trajectory, cell type identity is a cluster.
