Biostatistics in Practice

Experimental Design and Power

The Problem

Dr. Sarah Nguyen’s lab wants to run an RNA-seq experiment. They plan to compare gene expression in treated versus untreated cell lines to identify transcriptional responses to a new drug candidate. The budget allows for 12 sequencing libraries.

The question that will determine whether this experiment succeeds or fails: how should those 12 libraries be allocated?

Option A: 2 conditions x 6 biological replicates = 12 libraries Option B: 2 conditions x 3 biological replicates x 2 technical replicates = 12 Option C: 4 conditions x 3 biological replicates = 12 libraries Option D: 2 conditions x 2 biological replicates x 3 time points = 12 libraries

Each option answers a different question with different statistical power. Option B is almost always wrong (technical replicates add little value). Option A maximizes power for a single comparison. Option C enables multiple comparisons but with less power for each. Option D enables a time course but is severely underpowered.

This chapter is about making these decisions rigorously, using real variance estimates from pilot data and explicit power calculations. We will show that a poorly designed experiment wastes money, time, and biological material — and produces results that cannot be trusted.

Why It Matters

The cost of an underpowered experiment is not just statistical. It is ethical. Animals are sacrificed, patient samples are consumed, years of work are invested, and papers are published with results that do not replicate. A 30-minute power analysis before the experiment can prevent all of this.

Here is what happens when experiments are underpowered:

  1. Wasted resources. If your experiment has 20% power, you have an 80% chance of finding nothing — even when a real effect exists. The money, samples, and time are gone.

  2. Winner’s curse. The effects that DO reach significance in underpowered studies are systematically overestimated. Follow-up studies will show smaller effects, creating the illusion that the original finding “failed to replicate.”

  3. Publication bias. Only “significant” results get published, creating a literature full of inflated effects from underpowered studies. This distorts the field’s understanding of effect sizes.

  4. Ethical concerns. Using biological samples (especially patient tissue or animal models) in experiments too small to yield reliable conclusions is ethically questionable.

The solution is to calculate the required sample size BEFORE the experiment, using realistic effect size and variance estimates from pilot data or the literature.

Exploration

Estimating variance from pilot data

The most critical input to any power calculation is the variance. For RNA-seq, we need the within-group variance of log-transformed expression values.

# Extract expression columns (skip the gene ID column)
let all_cols = columns(expr)
let gene_col = all_cols[0]
let sample_cols = filter(all_cols, |c| c != gene_col)

println("Gene ID column: " + gene_col)
println("Sample columns: " + to_string(len(sample_cols)))
each(sample_cols, |s| println("  " + s))

# For each gene, calculate the within-group variance
# Use the first 3 samples as one group (for pilot estimation)
let pilot_cols = slice(sample_cols, 0, 3)

let gene_variances = map(range(0, min(len(expr), 5000)), |i| {
  let row = slice(expr, i, i + 1)
  let vals = map(pilot_cols, |c| {
    col_values(row, c) |> head(1) |> to_float()
  })
  # Log-transform (adding pseudocount)
  let log_vals = map(vals, |x| log2(x + 1.0))
  let m = mean(log_vals)
  let v = variance(log_vals)
  let s = stdev(log_vals)
  record(mean_expr: m, variance: v, stdev: s)
})

# Summary of variance estimates
let all_vars = map(gene_variances, |g| g.variance)
let all_sds = map(gene_variances, |g| g.stdev)
let all_means = map(gene_variances, |g| g.mean_expr)

Understanding effect sizes in RNA-seq

# Effect size in RNA-seq is typically measured as log2 fold change
# Cohen's d = log2FC / pooled_SD

# What log2 fold changes are biologically meaningful?
# |log2FC| = 0.5 means ~1.4-fold change (small)
# |log2FC| = 1.0 means ~2-fold change (moderate)
# |log2FC| = 2.0 means ~4-fold change (large)
# |log2FC| = 3.0 means ~8-fold change (very large)

let fc_table = [0.5, 1.0, 1.5, 2.0, 3.0]
println("Log2FC to fold-change conversion:")
each(fc_table, |lfc| {
  let fc = pow(2.0, lfc)
  println("  log2FC=" + to_string(lfc) + " -> " + to_string(round(fc, 2)) + "-fold change")
})

# Using the median SD from our pilot data, compute Cohen's d

The Analysis

Power calculation from first principles

Power is the probability of rejecting the null hypothesis when it is false. It depends on four quantities:

  • Effect size (how big the difference is)
  • Sample size (how many observations per group)
  • Significance level (alpha, usually 0.05)
  • Variance (how noisy the data are)
# Power for a two-sample t-test
# The non-centrality parameter is: delta = d * sqrt(n/2)
# where d = Cohen's d, n = sample size per group
#
# Power = P(|T| > t_crit | delta)
# Using the normal approximation:
# Power approximately = Phi(delta - z_alpha/2)
#   where z_alpha/2 = 1.96 for alpha=0.05 (two-sided)

# Calculate power for different sample sizes and effect sizes
let alpha = 0.05
let z_alpha = qnorm(1.0 - alpha / 2.0, 0.0, 1.0)

let sample_sizes = [3, 4, 5, 6, 8, 10, 12, 15, 20, 25, 30, 50]
let effect_sizes = [0.5, 0.8, 1.0, 1.5, 2.0]

println("Power table (alpha=" + to_string(alpha) + ", two-sided t-test):")
println("")

# Header
let header = "  n/group"
each(effect_sizes, |d| {
  let h = header + "  d=" + to_string(d)
})
println("n/group | d=0.5  d=0.8  d=1.0  d=1.5  d=2.0")
println("--------|--------------------------------------")

each(sample_sizes, |n| {
  let powers = map(effect_sizes, |d| {
    let delta = d * sqrt(n / 2.0)
    let power = pnorm(delta - z_alpha, 0.0, 1.0)
    round(power * 100, 1)
  })
  let line = to_string(n)
  let power_strs = map(powers, |p| to_string(p) + "%")
  println("  " + to_string(n) + "     | " + join(power_strs, "  "))
})

RNA-seq specific power calculation

# For RNA-seq, we use the variance estimates from our pilot data
# The typical workflow:
# 1. Get variance estimates from pilot data or similar published data
# 2. Decide on the minimum fold-change you want to detect
# 3. Calculate power across sample sizes
# 4. Choose the smallest n that gives >= 80% power

# Use the real variance estimates from GSE183947
let typical_sd = median(expr_sds)
let high_var_sd = quantile(expr_sds, 0.75)

println("  High-variance genes (Q75 SD): " + to_string(round(high_var_sd, 3)))

# Power for detecting 2-fold change (log2FC = 1)
let target_lfc = 1.0
println("")
println("Power to detect 2-fold change (log2FC=1):")

println("--------|----------------------------")

What fraction of genes can you detect at each sample size?

# For a fixed fold-change threshold, different genes have different
# power because they have different variances
# This is a more realistic way to think about RNA-seq power:
# "What fraction of truly DE genes will I detect?"

let target_lfcs = [1.0, 1.5, 2.0]
let test_ns = [3, 5, 8, 12, 20]

println("Fraction of expressed genes detectable at 80% power:")
println("(assuming all DE genes have the specified log2FC)")
println("")

each(target_lfcs, |lfc| {
  println("log2FC = " + to_string(lfc) + " (" + to_string(round(pow(2.0, lfc), 1)) + "-fold):")
  each(test_ns, |n| {
    # For each gene, calculate whether power >= 0.80
    let detectable = filter(expr_sds, |sd| {
      let d = lfc / sd
      let delta = d * sqrt(n / 2.0)
      let power = pnorm(delta - z_alpha, 0.0, 1.0)
      power >= 0.80
    })
    let pct = round(100.0 * len(detectable) / len(expr_sds), 1)
    println("  n=" + to_string(n) + ": " + to_string(pct) + "% of genes detectable")
  })
  println("")
})

The cost of underpowered studies

# Simulate the consequences of different sample sizes
# on a genome-wide study

# Set up: 1000 genes, 100 truly DE (10%)
# True DE genes have log2FC = 1.5
let n_genes = 1000
let n_de = 100
let n_null = n_genes - n_de
let true_lfc = 1.5
let gene_sd = typical_sd

println("Simulation: " + to_string(n_genes) + " genes, " + to_string(n_de) + " truly DE (log2FC=" + to_string(true_lfc) + ")")
println("")

each([3, 5, 8, 12, 20], |n| {
  # Simulate null genes
  let null_pvals = map(range(0, n_null), |i| {
    let g1 = rnorm(n, 8.0, gene_sd)
    let g2 = rnorm(n, 8.0, gene_sd)
    let res = ttest(g1, g2)
    res.p_value
  })

  # Simulate DE genes
  let de_pvals = map(range(0, n_de), |i| {
    let g1 = rnorm(n, 8.0, gene_sd)
    let g2 = rnorm(n, 8.0 + true_lfc, gene_sd)
    let res = ttest(g1, g2)
    res.p_value
  })

  # Apply BH correction
  let all_pvals = concat(null_pvals, de_pvals)
  let all_adjusted = p_adjust(all_pvals, "BH")

  # Split back into null and DE
  let null_adj = slice(all_adjusted, 0, n_null)
  let de_adj = slice(all_adjusted, n_null, n_genes)

  let fp = len(filter(null_adj, |p| p < 0.05))
  let tp = len(filter(de_adj, |p| p < 0.05))
  let fn_count = n_de - tp
  let total_sig = fp + tp

  let fdr = if total_sig > 0 then round(100.0 * fp / total_sig, 1) else 0.0
  let power = round(100.0 * tp / n_de, 1)
  let sensitivity = power

  println("n=" + to_string(n) + " per group:")
  println("  True positives: " + to_string(tp) + "/" + to_string(n_de) + " (power=" + to_string(power) + "%)")
  println("  False positives: " + to_string(fp) + "/" + to_string(n_null))
  println("  False negatives: " + to_string(fn_count) + "/" + to_string(n_de))
  println("  Total significant: " + to_string(total_sig))
  println("  Observed FDR: " + to_string(fdr) + "%")
  println("")
})

Biological vs technical replicates

# This is the most important design concept in genomics
# and the most commonly violated

# BIOLOGICAL REPLICATES: independent biological samples
# - Different patients, different mice, different cell cultures
# - Capture biological variability
# - THESE ARE WHAT COUNT FOR SAMPLE SIZE

# TECHNICAL REPLICATES: repeated measurements of the same sample
# - Same RNA extracted twice, same library sequenced twice
# - Capture technical variability
# - DO NOT increase your effective sample size for biological questions

# Demonstrate why:
# Scenario 1: 6 biological replicates, 1 technical rep each = n=6
# Scenario 2: 3 biological replicates, 2 technical reps each = n=3 (NOT n=6!)

let bio_sd = 1.5   # biological variability (between individuals)
let tech_sd = 0.3   # technical variability (between replicates)

# Total SD with biological replicates:
# Each observation is independent, SD = sqrt(bio_sd^2 + tech_sd^2)
let total_sd = sqrt(bio_sd * bio_sd + tech_sd * tech_sd)
println("Standard deviation components:")
println("  Biological SD: " + to_string(bio_sd))
println("  Technical SD: " + to_string(tech_sd))
println("  Total SD: " + to_string(round(total_sd, 3)))
println("")
println("Proportion of variance from biology: " + to_string(round(100.0 * bio_sd * bio_sd / (total_sd * total_sd), 1)) + "%")
println("Proportion of variance from technology: " + to_string(round(100.0 * tech_sd * tech_sd / (total_sd * total_sd), 1)) + "%")

# Averaging technical replicates reduces tech_sd but NOT bio_sd
# With k technical reps per biological sample:
# SD of average = sqrt(bio_sd^2 + tech_sd^2/k)
each([1, 2, 3, 5, 10], |k| {
  let avg_sd = sqrt(bio_sd * bio_sd + tech_sd * tech_sd / k)
  let reduction = round(100.0 * (1.0 - avg_sd / total_sd), 1)
  println("  k=" + to_string(k) + " tech reps: SD=" + to_string(round(avg_sd, 3)) + " (" + to_string(reduction) + "% reduction)")
})

println("")
println("Conclusion: doubling technical reps barely helps because")
println("biology dominates the variance. Spend the budget on more")
println("biological replicates instead.")

# Compare the two allocation strategies
# Budget: 12 libraries total

# Strategy A: 6 bio reps x 1 tech rep per group = n=6
let n_bio_a = 6
let sd_a = total_sd
let d_a = 1.0 / sd_a
let delta_a = d_a * sqrt(n_bio_a / 2.0)
let power_a = pnorm(delta_a - z_alpha, 0.0, 1.0)

# Strategy B: 3 bio reps x 2 tech reps per group = n=3 (with averaged tech reps)
let n_bio_b = 3
let sd_b = sqrt(bio_sd * bio_sd + tech_sd * tech_sd / 2.0)
let d_b = 1.0 / sd_b
let delta_b = d_b * sqrt(n_bio_b / 2.0)
let power_b = pnorm(delta_b - z_alpha, 0.0, 1.0)

println("Comparison for detecting 2-fold change:")
println("Strategy A (6 bio reps, 1 tech rep):")
println("  Effective SD: " + to_string(round(sd_a, 3)))
println("  Power: " + to_string(round(power_a * 100, 1)) + "%")
println("")
println("Strategy B (3 bio reps, 2 tech reps):")
println("  Effective SD: " + to_string(round(sd_b, 3)))
println("  Power: " + to_string(round(power_b * 100, 1)) + "%")
println("")
println("Strategy A has " + to_string(round(power_a * 100 - power_b * 100, 1)) + " percentage points more power")
println("for the SAME total cost (12 libraries)")

Paired vs unpaired designs

# Paired designs (before/after treatment on the same individual)
# remove inter-individual variability, dramatically increasing power

# In a paired design:
# We test the DIFFERENCES within each pair
# The variance of the difference = 2 * within_individual_variance
# NOT 2 * total_variance (which includes between-individual variance)

let between_ind_sd = 2.0  # large variability between individuals
let within_ind_sd = 0.5   # small variability within individuals (over time)
let unpaired_sd = sqrt(between_ind_sd * between_ind_sd + within_ind_sd * within_ind_sd)
let paired_diff_sd = sqrt(2.0) * within_ind_sd  # SD of paired differences

println("Paired vs Unpaired design:")
println("  Between-individual SD: " + to_string(between_ind_sd))
println("  Within-individual SD: " + to_string(within_ind_sd))
println("  Unpaired test SD: " + to_string(round(unpaired_sd, 3)))
println("  Paired test SD (of differences): " + to_string(round(paired_diff_sd, 3)))
println("")

# Power comparison
let target = 1.0  # detect log2FC = 1
each([5, 8, 10, 15, 20], |n| {
  let d_unpaired = target / unpaired_sd
  let d_paired = target / paired_diff_sd

  let delta_unp = d_unpaired * sqrt(n / 2.0)
  let delta_pr = d_paired * sqrt(n)  # paired test uses n, not n/2

  let power_unp = pnorm(delta_unp - z_alpha, 0.0, 1.0)
  let power_pr = pnorm(delta_pr - z_alpha, 0.0, 1.0)

  println("  n=" + to_string(n) + " per group: Unpaired=" + to_string(round(power_unp * 100, 1)) + "%, Paired=" + to_string(round(power_pr * 100, 1)) + "%")
})

println("")
println("Paired designs are dramatically more powerful when")
println("between-individual variability is high")

Blocking and randomization

# Blocking: grouping experimental units to control known sources
# of variability
#
# Example: processing RNA-seq libraries in two batches
# BAD: all treated in batch 1, all control in batch 2
#   -> confounds treatment with batch
# GOOD: half treated + half control in each batch
#   -> batch effect is orthogonal to treatment

# Randomization within blocks:
# Within each batch, randomly assign samples to conditions
# This prevents unconscious bias

# Example design for 12 samples (6 treated, 6 control) in 2 batches:
println("Balanced block design:")
println("")
println("Batch 1:")
println("  Treated:  T1, T2, T3")
println("  Control:  C1, C2, C3")
println("")
println("Batch 2:")
println("  Treated:  T4, T5, T6")
println("  Control:  C4, C5, C6")
println("")
println("Each batch has equal numbers of treated and control")
println("Batch effect cancels out in the treatment comparison")

# The variance reduction from blocking:
# Total variance = treatment_variance + batch_variance + error_variance
# With blocking: we REMOVE batch_variance from the error term
# Without blocking: batch_variance inflates the error term

let batch_sd = 0.8
let error_sd = 1.0
let total_unblocked = sqrt(batch_sd * batch_sd + error_sd * error_sd)
let total_blocked = error_sd

println("")
println("Variance with and without blocking:")
println("  Unblocked SD: " + to_string(round(total_unblocked, 3)))
println("  Blocked SD: " + to_string(round(total_blocked, 3)))
println("  Variance reduction: " + to_string(round(100.0 * (1.0 - total_blocked * total_blocked / (total_unblocked * total_unblocked)), 1)) + "%")

Pseudoreplication: the most common design mistake

# Pseudoreplication occurs when non-independent observations are
# treated as independent replicates
#
# Common examples in genomics:
#
# 1. CELLS from the same individual treated as biological replicates
#    If you grow 3 wells from the same cell line and call them n=3,
#    you have pseudoreplication. The true n=1 (one cell line).
#
# 2. MULTIPLE measurements per mouse
#    Taking 3 tissue samples from 1 mouse does not give n=3.
#    It gives n=1 with 3 technical replicates.
#
# 3. SINGLE-CELL data
#    1000 cells from one donor is NOT n=1000 for between-donor
#    comparisons. It is n=1 with 1000 cells per donor.
#
# 4. SEQUENCING LANES
#    Running the same library on 2 lanes gives 2 FASTQ files
#    but n=1. Merge and analyze as a single sample.

println("Pseudoreplication example:")
println("")
println("A study reports: 'n=30 cells per condition'")
println("But all 30 cells came from ONE donor per condition")
println("")
println("Truth: n=1 per condition (one donor)")
println("Claim: n=30 per condition (treating cells as replicates)")
println("")
println("Consequence: variance is underestimated by ~30x")
println("All p-values are far too small")
println("Almost everything appears 'significant'")

# Demonstrate the damage
let donor_mean_treated = 8.0
let donor_mean_control = 8.0  # no real difference
let between_donor_sd = 1.5
let within_donor_sd = 0.3     # cell-to-cell within one donor

# Correct analysis: compare donor means (n=1 vs n=1)
# No test is possible with n=1 per group!

# Pseudoreplicated analysis: treat cells as independent
let pseudo_treated = rnorm(30, donor_mean_treated, within_donor_sd)
let pseudo_control = rnorm(30, donor_mean_control, within_donor_sd)
let pseudo_result = ttest(pseudo_treated, pseudo_control)
println("")
println("Pseudoreplicated t-test (WRONG):")
println("  p-value: " + to_string(round(pseudo_result.p_value, 4)))
println("  This p-value is meaningless because cells are not independent")

Power for variant detection

# Different question: How many samples do I need to detect a variant
# at a given allele frequency?

# For a rare variant with frequency f in n individuals:
# The number of copies follows Binomial(2n, f) for diploid organisms
# We need at least 1 copy to detect the variant
# P(detect) = 1 - P(0 copies) = 1 - (1-f)^(2n)

let frequencies = [0.01, 0.005, 0.001, 0.0005, 0.0001]
let cohort_sizes = [100, 500, 1000, 5000, 10000, 50000, 100000]

println("Probability of observing at least one copy of a variant:")
println("(by allele frequency and cohort size)")
println("")

GWAS power calculation

# For GWAS, the effect sizes are much smaller than RNA-seq
# A typical odds ratio for a common variant is 1.1-1.3
# This requires thousands to hundreds of thousands of samples

# Power for a GWAS association test (simplified):
# Uses a chi-square approximation

# Key parameters:
# - Minor allele frequency (MAF)
# - Effect size (odds ratio or beta)
# - Sample size
# - Alpha (typically 5e-8 for genome-wide significance)

let gwas_alpha = 5e-8  # genome-wide significance threshold
let z_gwas = qnorm(1.0 - gwas_alpha / 2.0, 0.0, 1.0)

println("GWAS power analysis:")
println("Genome-wide significance threshold: p < " + to_string(gwas_alpha))
println("Required Z-score: " + to_string(round(z_gwas, 2)))
println("")

# For a quantitative trait:
# Power depends on variance explained (R^2)
# R^2 = 2 * MAF * (1-MAF) * beta^2

let mafs = [0.05, 0.10, 0.20, 0.30, 0.40]
let betas = [0.05, 0.10, 0.20]  # effect size in SD units

println("Minimum sample size for 80% power in GWAS:")
println("(quantitative trait, genome-wide significance)")
println("")

each(betas, |beta| {
  println("Effect size beta = " + to_string(beta) + " SD:")
  each(mafs, |maf| {
    let r2 = 2.0 * maf * (1.0 - maf) * beta * beta
    # Approximate n for 80% power:
    # n approximately = (z_alpha + z_beta)^2 / r2
    # where z_beta = 0.842 for 80% power
    let z_beta = 0.842
    let n_needed = ceil((z_gwas + z_beta) * (z_gwas + z_beta) / r2)
    println("  MAF=" + to_string(maf) + ": n=" + to_string(round(n_needed, 0)))
  })
  println("")
})

Interpretation

The power analyses in this chapter reveal several important truths:

1. n = 3 gives 20-30% power for moderate effects. This means you miss 70-80% of real biology. For RNA-seq, n = 6 per group is a practical minimum for discovery; n = 12 is needed for robust detection of 2-fold changes.

2. Biological replicates dominate technical replicates. When biological variance accounts for 95%+ of total variance (as is typical), adding technical replicates is almost worthless. Always maximize biological replicates.

3. Paired designs can be dramatically more powerful. When between-individual variability is high (which it usually is), a paired design effectively removes it, often doubling or tripling the effective sample size.

4. GWAS requires enormous samples. Effect sizes for common variants are tiny (explaining < 1% of variance per variant), requiring thousands to hundreds of thousands of samples for genome-wide significance.

5. Power is not a property of the study design alone. It depends on the specific gene’s variance, the effect size for that gene, and the significance threshold. Some genes are always easy to detect; others are always hard.

# Decision framework for experimental design

# Step 1: Define your primary question

Sample size calculation template

# Template for RNA-seq power calculation
# Fill in your own values

# YOUR PARAMETERS (modify these)
let my_target_lfc = 1.0       # minimum log2 fold change to detect
let my_sd = 1.2               # estimated within-group SD (from pilot)
let my_alpha = 0.05           # significance level
let my_power_target = 0.80    # desired power

# CALCULATION
let my_z_alpha = qnorm(1.0 - my_alpha / 2.0, 0.0, 1.0)
let my_z_power = qnorm(my_power_target, 0.0, 1.0)
let my_d = my_target_lfc / my_sd

# Sample size per group (approximate formula)
let n_needed = ceil(2.0 * pow((my_z_alpha + my_z_power) / my_d, 2.0))

println("=== RNA-seq Power Calculation ===")
println("Target log2FC: " + to_string(my_target_lfc))
println("Within-group SD: " + to_string(my_sd))
println("Cohen's d: " + to_string(round(my_d, 3)))
println("Alpha: " + to_string(my_alpha))
println("Target power: " + to_string(my_power_target))
println("")
println("REQUIRED SAMPLE SIZE: n = " + to_string(round(n_needed, 0)) + " per group")
println("TOTAL LIBRARIES: " + to_string(round(2.0 * n_needed, 0)))
println("")

# Verify with power calculation

Pitfalls

Pitfall 1: Using pilot effect sizes for power calculation

# Pilot studies overestimate effect sizes because only
# "significant" results from pilots are used for planning
# This is the "winner's curse" applied to study design
#
# If your pilot found log2FC = 2.5 for your gene of interest,
# the true effect is probably closer to 1.5-2.0
#
# Solution: use the LOWER BOUND of the confidence interval
# for effect size from the pilot, not the point estimate

let pilot_lfc = 2.5
let pilot_se = 0.8  # standard error of the pilot estimate

# The 80% CI lower bound is a better estimate for power calculation
let conservative_lfc = pilot_lfc - 1.28 * pilot_se  # 80% CI lower bound
println("Pilot log2FC: " + to_string(pilot_lfc))
println("Conservative estimate (80% CI lower): " + to_string(round(conservative_lfc, 2)))
println("Use " + to_string(round(conservative_lfc, 2)) + " for power calculation, not " + to_string(pilot_lfc))

Pitfall 2: Ignoring multiple testing in power calculations

# If you test 20,000 genes, your per-test alpha is NOT 0.05
# After BH correction, the effective per-test alpha is approximately
# alpha * (number of true positives / total tests)
#
# For a rough correction: use alpha / (number of tests) as your
# per-test alpha in the power calculation (Bonferroni approximation)
# This is conservative but gives a floor for the required n

let n_tests = 20000
let bonf_alpha = 0.05 / n_tests
let z_bonf = qnorm(1.0 - bonf_alpha / 2.0, 0.0, 1.0)

println("Single test alpha: 0.05")
println("Bonferroni-corrected alpha: " + to_string(bonf_alpha))
println("Required Z-score: " + to_string(round(z_bonf, 2)) + " (vs 1.96 for single test)")

# This substantially increases the required sample size
let d_example = 1.0
let z_power = 0.842

let n_single = ceil(2.0 * pow((1.96 + z_power) / d_example, 2.0))
let n_bonf = ceil(2.0 * pow((z_bonf + z_power) / d_example, 2.0))

println("")
println("Required n per group for d=1.0:")
println("  Single test: " + to_string(round(n_single, 0)))
println("  After Bonferroni for 20000 tests: " + to_string(round(n_bonf, 0)))
println("  (BH correction requires somewhat less than Bonferroni)")

Pitfall 3: Pseudoreplication in single-cell studies

# Single-cell RNA-seq creates a massive pseudoreplication temptation
# You might have 10,000 cells per condition
# But if all cells come from one donor per condition, n = 1
#
# The correct approach:
# 1. Summarize per-donor (average or pseudobulk)
# 2. Test between donors (the biological replicates)
# 3. Report the number of DONORS, not cells
#
# The cells provide information about WITHIN-DONOR heterogeneity
# The donors provide information about BETWEEN-DONOR differences
# Only the latter matters for generalizing to the population

println("Single-cell study design:")
println("")
println("WRONG: 2 donors x 5000 cells = '10000 replicates'")
println("RIGHT: 2 donors x 5000 cells = '2 biological replicates'")
println("")
println("To achieve n=6 per group in a scRNA-seq experiment,")
println("you need 6 DONORS per group (12 donors total)")
println("The number of cells per donor affects resolution")
println("but NOT the sample size for between-group comparisons")

Pitfall 4: Post-hoc power analysis

# NEVER compute "observed power" after seeing the results
# This is circular and provides no useful information
#
# If you found p = 0.06 and compute observed power = 48%,
# this does NOT mean "you almost had enough power"
# The observed power is a deterministic function of the p-value
# It tells you nothing beyond what the p-value already tells you
#
# The right time for power analysis is BEFORE the experiment
# After the experiment, report effect sizes with confidence intervals
# instead of power

println("Post-hoc power analysis is NEVER appropriate")
println("")
println("Before experiment: Power analysis guides design")
println("After experiment: Report effect sizes and CIs")
println("")
println("The observed power is:")
println("  High when p is small (you already know this from p)")
println("  Low when p is large (you already know this from p)")
println("  It adds no new information")

Pitfall 5: Confusing precision with power

# More data gives you more PRECISION (narrower confidence intervals)
# AND more POWER (probability of detecting an effect)
# These are related but not the same
#
# You might have enough power to detect a 2-fold change
# but not enough precision to estimate the fold change accurately
#
# For estimation, the CI width matters:
# CI width approximately = 2 * z * SD / sqrt(n)

let target_precision = 0.5  # want CI width < 0.5 log2FC units
let example_sd = 1.2

# Required n for this precision
let n_for_precision = ceil(pow(2.0 * 1.96 * example_sd / target_precision, 2.0))
println("Required n for CI width < " + to_string(target_precision) + " log2FC:")
println("  n = " + to_string(round(n_for_precision, 0)) + " per group")
println("")
println("This may be MORE than needed for 80% power")
println("Design for the more demanding requirement")

Where Next?

This chapter has given you the tools to design experiments that can actually answer your questions. The key principles:

  1. Calculate power before the experiment. Use real variance estimates from pilot data or public datasets. Be conservative about effect sizes.

  2. Maximize biological replicates. Technical replicates provide negligible benefit when biological variance dominates (which it always does).

  3. Consider paired designs. If you can measure the same individual before and after treatment, the power gain is enormous.

  4. Block known confounders. Balance batches, sexes, ages, and other covariates across conditions.

  5. Avoid pseudoreplication. Cells are not donors. Lanes are not libraries. Technical replicates are not biological replicates.

  6. Adjust for multiple testing. Genome-wide studies require either larger samples or acceptance of lower power per gene.

  7. Pre-register your analysis. Decide your statistical approach before looking at the data.

With solid foundations in statistical thinking (Chapter 1), exploratory analysis (Chapter 2), distributional knowledge (Chapter 3), and experimental design (this chapter), you are equipped to design, execute, and analyze genomics experiments that produce reliable, reproducible results.
