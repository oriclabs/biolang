Biostatistics in Practice

Thinking Statistically About Biology

The Problem

Dr. Elena Vasquez stares at her screen. Her collaborator has sent a spreadsheet with RNA-seq results from a breast cancer study. The email reads: “Great news! We found 2,847 differentially expressed genes between tumor and normal tissue (p < 0.05). The results are very significant.”

Elena opens the file. Three tumor samples. Three normal samples. n = 3 per group. She winces. Not because the biology is wrong — breast cancer certainly involves massive transcriptional changes — but because with n = 3, the statistics are almost guaranteed to be misleading. Some of those 2,847 genes are real. Many are not. And there is no way to tell which is which from this experiment alone.

This chapter is about developing the instinct to ask the right questions before trusting a number. What does a p-value actually mean? Why is effect size more important than significance? Why does sample size matter so profoundly in genomics? And what should you do when — as happens constantly in biology — the ideal experiment is too expensive to run?

We will work through these questions using real breast cancer RNA-seq data from GEO dataset GSE183947, a study that examined gene expression in breast cancer tissue using RNA sequencing with FPKM quantification.

Why It Matters

The reproducibility crisis in biomedical research is not abstract. A 2012 study by Amgen scientists attempted to reproduce 53 “landmark” cancer biology papers. They could reproduce only 6 — an 11% success rate. A similar effort by Bayer found that only 25% of published preclinical studies could be confirmed.

Underpowered studies are a primary driver of irreproducibility. When you have too few samples, several pathological things happen simultaneously:

  1. Inflated effect sizes. Only the largest (often spurious) effects reach significance with small samples. This is called the “winner’s curse.” The effects that survive your p < 0.05 filter are systematically overestimated.

  2. High false discovery rates. With 20,000 genes and n = 3, you are running 20,000 tests. Even with multiple testing correction, the statistical power is so low that a large fraction of “significant” results are false positives.

  3. Missed real biology. Conversely, many genes with genuine, biologically meaningful expression changes will not reach significance. You are blind to moderate effects.

  4. Unstable variance estimates. With only 3 observations per group, the sample variance is extremely noisy. A single outlier can dominate. The t-statistic denominator becomes unreliable.

These are not theoretical concerns. They affect every experiment, every paper, every clinical decision that rests on statistical evidence from small samples.

Why log-transformation is essential

# Log2-transform the expression values
# Add a small pseudocount to handle zeros
let pseudocount = 1.0
let log_values = map(first_sample, |x| log2(x + pseudocount))

println("Log2(FPKM+1) summary:")

  |> save_plot("plots/log2_fpkm_distribution.svg")

The raw FPKM distribution is massively right-skewed. A handful of highly expressed genes (ribosomal proteins, actins, histones) dominate the scale. Log-transformation compresses this range and makes the distribution approximately normal, which is a requirement for many statistical tests.

This is not a mathematical trick. It reflects the biology: gene expression regulation is multiplicative (fold-changes), not additive. A 2-fold change from 100 to 200 FPKM is biologically comparable to a 2-fold change from 1000 to 2000. The log scale captures this: log2(200) - log2(100) = log2(2000) - log2(1000) = 1.

The Analysis

Now we arrive at the core question: which genes are differentially expressed between tumor and normal tissue? But before we rush to p-values, we need to understand what we are actually computing.

What is a p-value, really?

A p-value answers exactly one question: “If there were truly no difference between the groups, how often would I see a test statistic this extreme or more extreme?”

It does NOT answer:

  • “What is the probability that this gene is differentially expressed?” (That would require a Bayesian posterior.)
  • “How large is the effect?” (That is the effect size.)
  • “Is this result biologically important?” (That requires domain knowledge.)
  • “Will this replicate?” (That depends on power and effect size.)

Let us make this concrete with a single gene.

# Simulate the thought experiment behind a p-value
# Suppose gene X has identical expression in tumor and normal
# What distribution of t-statistics would we see by chance?

# First, let us see a real gene comparison

let tumor_cols = slice(sample_cols, 0, 3)

The t-test with n = 3

# Perform a t-test on this gene
let result = ttest(tumor_log, normal_log)

# With n=3 per group, df = 4 (approximately)
# The t-distribution with 4 degrees of freedom has very heavy tails
# This means large t-statistics are common by chance alone

Why n = 3 is dangerous: the variance problem

# The core issue: with n=3, variance estimation is terrible
# Let us demonstrate by looking at the standard deviation
# of our tumor samples

let tumor_sd = stdev(tumor_log)
let normal_sd = stdev(normal_log)
println("Tumor SD (n=3): " + to_string(tumor_sd))
println("Normal SD (n=3): " + to_string(normal_sd))

# With only 3 values, the SD estimate has huge uncertainty
# The 95% confidence interval for sigma given n=3 is approximately
# [0.52 * s, 6.28 * s] — a 12-fold range!
# This means the denominator of your t-statistic is unreliable

let sd_lower = tumor_sd * 0.52
let sd_upper = tumor_sd * 6.28
println("95% CI for true SD: [" + to_string(round(sd_lower, 3)) + ", " + to_string(round(sd_upper, 3)) + "]")
println("That is a " + to_string(round(sd_upper / sd_lower, 1)) + "-fold range of uncertainty")

Genome-wide testing: the multiple testing catastrophe

# Now imagine running this test for every gene
# Let us do it for the first 1000 genes as a demonstration

let results = []
let n_genes = min(len(expr), 1000)

# Process genes in bulk
let gene_subset = take(expr, n_genes)

let test_results = map(range(0, n_genes), |i| {
  let row = slice(gene_subset, i, i + 1)
  let t_vals = map(tumor_cols, |c| {
    col_values(row, c) |> take(1) |> to_float()
  })
  let n_vals = map(normal_cols, |c| {
    col_values(row, c) |> take(1) |> to_float()
  })

  let t_log = map(t_vals, |x| log2(x + 1.0))
  let n_log = map(n_vals, |x| log2(x + 1.0))

  let t_mean = mean(t_log)
  let n_mean = mean(n_log)
  let lfc = t_mean - n_mean
  let res = ttest(t_log, n_log)

  record(
    gene_index: i,
    log2fc: lfc,
    t_stat: res.statistic,
    p_value: res.p_value
  )
})

let results_table = table(test_results)
println("Tested " + to_string(n_genes) + " genes")

How many “significant” results do we get?

# Count genes with p < 0.05 (unadjusted)
let raw_pvals = map(test_results, |r| r.p_value)
let sig_raw = filter(raw_pvals, |p| p < 0.05)
println("Genes with p < 0.05 (unadjusted): " + to_string(len(sig_raw)))
println("Percentage: " + to_string(round(100.0 * len(sig_raw) / n_genes, 1)) + "%")

# Under the null hypothesis (no real differences), we expect
# 5% of genes to be "significant" by chance alone
# With 1000 tests, that is 50 false positives expected
println("Expected false positives at alpha=0.05: " + to_string(round(0.05 * n_genes, 0)))

# Now apply multiple testing correction
let adjusted = p_adjust(raw_pvals, "BH")  # Benjamini-Hochberg
let sig_adj = filter(adjusted, |p| p < 0.05)
println("Genes with adjusted p < 0.05 (BH): " + to_string(len(sig_adj)))

# The Benjamini-Hochberg procedure controls the False Discovery Rate (FDR)
# FDR = expected proportion of false positives among rejected hypotheses

Understanding False Discovery Rate

# FDR is subtle. Let us be precise about what it means.
#
# Suppose we reject 100 genes at FDR < 0.05.
# This means: among those 100 genes, we expect about 5 to be
# false positives. We do not know WHICH 5, but the overall
# proportion of mistakes is controlled.
#
# This is different from FWER (Family-Wise Error Rate), which
# controls the probability of making even ONE false positive.
# Bonferroni correction controls FWER but is very conservative
# for genomics (thousands of tests).

let adjusted_bonf = p_adjust(raw_pvals, "bonferroni")
let sig_bonf = filter(adjusted_bonf, |p| p < 0.05)
println("Genes with Bonferroni p < 0.05: " + to_string(len(sig_bonf)))

# Bonferroni is typically too conservative for genomics
# BH (FDR) is the standard in the field

Effect size: what p-values cannot tell you

# A p-value tells you whether an effect exists (under certain assumptions)
# It does NOT tell you how large the effect is
# With enough samples, even trivially small differences become "significant"

# Effect size for a two-group comparison is typically measured as:
# Cohen's d = (mean1 - mean2) / pooled_SD
# Or simply log2 fold change in genomics

# Let us examine the relationship between effect size and significance
let effect_sizes = map(test_results, |r| abs(r.log2fc))
let neg_log_p = map(test_results, |r| -log10(r.p_value + 1e-300))

# Create a volcano plot: effect size (x) vs significance (y)
volcano(effect_sizes, neg_log_p, "plots/volcano_n3.svg")
  |> save_plot("plots/volcano_n3.svg")

println("Effect size statistics:")
println("  Mean |log2FC|: " + to_string(round(mean(effect_sizes), 3)))
println("  Median |log2FC|: " + to_string(round(median(effect_sizes), 3)))

# In practice, we want BOTH:
# 1. Statistical significance (adjusted p < 0.05)
# 2. Meaningful effect size (|log2FC| > 1, i.e., 2-fold change)
let sig_and_large = filter(test_results, |r| {
  let adj_p = p_adjust([r.p_value], "BH") |> take(1)
  adj_p < 0.05 && abs(r.log2fc) > 1.0
})
println("Genes with adjusted p<0.05 AND |log2FC|>1: " + to_string(len(sig_and_large)))

The danger zone: statistical significance without biological significance

# Consider a gene with a log2 fold change of 0.1
# That is a 7% change in expression. Even if the p-value is 0.001,
# a 7% change is rarely biologically meaningful.

# Conversely, consider a gene with log2FC = 3 (8-fold change)
# but p = 0.08. With n=3, this might not reach significance
# due to high variance, but an 8-fold change in a cancer gene
# is biologically very important.

# This is why genomics has adopted the "volcano plot" standard:
# plot -log10(p) vs log2FC, and highlight genes that pass
# BOTH thresholds.

# Let us classify our results
let categories = map(test_results, |r| {
  let adj_p = p_adjust([r.p_value], "BH") |> take(1)
  let sig = adj_p < 0.05
  let large = abs(r.log2fc) > 1.0
  if sig && large then "significant_and_large"
  else if sig && !large then "significant_but_small"
  else if !sig && large then "large_but_not_significant"
  else "neither"
})

let category_counts = value_counts(categories)
println("Gene categories:")
println(category_counts)

Type I and Type II errors

# Type I error (false positive): concluding a gene is DE when it is not
# Type II error (false negative): missing a gene that truly is DE
#
# With n=3:
# - Type I rate is controlled by our alpha threshold (0.05)
# - Type II rate (1 - power) is VERY HIGH for moderate effects
#
# Power = probability of detecting a real effect
# For a t-test with n=3 per group and Cohen's d = 1.0:
# Power is approximately 0.22 (22%)
#
# This means we MISS 78% of genes with a real, large effect!

# Let us calculate approximate power for different effect sizes
# Power for a two-sample t-test depends on:
# - n (sample size per group)
# - d (effect size, Cohen's d)
# - alpha (significance level)

# Using the relationship between power and the non-central t-distribution
# For n=3, alpha=0.05 (two-sided), approximate power:
let effect_d_values = [0.5, 0.8, 1.0, 1.5, 2.0, 3.0]
let n_per_group = 3
let alpha = 0.05

# The critical t-value for df=4, alpha=0.05 two-sided
# t_crit approximately 2.776
let t_crit = 2.776

each(effect_d_values, |d| {
  # Non-centrality parameter: delta = d * sqrt(n/2)
  let delta = d * sqrt(n_per_group / 2.0)
  # Approximate power using normal approximation
  # (This is a rough approximation for illustration)
  let z_alpha = 1.96
  let z_power = delta - z_alpha
  let approx_power = pnorm(z_power, 0.0, 1.0)
  println("  d=" + to_string(d) + ": approximate power = " + to_string(round(approx_power * 100, 1)) + "%")
})

What happens when we simulate the truth

# To really understand the n=3 problem, let us simulate data
# where we KNOW the truth

# Generate 1000 "genes":
# - 900 are truly NOT differentially expressed (null genes)
# - 100 ARE truly differentially expressed with log2FC = 1.5

let n_null = 900
let n_de = 100
let true_mean = 8.0  # average log2 expression
let true_sd = 1.5    # within-group standard deviation

# Null genes: same distribution in both groups
let null_pvals = map(range(0, n_null), |i| {
  let group1 = rnorm(3, true_mean, true_sd)
  let group2 = rnorm(3, true_mean, true_sd)
  let res = ttest(group1, group2)
  res.p_value
})

# DE genes: group2 shifted by 1.5 log2 units
let de_pvals = map(range(0, n_de), |i| {
  let group1 = rnorm(3, true_mean, true_sd)
  let group2 = rnorm(3, true_mean + 1.5, true_sd)
  let res = ttest(group1, group2)
  res.p_value
})

# How many null genes are falsely called significant?
let false_positives = len(filter(null_pvals, |p| p < 0.05))
println("False positives (null genes with p<0.05): " + to_string(false_positives) + " / " + to_string(n_null))
println("False positive rate: " + to_string(round(100.0 * false_positives / n_null, 1)) + "%")

# How many DE genes are correctly detected?
let true_positives = len(filter(de_pvals, |p| p < 0.05))
println("True positives (DE genes with p<0.05): " + to_string(true_positives) + " / " + to_string(n_de))
println("Power (sensitivity): " + to_string(round(100.0 * true_positives / n_de, 1)) + "%")

# Among all "significant" results, what fraction are false?
let total_sig = false_positives + true_positives
let fdr_observed = false_positives / total_sig
println("Observed FDR: " + to_string(round(100.0 * fdr_observed, 1)) + "%")
println("Out of " + to_string(total_sig) + " significant calls, " + to_string(false_positives) + " are false")

Exploring additional sample sizes systematically

# To see the full picture, run the simulation across many sample sizes
# and track power, FDR, and total significant counts

let sample_sizes_to_test = [3, 4, 5, 6, 8, 10, 12, 15, 20, 25, 30]

let sim_results = map(sample_sizes_to_test, |n| {
  let null_ps = map(range(0, n_null), |i| {
    let g1 = rnorm(n, true_mean, true_sd)
    let g2 = rnorm(n, true_mean, true_sd)
    let res = ttest(g1, g2)
    res.p_value
  })

  let de_ps = map(range(0, n_de), |i| {
    let g1 = rnorm(n, true_mean, true_sd)
    let g2 = rnorm(n, true_mean + 1.5, true_sd)
    let res = ttest(g1, g2)
    res.p_value
  })

  let fp = len(filter(null_ps, |p| p < 0.05))
  let tp = len(filter(de_ps, |p| p < 0.05))
  let total = fp + tp
  let fdr = if total > 0 then round(100.0 * fp / total, 1) else 0.0
  let power = round(100.0 * tp / n_de, 1)

  record(
    n: n,
    true_positives: tp,
    false_positives: fp,
    total_significant: total,
    power_pct: power,
    fdr_pct: fdr
  )
})

println("Comprehensive simulation results:")
println("n/group | Power   FDR     Total Sig  TP   FP")
println("--------|--------------------------------------")
each(sim_results, |r| {
  println("  " + to_string(r.n) + "     | " + to_string(r.power_pct) + "%   " + to_string(r.fdr_pct) + "%   " + to_string(r.total_significant) + "       " + to_string(r.true_positives) + "   " + to_string(r.false_positives))
})

# Visualize the power curve from the simulation

The winner’s curse: inflated effect sizes in underpowered studies

# The winner's curse is subtle but devastating
# In underpowered studies, only the largest observed effects
# pass the significance threshold — but these are overestimates

# Simulate: true log2FC = 1.0, but with n=3, what LFC do we observe
# among the "significant" results?

let observed_lfcs_n3 = []
let observed_lfcs_n20 = []

let n_sims = 1000

# With n=3
let sig_lfcs_3 = map(range(0, n_sims), |i| {
  let g1 = rnorm(3, true_mean, true_sd)
  let g2 = rnorm(3, true_mean + 1.0, true_sd)  # true LFC = 1.0
  let res = ttest(g1, g2)
  let observed_lfc = mean(g2) - mean(g1)
  record(lfc: observed_lfc, significant: res.p_value < 0.05)
})

let sig_only_3 = filter(sig_lfcs_3, |r| r.significant)
let sig_lfc_vals_3 = map(sig_only_3, |r| r.lfc)

# With n=20
let sig_lfcs_20 = map(range(0, n_sims), |i| {
  let g1 = rnorm(20, true_mean, true_sd)
  let g2 = rnorm(20, true_mean + 1.0, true_sd)
  let res = ttest(g1, g2)
  let observed_lfc = mean(g2) - mean(g1)
  record(lfc: observed_lfc, significant: res.p_value < 0.05)
})

let sig_only_20 = filter(sig_lfcs_20, |r| r.significant)
let sig_lfc_vals_20 = map(sig_only_20, |r| r.lfc)

println("Winner's curse demonstration (true log2FC = 1.0):")
println("")
println("With n=3:")
println("  Significant results: " + to_string(len(sig_only_3)) + " / " + to_string(n_sims))
if len(sig_lfc_vals_3) > 0 then {
  println("  Mean observed LFC among significant: " + to_string(round(mean(sig_lfc_vals_3), 3)))
  println("  Median observed LFC among significant: " + to_string(round(median(sig_lfc_vals_3), 3)))
}
println("")
println("With n=20:")
println("  Significant results: " + to_string(len(sig_only_20)) + " / " + to_string(n_sims))
if len(sig_lfc_vals_20) > 0 then {
  println("  Mean observed LFC among significant: " + to_string(round(mean(sig_lfc_vals_20), 3)))
  println("  Median observed LFC among significant: " + to_string(round(median(sig_lfc_vals_20), 3)))
}

# With n=3, the observed LFC among significant results is inflated
# (well above the true value of 1.0)
# With n=20, the observed LFC is close to the truth
println("")
println("The true effect is 1.0. With n=3, significant results")
println("overestimate it. This is the winner's curse.")

Nonparametric alternatives

# When you cannot assume normality (or want to be safe with small n),
# nonparametric tests provide valid p-values without distributional assumptions

# The Wilcoxon rank-sum test (Mann-Whitney U) is the nonparametric
# equivalent of the two-sample t-test

let group_a = [8.2, 7.9, 8.5, 9.1, 7.6]
let group_b = [10.1, 9.8, 10.4, 11.2, 9.5]

let t_result = ttest(group_a, group_b)
let w_result = wilcoxon(group_a, group_b)

println("Parametric vs nonparametric comparison:")
println("  t-test p-value: " + to_string(round(t_result.p_value, 4)))
println("  Wilcoxon p-value: " + to_string(round(w_result.p_value, 4)))

# For normally distributed data, the t-test is more powerful
# For non-normal data, the Wilcoxon test can be more powerful
# With large n, both give similar results (by the CLT)

# The Wilcoxon test is based on RANKS, not values
# It tests whether one group tends to have larger values than the other
# It is robust to outliers because ranks are bounded

This simulation reveals the brutal arithmetic of underpowered studies. With n = 3, you detect maybe 20-30% of truly differentially expressed genes while accumulating a false discovery rate that can exceed 50%. With n = 10, power jumps to 70-90% and the FDR drops dramatically.

# Mix null and DE p-values — this is what real data looks like

# The histogram should show:
# - A spike near 0 (from truly DE genes)
# - A flat plateau from 0.1 to 1.0 (from null genes)
#
# The height of the plateau estimates the proportion of null genes (pi0)
# The area of the spike above the plateau estimates the number of true positives
#
# If the histogram shows strange shapes (dips, bumps, U-shapes),
# something is wrong with the data or the test assumptions

Interpretation

What have we learned from this analysis? Several things that every biologist working with high-throughput data should internalize:

1. p < 0.05 means less than you think.

A p-value of 0.03 for a single gene means there is a 3% chance of seeing this extreme a result if the gene is truly not differentially expressed. But when you test 20,000 genes, you expect 600 false positives at p < 0.05. The p-value does not account for multiple testing — you must apply FDR correction.

2. Effect size is often more important than significance.

A gene with log2FC = 0.1 and p = 0.001 has a statistically detectable but biologically trivial change. A gene with log2FC = 3.0 and p = 0.08 has a massive, potentially important change that your small study lacked power to confirm. Always report effect sizes alongside p-values.

3. n = 3 is not enough for discovery.

With 3 samples per group, you have approximately 20-30% power to detect even large effects (Cohen’s d = 1.5). This means you miss 70-80% of real biology. The genes you do detect have inflated effect size estimates (winner’s curse).

4. Variance estimation is the bottleneck.

The t-test requires estimating variance within each group. With n = 3, you have only 2 degrees of freedom for variance estimation. This is why methods like DESeq2 and edgeR share information across genes to stabilize variance estimates — a technique called empirical Bayes shrinkage.

5. The p-value distribution is diagnostic.

If your p-value histogram under the null is not flat, your assumptions are violated. Common causes: batch effects, sample contamination, wrong distributional assumption, or confounding variables.

What the results do NOT mean

It is equally important to be clear about limitations:

  • Finding 200 significant genes does not mean there are only 200 DE genes. Many more may exist but fall below the detection threshold.

  • A non-significant result (p = 0.3) does not mean the gene is not differentially expressed. It means you do not have enough evidence to conclude that it is, given your sample size.

  • Genes with the smallest p-values are not necessarily the most biologically important. They might simply have the lowest variance or the most consistent expression.

Decision

Given an RNA-seq experiment with n = 3 per group, here is a principled workflow:

# Step 1: Quality control and exploration (always)
# - Check distributions, correlations, outliers
# - PCA to visualize sample grouping

# Step 2: Use methods designed for small samples
# DESeq2/edgeR (external tools) share variance information
# across genes. In BioLang, we can approximate this concept:

# Compute a global variance estimate across all genes
let all_gene_vars = map(range(0, min(len(expr), 500)), |i| {
  let row = slice(expr, i, i + 1)
  let all_vals = map(sample_cols, |c| {
    col_values(row, c) |> take(1) |> to_float()
  })
  let log_vals = map(all_vals, |x| log2(x + 1.0))
  variance(log_vals)
})

let global_var = median(all_gene_vars)
println("Global median variance estimate: " + to_string(round(global_var, 4)))
println("This can be used to stabilize per-gene variance estimates")

# Step 3: Apply strict thresholds
# - Use FDR < 0.05 (not raw p < 0.05)
# - Require |log2FC| > 1 (2-fold change minimum)
# - Consider these results as HYPOTHESIS-GENERATING, not confirmatory

# Step 4: Validate key findings
# - qPCR validation of top candidates
# - Check against known biology (pathway enrichment)
# - Plan a larger follow-up experiment for confirmation

Pitfalls

Pitfall 1: Treating technical replicates as biological replicates

# WRONG: sequencing the same RNA extraction twice
# and calling it n=2
#
# Technical replicates measure machine variability
# Biological replicates measure biological variability
# Only biological replicates count for sample size
#
# If your "3 replicates" are 3 sequencing runs from the same
# mouse, you have n=1, not n=3

# The variance you care about is BIOLOGICAL variance
# Technical variance is typically much smaller
# Using technical replicates inflates your effective sample size
# and leads to false discoveries

Pitfall 2: Cherry-picking genes

# It is tempting to focus on the most significant results
# and build a story around them. This is p-hacking.
#
# Example of what NOT to do:
# 1. Run 20,000 tests
# 2. Find 200 "significant" genes
# 3. Select the 10 that fit your hypothesis
# 4. Report only those 10 with impressive p-values
#
# This is scientifically fraudulent even if unintentional.
# The correction for multiple testing must account for ALL
# tests performed, not just the ones you report.

Pitfall 3: Confusing correlation with causation in expression data

# Finding that Gene A and Gene B are co-expressed
# (high correlation) does not mean:
# - A regulates B
# - B regulates A
# - They are in the same pathway
#
# It means their expression levels vary together across samples.
# This could be because:
# - They share a common regulator
# - They are in the same cell type (which varies across samples)
# - They are both affected by a batch effect
# - Pure coincidence (with enough genes, some will correlate)

# Demonstrate with random data
let random1 = rnorm(100, 0.0, 1.0)
let random2 = rnorm(100, 0.0, 1.0)
let r_random = cor(random1, random2)
println("Correlation between random vectors: " + to_string(round(r_random, 4)))
# Even this will sometimes be "significant" with enough tests

Pitfall 4: Ignoring batch effects

# Samples processed on different days, by different technicians,
# or on different sequencing lanes can have systematic differences
# that have nothing to do with biology.
#
# If all your tumor samples were processed in batch 1 and all
# normal samples in batch 2, EVERY gene will appear DE —
# but you are detecting the batch effect, not cancer biology.
#
# Solution: randomize sample processing across batches
# Or at minimum, include batch as a covariate in your model.
#
# PCA is the quickest way to check: if PC1 separates batches
# rather than biological groups, you have a problem.

Pitfall 5: The log-fold-change trap for lowly expressed genes

# Genes with very low expression can show enormous fold changes
# that are meaningless

# Example: gene has FPKM values of [0.01, 0.02, 0.01] in normal
# and [0.1, 0.2, 0.15] in tumor
let low_normal = [0.01, 0.02, 0.01]
let low_tumor = [0.1, 0.2, 0.15]

let lfc = mean(map(low_tumor, |x| log2(x + 1.0))) - mean(map(low_normal, |x| log2(x + 1.0)))
println("Log2 fold change: " + to_string(round(lfc, 2)))
# This shows a large fold change, but the absolute expression
# is so low that it is likely noise

# Solution: filter out lowly expressed genes before testing
# A common threshold: keep genes with FPKM > 1 in at least
# half the samples in at least one group

Pitfall 6: Not checking test assumptions

# The t-test assumes:
# 1. Independent observations (violated by batch effects, family relatedness)
# 2. Normal distribution within each group (approximately OK for log-expression)
# 3. Equal variances (use Welch's t-test if not)
#
# Check normality with a QQ-plot or Shapiro-Wilk test
# Check equal variance by comparing group standard deviations

# For a specific gene
let group1_log = [8.2, 7.9, 8.5]
let group2_log = [10.1, 9.8, 10.4]

println("Group 1 SD: " + to_string(round(stdev(group1_log), 3)))
println("Group 2 SD: " + to_string(round(stdev(group2_log), 3)))

# If SDs differ by more than 2-fold, equal variance assumption
# is questionable. Use Welch's t-test (default in most software).
# For non-normal data, consider the Wilcoxon rank-sum test.
let wilcox_result = wilcoxon(group1_log, group2_log)
println("Wilcoxon p-value: " + to_string(wilcox_result.p_value))
