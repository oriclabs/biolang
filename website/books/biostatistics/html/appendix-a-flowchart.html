Biostatistics in Practice

Appendix A: Statistical Decision Flowchart

This appendix provides a practical decision tree for choosing the right statistical test. Start with your question and data type, then follow the branches to the appropriate BioLang function.

Step 1: What Is Your Question?

Before choosing a test, classify your question:

  • Comparing groups: Is there a difference between two or more groups?
  • Association: Are two variables related?
  • Prediction: Can one variable predict another?
  • Structure: Are there natural groups in the data?
  • Survival: Does time-to-event differ between groups?
  • Combining evidence: What is the pooled result across studies?

Step 2: Comparing Two Groups

Are the observations paired or independent?

Paired means each observation in group A has a matched observation in group B (e.g., before/after treatment on the same patient, matched tumor/normal from the same individual).

Independent means the observations in the two groups come from different subjects with no inherent pairing.

Paired observations

If your data is continuous and approximately normal:

# Paired t-test
# Example: expression before vs after treatment, same patients
let before = [5.2, 4.8, 6.1, 5.5, 4.9, 5.8, 6.3]
let after = [4.1, 3.9, 5.2, 4.6, 3.8, 4.7, 5.1]
let result = ttest_paired(before, after)
println("Paired t-test: p=" + str(round(result.pvalue, 4)))

If your data is continuous but not normal, or ordinal:

# Wilcoxon signed-rank test (paired version)
let result = wilcoxon(before, after)
println("Wilcoxon signed-rank: p=" + str(round(result.pvalue, 4)))

Independent observations, continuous outcome

If data is approximately normal (check with QQ plot, Shapiro-Wilk):

# Two-sample t-test (Welch's, unequal variance)
let group_a = [12.3, 14.5, 11.8, 13.2, 15.1]
let group_b = [9.8, 10.2, 8.5, 11.0, 9.3]
let result = ttest(group_a, group_b)
println("Two-sample t-test: p=" + str(round(result.pvalue, 4)))

If data is not normal, or has outliers, or is ordinal:

# Wilcoxon rank-sum test (Mann-Whitney U)
let result = wilcoxon(group_a, group_b)
println("Wilcoxon rank-sum: p=" + str(round(result.pvalue, 4)))

Independent observations, categorical outcome

If you have a 2x2 table (two groups, two outcomes):

# If expected cell counts >= 5: chi-squared test
let observed = [45, 155, 12, 188]  # flattened 2x2
let result = chi_square(observed)
println("Chi-squared: p=" + str(round(result.pvalue, 4)))

# If any expected cell count < 5: Fisher's exact test
let table = [[45, 155], [12, 188]]
let result = fisher_exact(table)
println("Fisher's exact: p=" + str(round(result.pvalue, 4)))

One-sample test (comparing to a known value)

# One-sample t-test: is the mean different from a hypothesized value?
let measurements = [5.2, 4.8, 5.5, 5.1, 4.9, 5.3]
let result = ttest_one(measurements, mu: 5.0)
println("One-sample t-test: p=" + str(round(result.pvalue, 4)))

Step 3: Comparing Three or More Groups

Are the observations independent?

If independent and continuous, approximately normal:

# One-way ANOVA
let group1 = [12.1, 13.4, 11.8, 14.2]
let group2 = [9.5, 10.1, 8.8, 11.3]
let group3 = [15.2, 16.0, 14.5, 15.8]
let result = anova(group1, group2, group3)
println("ANOVA F-test: p=" + str(round(result.pvalue, 4)))
# If significant, follow up with pairwise t-tests + p_adjust

If independent and not normal or ordinal:

# Kruskal-Wallis test (use ks_test as non-parametric alternative)
# For pairwise comparisons: Wilcoxon with p_adjust
let p1 = wilcoxon(group1, group2).pvalue
let p2 = wilcoxon(group1, group3).pvalue
let p3 = wilcoxon(group2, group3).pvalue
let adjusted = p_adjust([p1, p2, p3], "BH")
println("Adjusted p-values: " + str(adjusted))

If categorical outcome (contingency table larger than 2x2):

# Chi-squared test of independence
let observed = [30, 20, 50, 45, 35, 20]  # flattened contingency table
let result = chi_square(observed)
println("Chi-squared: p=" + str(round(result.pvalue, 4)))

Step 4: Association Between Two Variables

Both continuous?

If linear relationship expected and bivariate normal:

# Pearson correlation
let x = [1.2, 2.3, 3.1, 4.5, 5.2]
let y = [2.1, 4.0, 5.8, 8.2, 10.1]
let result = cor(x, y)
println("Pearson r: " + str(round(result, 4)))

If monotonic but not necessarily linear, or non-normal:

# Spearman rank correlation
let result = spearman(x, y)
println("Spearman rho: " + str(round(result.rho, 4)) +
        " p=" + str(round(result.pvalue, 4)))

If robustness to outliers is critical:

# Kendall's tau (more robust than Spearman)
let result = kendall(x, y)
println("Kendall tau: " + str(round(result.tau, 4)) +
        " p=" + str(round(result.pvalue, 4)))

One continuous, one categorical (2+ categories)?

This is the same as comparing groups (Step 2 or Step 3). The “association” between a continuous variable and a categorical variable is tested by comparing the continuous variable across the categories.

Step 5: Prediction and Modeling

Continuous outcome, one or more continuous predictors?

# Linear regression
let outcome = [10, 12, 14, 16, 18]
let predictor = [1, 2, 3, 4, 5]
let result = lm(outcome, predictor)
println("Linear model: R2=" + str(round(result.r_squared, 4)) +
        " p=" + str(round(result.pvalue, 4)))

Binary outcome (yes/no, case/control)?

# Logistic regression (GLM with binomial family)
let outcome = [0, 0, 1, 1, 1, 0, 1, 1]
let predictor_table = table([
  { age: 45, bmi: 22 },
  { age: 50, bmi: 25 },
  { age: 55, bmi: 28 },
  { age: 60, bmi: 30 },
  { age: 65, bmi: 32 },
  { age: 48, bmi: 23 },
  { age: 62, bmi: 29 },
  { age: 70, bmi: 31 }
])
let result = glm(outcome, predictor_table, family: "binomial")
println("Logistic regression: " + str(result))

Count outcome (number of mutations, reads)?

# Poisson or negative binomial GLM
let counts = [5, 12, 3, 45, 8, 22]
let result = glm(counts, predictor_table, family: "poisson")
println("Poisson GLM: " + str(result))

Time-to-event outcome?

# Cox proportional hazards model
let times = [12, 24, 6, 36, 18, 30]
let events = [1, 0, 1, 0, 1, 1]  # 1=event, 0=censored
let covariates = table([{ age: 55, stage: 2 }, { age: 60, stage: 1 },
  { age: 48, stage: 3 }, { age: 65, stage: 1 },
  { age: 52, stage: 2 }, { age: 70, stage: 3 }])
let result = cox_ph(times, events, covariates)
println("Cox PH: " + str(result))

# Kaplan-Meier survival curves (non-parametric)
let km = kaplan_meier(times, events)
println("Median survival: " + str(km.median_survival))

Step 6: Multivariate Structure

Dimensionality reduction?

# PCA — linear, deterministic, quantitative
let result = pca(data_matrix, n_components: 10)

# UMAP — nonlinear, for visualization
let result = umap(pca_coords, n_neighbors: 15, min_dist: 0.1)

# t-SNE — nonlinear, local structure emphasis
let result = tsne(pca_coords, perplexity: 30)

Clustering?

# K-means — when you know K, spherical clusters
let result = kmeans(data_matrix, k: 5)

# Leiden — graph-based, no K required, standard for single-cell
let labels = leiden(pca_coords, resolution: 1.0)

Step 7: Multiple Testing Correction

Whenever you perform more than one test, adjust p-values:

# Benjamini-Hochberg (FDR control) — standard for genomics
let adjusted = p_adjust(pvalues, "BH")

# Bonferroni (FWER control) — conservative, for few tests
let adjusted = p_adjust(pvalues, "bonferroni")

Decision Rules of Thumb

When in doubt about normality: Use the non-parametric alternative. You lose very little power if the data actually is normal (~95% efficiency for Wilcoxon vs t-test), but you avoid catastrophic Type I error if it is not.

When in doubt about the number of groups: Start with the omnibus test (ANOVA or chi-squared), then do pairwise comparisons with FDR correction. Never skip the omnibus test and jump to pairwise comparisons.

When in doubt about multiple testing: Correct. Always. The BH method at FDR < 0.05 is the standard for genomics.

When in doubt about independence: If observations might be correlated (repeated measures, family data, spatial data), standard tests give falsely small p-values. Use mixed models or account for the correlation.

When in doubt about which regression: Start with the simplest model (linear, logistic, or Cox depending on outcome type). Add complexity only if diagnostics indicate the simple model is inadequate.
