Biostatistics in Practice

Exploratory Data Analysis

The Problem

A new postdoc walks into the lab meeting with a fresh dataset. The PI loaded RNA-seq expression data from the GTEx consortium — 56,200 genes across 54 human tissues — and asks the simplest possible question: “What does it look like?”

Not “which genes are significant.” Not “confirm my hypothesis.” Just: what does it look like?

This is exploratory data analysis (EDA), and it is the most undervalued step in all of bioinformatics. In a field obsessed with p-values and publication, EDA is the thing you do before the thing you want to do. It is looking at the shape of the data, finding patterns you did not expect, and discovering problems that would invalidate your downstream analysis.

John Tukey, who coined the term, put it this way: “Far better an approximate answer to the right question, which is often vague, than an exact answer to the wrong question, which can always be made precise.”

This chapter teaches you how to look at biological data systematically. We will use the GTEx gene expression dataset — one of the largest and most valuable public expression resources — and build an EDA toolkit that you will use on every dataset for the rest of your career.

Why It Matters

Here are things that EDA catches that no statistical test will:

  1. Mislabeled samples. A liver sample that clusters with brain samples on PCA is probably mislabeled. No differential expression analysis will flag this — it will simply produce wrong results.

  2. Batch effects. If your samples separate by processing date rather than biological condition on PCA, your entire experiment is confounded. Testing will give you thousands of “significant” results that reflect the batch, not the biology.

  3. Outliers. A single outlier sample with 10x higher expression in all genes (perhaps due to a library preparation failure) will distort every mean, every variance estimate, and every t-test.

  4. Wrong distributional assumptions. If your data are bimodal, zero- inflated, or heteroscedastic, many standard tests will be invalid. You need to see this before choosing your method.

  5. Unexpected structure. Your data might contain subgroups you did not know about. EDA reveals these before they become confounders.

Skipping EDA is like performing surgery without imaging. You might get lucky. You might not.

The Data

We will use the GTEx (Genotype-Tissue Expression) project’s median gene expression data. This file contains the median TPM (Transcripts Per Million) for each gene across 54 human tissues, calculated from thousands of donors.

# Download GTEx v8 median gene-level TPM
# This is a GCT-format file (Gene Cluster Text)
let url = "https://storage.googleapis.com/adult-gtex/bulk-gex/v8/rna-seq/GTEx_Analysis_2017-06-05_v8_RNASeQCv1.1.9_gene_median_tpm.gct.gz"
let raw_file = download(url, "data/gtex_median_tpm.gct.gz")

# GCT format has 2 header lines, then a tab-delimited matrix
# Line 1: #1.2
# Line 2: number_of_genes  number_of_samples
# Line 3: header row (Name, Description, tissue1, tissue2, ...)
# Line 4+: data rows

# Read as TSV, skipping the first 2 lines
let gtex = read_tsv(raw_file)

println("GTEx dataset dimensions:")
println("  Genes: " + to_string(len(gtex)))
println("  Columns: " + to_string(ncols(gtex)))

# Examine column names — these are the tissue types
let all_cols = columns(gtex)
println("First 10 columns:")
each(slice(all_cols, 0, 10), |c| println("  " + c))

# The first two columns are gene identifiers
# Name = Ensembl gene ID (e.g., ENSG00000000003.15)
# Description = gene symbol (e.g., TSPAN6)
# Remaining columns are tissue names

let id_cols = ["Name", "Description"]
let tissue_cols = filter(all_cols, |c| !contains(id_cols, c))
println("Number of tissues: " + to_string(len(tissue_cols)))
println("Tissues:")
each(tissue_cols, |t| println("  " + t))

Exploration

Step 1: How are expression values distributed?

This is always the first question. The shape of the distribution determines which statistics are meaningful and which are misleading.

# Get TPM values for a single tissue
let brain_cortex = col_values(gtex, "Brain - Cortex")

# Basic summary statistics

println("  n genes: " + to_string(len(brain_cortex)))
println("  Mean: " + to_string(round(mean(brain_cortex), 2)))
println("  Median: " + to_string(round(median(brain_cortex), 2)))
println("  SD: " + to_string(round(stdev(brain_cortex), 2)))
println("  Min: " + to_string(round(min(brain_cortex), 4)))
println("  Max: " + to_string(round(max(brain_cortex), 2)))
println("  Skewness indicator (mean/median): " + to_string(round(mean(brain_cortex) / median(brain_cortex), 2)))

# Quantiles
println("  Q25: " + to_string(round(quantile(brain_cortex, 0.25), 3)))
println("  Q75: " + to_string(round(quantile(brain_cortex, 0.75), 3)))
println("  Q95: " + to_string(round(quantile(brain_cortex, 0.95), 2)))
println("  Q99: " + to_string(round(quantile(brain_cortex, 0.99), 2)))

The mean is much larger than the median. This is the hallmark of a right- skewed distribution, and it is universal in gene expression data. A handful of very highly expressed genes (think ribosomal proteins with TPM > 10,000) pull the mean upward while most genes have modest or zero expression.

# How many genes have zero or near-zero expression?

let near_zero = len(filter(brain_cortex, |x| x < 0.1))
let low_count = len(filter(brain_cortex, |x| x < 1.0))
let total = len(brain_cortex)

println("Expression level breakdown:")

println("  Near-zero (TPM < 0.1): " + to_string(near_zero) + " (" + to_string(round(100.0 * near_zero / total, 1)) + "%)")

# Which genes are highly expressed in one tissue but not others?

# Define "tissue-specific" as: expressed > 10 TPM in the tissue

# For each gene, count how many tissues it is expressed in

Step 6: PCA — the most important EDA technique in genomics

Principal Component Analysis reduces the dimensionality of the data while preserving as much variance as possible. In tissue expression data, it reveals which tissues are most similar and which are most different.

# Build a matrix of tissue expression profiles
# Each row is a tissue, each column is a gene
# (Transpose of the usual gene x sample matrix)
let expr_matrix = matrix(tissue_vectors)

# PCA on the tissue profiles
let pca_result = pca(expr_matrix)

println("PCA on tissue expression profiles:")
println("Variance explained by each PC:")
let total_var = sum(pca_result.eigenvalues)
each(enumerate(pca_result.eigenvalues), |pair| {
  let i = pair[0]
  let ev = pair[1]
  let pct = round(100.0 * ev / total_var, 1)
  if i < 10 then {
    println("  PC" + to_string(i + 1) + ": " + to_string(pct) + "%")
  }
})

# Visualize the first two principal components
let pc1 = col_values(pca_result.scores, 0)
let pc2 = col_values(pca_result.scores, 1)

# Create PCA plot
pca_plot(pca_result, "plots/tissue_pca.svg")
  |> save_plot("plots/tissue_pca.svg")

# Also create a manual scatter plot for more control
scatter(pc1, pc2, "plots/tissue_pca_scatter.svg")
  |> save_plot("plots/tissue_pca_scatter.svg")

# Label each point
println("PCA coordinates:")
each(enumerate(compare_tissues), |pair| {
  let i = pair[0]
  let tissue = pair[1]
  println("  " + tissue + ": PC1=" + to_string(round(pc1[i], 2)) + ", PC2=" + to_string(round(pc2[i], 2)))
})

Interpreting the PCA plot:

  • PC1 typically separates the most transcriptionally distinct tissues. Brain tissues often separate from everything else because the brain transcriptome is uniquely enriched in neuron-specific genes.

  • PC2 captures the next largest source of variation. Often this separates other tissue groups (e.g., blood from solid tissues).

  • Tissues that cluster together share similar expression programs. The three brain regions should cluster tightly. The two heart tissues should be near each other.

  • An outlier tissue that does not cluster with its expected group warrants investigation.

Step 7: Outlier detection

# Outlier detection in high-dimensional data
# Method 1: Mahalanobis distance (simplified version)
# A sample far from the center of the PCA cloud is suspicious

let pc1_mean = mean(pc1)
let pc2_mean = mean(pc2)
let pc1_sd = stdev(pc1)
let pc2_sd = stdev(pc2)

let distances = map(enumerate(compare_tissues), |pair| {
  let i = pair[0]
  let tissue = pair[1]
  let d1 = (pc1[i] - pc1_mean) / pc1_sd
  let d2 = (pc2[i] - pc2_mean) / pc2_sd
  let d = sqrt(d1 * d1 + d2 * d2)
  record(tissue: tissue, pca_distance: round(d, 3))
})

let dist_sorted = sort(distances, |a, b| b.pca_distance - a.pca_distance)
println("Tissues ranked by PCA distance from center:")
each(dist_sorted, |d| println("  " + d.tissue + ": " + to_string(d.pca_distance)))

# Method 2: Expression profile distance
# How different is each tissue from the "average tissue"?

let mean_profile = map(range(0, len(tissue_vectors[0])), |gene_i| {
  let gene_vals = map(tissue_vectors, |tv| tv[gene_i])
  mean(gene_vals)
})

let profile_distances = map(enumerate(compare_tissues), |pair| {
  let i = pair[0]
  let tissue = pair[1]
  let tv = tissue_vectors[i]
  let diffs = map(range(0, len(tv)), |j| {
    pow(tv[j] - mean_profile[j], 2.0)
  })
  let rmsd = sqrt(mean(diffs))
  record(tissue: tissue, rmsd: round(rmsd, 4))
})

let profile_sorted = sort(profile_distances, |a, b| b.rmsd - a.rmsd)
println("Tissues ranked by expression profile distance from mean:")
each(profile_sorted, |d| println("  " + d.tissue + ": RMSD=" + to_string(d.rmsd)))

# A tissue with RMSD > 2x the median RMSD warrants closer inspection
let median_rmsd = median(map(profile_distances, |d| d.rmsd))
println("Median RMSD: " + to_string(round(median_rmsd, 4)))
println("Outlier threshold (2x median): " + to_string(round(2.0 * median_rmsd, 4)))

Step 8: Housekeeping genes vs tissue-specific genes

# Housekeeping genes: expressed in all tissues at moderate-high levels
# These have low coefficient of variation across tissues

let gene_variability = map(range(0, min(len(gtex), 5000)), |i| {
  let row = slice(gtex, i, i + 1)
  let gene_name = col_values(row, "Description") |> take(1)

  let tissue_vals = map(tissue_cols, |t| {
    col_values(row, t) |> take(1) |> to_float()
  })

  let m = mean(tissue_vals)
  let s = stdev(tissue_vals)
  let cv = if m > 0.0 then s / m else 0.0

  record(
    gene: gene_name,
    mean_tpm: round(m, 2),
    sd_tpm: round(s, 2),
    cv: round(cv, 3),
    min_tpm: round(min(tissue_vals), 2),
    max_tpm: round(max(tissue_vals), 2)
  )
})

# Filter for expressed genes (mean > 5 TPM)
let expressed_variable = filter(gene_variability, |g| g.mean_tpm > 5.0)

# Most stable (housekeeping) genes
let housekeeping = sort(expressed_variable, |a, b| a.cv - b.cv)
println("Top 20 most stable (housekeeping) genes:")
each(take(housekeeping, 20), |g| {
  println("  " + to_string(g.gene) + ": mean=" + to_string(g.mean_tpm) + ", CV=" + to_string(g.cv))
})

# Most variable genes
let most_variable = sort(expressed_variable, |a, b| b.cv - a.cv)
println("")
println("Top 20 most variable genes:")
each(take(most_variable, 20), |g| {
  println("  " + to_string(g.gene) + ": mean=" + to_string(g.mean_tpm) + ", CV=" + to_string(g.cv))
})

The Analysis

Now that we have explored the data visually, let us formalize our observations with systematic analyses.

Systematic tissue clustering

# Hierarchical clustering using correlation distance
# 1 - correlation gives a distance metric: similar tissues have small distance

let all_tissue_vectors = map(tissue_cols, |tissue| {
  let raw = col_values(expressed_genes, tissue)
  map(raw, |x| log2(x + 1.0))
})

# Compute the full correlation matrix
let full_cor = cor_matrix(all_tissue_vectors)

# Convert to distance matrix (1 - r)
let n = len(tissue_cols)
let dist_vals = map(range(0, n), |i| {
  map(range(0, n), |j| {
    round(1.0 - full_cor[i][j], 4)
  })
})

let full_dist = dist_matrix(dist_vals)

println("Distance matrix (1 - correlation) for selected tissues:")
println(full_dist)

heatmap(full_cor, "plots/full_tissue_correlation.svg")
  |> save_plot("plots/full_tissue_correlation.svg")

Gene filtering for downstream analysis

# Before any statistical testing, filter genes appropriately
# This is a critical EDA step that affects all downstream results

# Common filter: require TPM >= 1 in at least 20% of tissues
let min_tissues = ceil(len(tissue_cols) * 0.2)
println("Requiring expression in at least " + to_string(min_tissues) + " tissues")

let filtered_genes = filter_by(gtex, |row| {
  let vals = map(tissue_cols, |t| col_values(row, t) |> take(1) |> to_float())
  let n_expressed = len(filter(vals, |x| x >= 1.0))
  n_expressed >= min_tissues
})

println("Genes before filtering: " + to_string(len(gtex)))

Detecting bimodal distributions

# Some genes have bimodal expression across tissues
# (highly expressed in some tissues, off in others)
# This indicates tissue-specific regulation

# A simple bimodality detector: ratio of variance to mean
# (on log scale) combined with the distribution shape

let bimodal_candidates = map(range(0, min(len(filtered_genes), 3000)), |i| {
  let row = slice(filtered_genes, i, i + 1)
  let gene = col_values(row, "Description") |> take(1)

  let vals = map(tissue_cols, |t| {
    col_values(row, t) |> take(1) |> to_float()
  })
  let log_vals = map(vals, |x| log2(x + 1.0))

  let m = mean(log_vals)
  let s = stdev(log_vals)
  let med = median(log_vals)

  # Bimodal indicator: high variance and mean != median
  let bimodal_score = s * abs(m - med)

  record(
    gene: gene,
    bimodal_score: round(bimodal_score, 3),
    mean_log: round(m, 2),
    sd_log: round(s, 2),
    median_log: round(med, 2)
  )
})

let bimodal_sorted = sort(bimodal_candidates, |a, b| b.bimodal_score - a.bimodal_score)
println("Top candidate bimodal genes:")
each(take(bimodal_sorted, 15), |g| {
  println("  " + to_string(g.gene) + ": score=" + to_string(g.bimodal_score) + ", mean=" + to_string(g.mean_log) + ", sd=" + to_string(g.sd_log))
})

Summary statistics that matter

# The right summary statistic depends on the distribution shape:
#
# Symmetric (normal-ish):
#   Center: mean or median (both work)
#   Spread: standard deviation
#
# Right-skewed (raw expression):
#   Center: MEDIAN (mean is inflated by outliers)
#   Spread: IQR (interquartile range)
#
# Zero-inflated:
#   Report zero proportion separately
#   Summarize non-zero values with median
#
# Bimodal:
#   Neither mean nor median is meaningful
#   Report the two modes separately

# Demonstrate with brain cortex data
let iqr = quantile(brain_cortex, 0.75) - quantile(brain_cortex, 0.25)
println("Brain Cortex TPM - Appropriate summaries:")
println("  For raw TPM (skewed):")
println("    Median: " + to_string(round(median(brain_cortex), 4)))
println("    IQR: " + to_string(round(iqr, 4)))
println("    Mean: " + to_string(round(mean(brain_cortex), 2)) + " (misleading!)")
println("")
println("  For log2(TPM+1) (symmetric):")
println("    Mean: " + to_string(round(mean(brain_log), 3)))
println("    SD: " + to_string(round(stdev(brain_log), 3)))

The EDA checklist

# Every new dataset should go through this checklist:
#
# 1. DIMENSIONS
#    - How many genes? How many samples?
#    - Any unexpected missing data?
#
# 2. DISTRIBUTIONS
#    - Raw data shape (usually right-skewed for expression)
#    - Log-transformed shape (should be ~normal for expression)
#    - Zero inflation (what fraction of genes are zero?)
#    - Outlier values (max value, 99th percentile)
#
# 3. SAMPLE QUALITY
#    - Per-sample summary statistics (mean, SD, zero fraction)
#    - Sample-sample correlation (within-group > 0.95?)
#    - PCA (samples cluster by biology, not by batch?)
#
# 4. GENE FILTERING
#    - Remove lowly expressed genes
#    - Remove genes with zero variance
#    - Document how many genes removed
#
# 5. NORMALIZATION
#    - What normalization was applied?
#    - Do library sizes differ dramatically?
#    - Are distributions comparable across samples?
#
# 6. STRUCTURE
#    - Expected groupings visible in PCA?
#    - Any unexpected subclusters?
#    - Correlation structure makes biological sense?
#
# 7. ANOMALIES
#    - Outlier samples?
#    - Batch effects visible?
#    - Any variables confounded with the factor of interest?

Interpretation

Exploratory data analysis of the GTEx dataset reveals several important features that are general to gene expression data:

Most genes are not expressed in any given tissue. Typically 40-60% of annotated genes have zero or near-zero expression. This reflects the fact that most genes are tissue-specific or lineage-specific. Keeping these in your analysis inflates the multiple testing burden and can distort normalization.

Expression distributions are universally right-skewed. A small number of genes consume a large fraction of the transcriptome. The top 100 genes can account for 30-50% of total TPM. This motivates log-transformation as a standard preprocessing step.

Tissues cluster by lineage. Brain regions cluster together, heart chambers cluster together, and so on. This is expected and reassuring — it means the data capture real biology. If your experimental samples do not cluster by expected groups, something is wrong.

Housekeeping genes have low variance. Genes like GAPDH, ACTB, and ribosomal proteins are expressed at similar levels across all tissues. These serve as internal controls and reference genes for normalization.

Tissue-specific genes drive the principal components. The genes with the highest PC1 loadings are typically the most tissue-specific. In a disease study, the genes driving separation between conditions are your candidates for follow-up.

Decision

After thorough EDA, you should be able to answer:

  1. Is the data quality sufficient for your question? If samples do not cluster by biology, or if outliers dominate, no amount of statistical testing will save the experiment.

  2. Which genes to include? Your filtering strategy affects everything downstream. Document it clearly.

  3. Which normalization to use? TPM, FPKM, CPM, or quantile normalization each have trade-offs. EDA helps you choose.

  4. What statistical method is appropriate? If distributions are non-normal even after transformation, use nonparametric tests. If there are confounders, use regression.

  5. What are the expected effect sizes? Looking at the variance in your data tells you what effects you can realistically detect.

# Decision framework summary

Pitfalls

Pitfall 1: Analyzing without looking

# The most common mistake is jumping straight to testing
# without exploring the data first.
#
# Example: You run DESeq2 on samples that include a mislabeled
# sample. You get hundreds of "significant" genes that are
# actually driven by the mislabeled sample. A PCA plot would
# have caught this in 30 seconds.
#
# Rule: ALWAYS plot PCA before any statistical test.
# It takes 5 minutes and can save weeks of wasted analysis.

Pitfall 2: Over-filtering genes

# If you filter too aggressively, you remove real biology.
# If you filter too loosely, you test too many genes and
# inflate the multiple testing burden.
#
# A gene with TPM = 0.5 in all normal samples and TPM = 5.0
# in all tumor samples would be removed by a "TPM > 1 in all
# samples" filter, even though it shows a 10-fold change.
#
# Better: require expression above threshold in at least one GROUP
# Not in all samples.

# WRONG: Filter genes with mean TPM > 1 across ALL samples
# RIGHT: Filter genes with mean TPM > 1 in at least one group

Pitfall 3: Log-transforming zeros

# log2(0) is negative infinity. You cannot log-transform zeros.
# Common solutions:
# 1. Add a pseudocount: log2(x + 1) or log2(x + 0.5)
# 2. Use a variance-stabilizing transformation (VST)
# 3. Filter zeros before transforming
#
# The pseudocount choice matters for lowly expressed genes:
# log2(0 + 1) = 0
# log2(0 + 0.1) = -3.32
# log2(0 + 0.01) = -6.64
#
# Different pseudocounts give different results for low-expression genes
# This is why purpose-built transformations (VST, rlog) are preferred

let test_vals = [0.0, 0.01, 0.1, 1.0, 10.0, 100.0]
println("Effect of pseudocount on log-transformation:")
each(test_vals, |x| {
  let l1 = log2(x + 1.0)
  let l05 = log2(x + 0.5)
  let l01 = log2(x + 0.01)
  println("  TPM=" + to_string(x) + ": log2(x+1)=" + to_string(round(l1, 2)) + ", log2(x+0.5)=" + to_string(round(l05, 2)) + ", log2(x+0.01)=" + to_string(round(l01, 2)))
})

Pitfall 4: Ignoring the compositional nature of expression data

# TPM values sum to 1 million per sample.
# This means they are compositional: increasing one gene's TPM
# necessarily decreases others'.
#
# If one highly expressed gene changes 10-fold, it can shift
# the TPM of ALL other genes even if they are unchanged.
#
# This is called "compositional bias" and it is a real problem
# in RNA-seq analysis.
#
# Consider:
# Before: Gene A = 100,000 TPM (10% of transcriptome)
# After:  Gene A = 500,000 TPM (50% of transcriptome)
#
# Gene B was 1000 TPM in both conditions. But:
# Before: Gene B = 1000 / 1,000,000 = 0.1%
# After:  Gene B = 1000 / 1,000,000 = 0.1%... wait, TPM re-normalizes
# After normalization, Gene B's TPM drops even though it did not change!
#
# Methods that use size factors (DESeq2) or TMM normalization (edgeR)
# account for this. Raw TPM comparisons do not.

Pitfall 5: Overinterpreting PCA

# PCA shows the LARGEST sources of variation
# These may not be biologically interesting
#
# PC1 might separate by:
# - Sex (if you have mixed-sex samples)
# - Batch (if processing dates differ)
# - Library size (if some samples have more reads)
# - Cell type composition (if tissue heterogeneity varies)
#
# None of these are your biological variable of interest
# but they can dominate the PCA
#
# Always check what is driving the principal components
# before concluding that samples cluster by condition

# Example: correlate PC1 scores with known covariates
# If PC1 correlates with batch more than condition,
# you need batch correction before proceeding

Pitfall 6: Mean as representative value for skewed data

# The mean of gene expression values (especially raw counts or TPM)
# is NOT a good representative value

let example_tpms = [0.0, 0.0, 0.0, 0.1, 0.3, 1.2, 3.5, 15.0, 120.0, 8500.0]
println("Example TPMs: " + to_string(example_tpms))
println("Mean: " + to_string(round(mean(example_tpms), 2)))
println("Median: " + to_string(round(median(example_tpms), 2)))

# The mean (864) is dominated by the single high value (8500)
# The median (0.75) better represents the "typical" gene
# Always use median for skewed distributions

Where Next?

This chapter has equipped you with a systematic approach to exploring any gene expression dataset. The key lessons are:

  1. Always look before you test. Five minutes of EDA can prevent weeks of wasted analysis.

  2. Understand your distributions. Gene expression is right-skewed, zero- inflated, and compositional. These properties determine which methods work.

  3. PCA is your single most valuable tool. It reveals sample quality, batch effects, outliers, and biological structure simultaneously.

  4. The right summary statistic depends on the distribution. Median for skewed data, mean for symmetric. Never report a mean without also reporting the shape.

  5. Document your preprocessing. Every filtering, transformation, and normalization choice affects downstream results. Write it down.
