Pipelines

Bioinformatics workflows are rarely a single step. A typical analysis chains together quality control, alignment, variant calling, filtering, and annotation. BioLang builds pipelines through pipe-first composition using the |> operator. There is no special pipeline keyword — a pipeline is simply a sequence of pipe chains, variables, and function calls that transform data step by step.

Key idea:

In BioLang, a |> f(b) desugars to f(a, b). Every function takes the primary data as its first argument, so any function slots naturally into a pipe chain. Pipelines are just chains of |> calls stored in variables.

Pipe Chains as Pipelines

The simplest pipeline is a single pipe chain. Each |> feeds the result of the left side as the first argument to the right side: 

# Read FASTQ, filter by quality, compute GC content
let reads = read_fastq("sample.fq.gz") |> collect()
let filtered = reads |> filter(|r| mean_phred(r.quality) >= 25)
let gc_vals = filtered |> map(|r| gc_content(r.seq))
println(f"Passed: {len(filtered)}/{len(reads)}, Mean GC: {mean(gc_vals)}")

Each variable captures an intermediate result. The pipe operator has the lowest precedence, so expressions on either side evaluate fully before piping. Newlines after |> are suppressed, so chains can span multiple lines freely.

Multi-Step Analysis Patterns

Real analyses involve multiple stages: loading data, filtering, transforming, grouping, and summarizing. Break each stage into its own variable for clarity: 

# Variant analysis pipeline
let variants = read_vcf("calls.vcf") |> collect()
let high_qual = variants |> filter_by("quality", ">=", 30)
let classified = high_qual |> classify_variants()
let by_chrom = classified |> group_by("chrom")

println(f"Total: {len(variants)}, Filtered: {len(high_qual)}")

# Summarize per chromosome
for entry in by_chrom {
  let chrom = entry.key
  let count = len(entry.values)
  println(f"  {chrom}: {count} variants")
}

You can also write a single long chain when the logic is linear: 

let top_genes = read_csv("deseq2_results.csv")
  |> filter(|r| r.padj < 0.05)
  |> filter(|r| abs(r.log2FoldChange) > 1.0)
  |> arrange("-log2FoldChange")
  |> head(20)
  |> select("gene", "log2FoldChange", "padj")

Data Transformation Pipelines

BioLang provides a full set of data transformation functions that are designed to work with pipes: filter, map, mutate, group_by, summarize, select, arrange, and more. 

# filter: keep rows matching a predicate
let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)

# map: transform each element
let lengths = reads |> map(|r| len(r.seq))

# mutate: add or modify columns in a table
let annotated = data
  |> mutate(rate: |r| r.passing / r.total * 100)
  |> mutate(status: |r| if r.rate > 90 then "PASS" else "FAIL")

# group_by + summarize: aggregate by key
let summary = data
  |> group_by("cohort")
  |> summarize(
    mean_depth: |g| mean(g.depth),
    sample_count: |g| len(g)
  )

# select + arrange: pick columns and sort
let report = summary
  |> select("cohort", "mean_depth", "sample_count")
  |> arrange("-mean_depth")

These compose into complete ETL pipelines: 

let cohort_report = read_csv("samples.csv")
  |> filter(|r| r.depth > 30)
  |> mutate(rate: |r| r.passing / r.total * 100)
  |> group_by("cohort")
  |> summarize(
    mean_depth: |g| mean(g.depth),
    mean_rate: |g| mean(g.rate),
    n: |g| len(g)
  )
  |> arrange("-mean_depth")

println(cohort_report)

Parallel Processing

For CPU-intensive work across multiple samples, par_map distributes iterations across available cores. Each iteration runs independently and results are collected into a list: 

let samples = [
  {name: "tumor_01", path: "t01_R1.fq.gz"},
  {name: "tumor_02", path: "t02_R1.fq.gz"},
  {name: "normal_01", path: "n01_R1.fq.gz"},
  {name: "normal_02", path: "n02_R1.fq.gz"},
]

let results = samples |> par_map(|s| {
  let reads = read_fastq(s.path) |> collect()
  let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
  let gc = passed |> map(|r| gc_content(r.seq)) |> mean()
  {sample: s.name, total: len(reads), passed: len(passed), mean_gc: gc}
})

# results is a list of records, one per sample
for r in results {
  println(f"{r.sample}: {r.passed}/{r.total} passed, GC={r.mean_gc:.3}")
}

Use par_map when each iteration is independent and the per-item cost is significant (file I/O, sequence processing, external tool calls). For cheap operations on large lists, regular map avoids threading overhead.

Shell Integration for External Tools

Many bioinformatics pipelines rely on external tools like BWA, samtools, GATK, and bcftools. BioLang integrates these via the shell function: 

# Align reads with BWA and sort with samtools
shell("bwa-mem2 mem -t 16 GRCh38.fa R1.fq.gz R2.fq.gz | samtools sort -o aligned.bam")
shell("samtools index aligned.bam")

# Call variants with GATK
shell("gatk HaplotypeCaller -R GRCh38.fa -I aligned.bam -O raw.vcf.gz")

# Filter with bcftools
shell("bcftools view -f PASS raw.vcf.gz -Oz -o filtered.vcf.gz")
shell("bcftools index filtered.vcf.gz")

Shell commands integrate naturally with pipe chains. Mix native BioLang analysis with external tools in the same script: 

# Native QC, then shell alignment, then native variant analysis
let reads = read_fastq("sample_R1.fq.gz") |> collect()
let passed = reads |> filter(|r| mean_phred(r.quality) >= 25) |> len()
println(f"QC: {passed}/{len(reads)} reads pass Q25")

shell("bwa-mem2 mem -t 16 GRCh38.fa sample_R1.fq.gz sample_R2.fq.gz | samtools sort -o aligned.bam")
shell("samtools index aligned.bam")
shell("gatk HaplotypeCaller -R GRCh38.fa -I aligned.bam -O raw.vcf.gz")

let variants = read_vcf("raw.vcf.gz") |> collect()
let snps = variants |> filter(|v| v.is_snp)
let indels = variants |> filter(|v| v.is_indel)
println(f"SNPs: {len(snps)}, Indels: {len(indels)}")

Error Handling in Pipelines

Use try/catch blocks to handle errors gracefully within multi-step pipelines. This prevents a single failure from crashing the entire workflow: 

# Wrap a multi-step pipeline in try/catch
let result = try {
  let data = read_csv("samples.csv")
  data |> filter(|r| r.depth > 30)
      |> mutate(rate: |r| r.passing / r.total * 100)
      |> group_by("cohort")
      |> summarize(mean_depth: |g| mean(g.depth))
} catch e {
  println("Pipeline error: " + e)
  nil
}

For per-item error handling, wrap the transformation inside map: 

# Process each sample, skip failures
let results = samples |> map(|s| {
  try {
    let reads = read_fastq(s.path) |> collect()
    let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
    {sample: s.name, status: "ok", count: len(passed)}
  } catch e {
    println(f"Warning: {s.name} failed: {e}")
    {sample: s.name, status: "error", count: 0}
  }
})

let succeeded = results |> filter(|r| r.status == "ok")
println(f"{len(succeeded)}/{len(results)} samples processed successfully")

Notifications

Long-running pipelines benefit from notifications when they complete or encounter errors. Use notify to send alerts: 

# Notify on pipeline completion
let results = samples |> par_map(|s| {
  let reads = read_fastq(s.path) |> collect()
  let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
  {sample: s.name, total: len(reads), passed: len(passed)}
})

notify({
  title: "QC Pipeline Complete",
  fields: {
    samples: len(results),
    total_reads: results |> map(|r| r.total) |> sum(),
    passed_reads: results |> map(|r| r.passed) |> sum()
  }
})

Example: FASTQ Quality Control Pipeline

A complete QC pipeline from raw reads through filtering and reporting:

# FASTQ Quality Control Pipeline
let samples = [
  {id: "S1", r1: "S1_R1.fq.gz"},
  {id: "S2", r1: "S2_R1.fq.gz"},
  {id: "S3", r1: "S3_R1.fq.gz"},
]

let qc_results = samples |> par_map(|s| {
  let reads = read_fastq(s.r1) |> collect()
  let total = len(reads)

  # Quality filtering
  let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
  let pass_rate = len(passed) / total * 100

  # GC content distribution
  let gc_vals = passed |> map(|r| gc_content(r.seq))
  let mean_gc = mean(gc_vals)

  # Length statistics
  let lengths = passed |> map(|r| len(r.seq))

  {
    sample: s.id,
    total_reads: total,
    passed_reads: len(passed),
    pass_rate: pass_rate,
    mean_gc: mean_gc,
    mean_length: mean(lengths),
  }
})

# Summary report
println("=== QC Summary ===")
for r in qc_results {
  println(f"{r.sample}: {r.pass_rate:.1}% pass, GC={r.mean_gc:.3}, len={r.mean_length:.0}")
}

Example: Variant Calling Pipeline

End-to-end pipeline from alignment through variant filtering:

# Variant Calling Pipeline
let sample_id = "tumor_01"
let r1 = "tumor_01_R1.fq.gz"
let r2 = "tumor_01_R2.fq.gz"
let ref = "GRCh38.fa"
let bam = sample_id + ".sorted.bam"

# Step 1: Align and sort
shell(f"bwa-mem2 mem -t 16 {ref} {r1} {r2} | samtools sort -@ 8 -o {bam}")
shell(f"samtools index {bam}")

# Step 2: Call variants
shell(f"gatk HaplotypeCaller -R {ref} -I {bam} -O raw.vcf.gz")

# Step 3: Native variant analysis
let variants = read_vcf("raw.vcf.gz") |> collect()

let high_qual = variants |> filter(|v| v.quality >= 30)
let snps = high_qual |> filter(|v| v.is_snp)
let indels = high_qual |> filter(|v| v.is_indel)

println(f"Total variants: {len(variants)}")
println(f"High quality (Q>=30): {len(high_qual)}")
println(f"SNPs: {len(snps)}, Indels: {len(indels)}")

# Step 4: Per-chromosome summary
let by_chrom = high_qual |> group_by("chrom")
for entry in by_chrom {
  println(f"  {entry.key}: {len(entry.values)} variants")
}

Example: RNA-seq Differential Expression

Quantification and analysis of RNA-seq data:

# RNA-seq Analysis Pipeline
let samples = [
  {id: "ctrl_1", r1: "ctrl_1_R1.fq.gz", group: "control"},
  {id: "ctrl_2", r1: "ctrl_2_R1.fq.gz", group: "control"},
  {id: "treat_1", r1: "treat_1_R1.fq.gz", group: "treatment"},
  {id: "treat_2", r1: "treat_2_R1.fq.gz", group: "treatment"},
]

# Step 1: QC all samples in parallel
let qc = samples |> par_map(|s| {
  let reads = read_fastq(s.r1) |> collect()
  let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
  {id: s.id, group: s.group, total: len(reads), passed: len(passed)}
})

println("=== QC Results ===")
for r in qc {
  let rate = r.passed / r.total * 100
  println(f"{r.id} ({r.group}): {rate:.1}% pass rate")
}

# Step 2: Align each sample
for s in samples {
  let bam = s.id + ".sorted.bam"
  shell(f"hisat2 -x genome -U {s.r1} | samtools sort -o {bam}")
  shell(f"samtools index {bam}")
}

# Step 3: Quantify and analyze counts
let counts = read_csv("featurecounts_matrix.csv")

let de_genes = counts
  |> filter(|r| r.padj < 0.05)
  |> filter(|r| abs(r.log2FoldChange) > 1.0)
  |> arrange("-log2FoldChange")

let up = de_genes |> filter(|r| r.log2FoldChange > 0)
let down = de_genes |> filter(|r| r.log2FoldChange < 0)

println(f"\nDE genes (padj<0.05, |log2FC|>1): {len(de_genes)}")
println(f"  Upregulated: {len(up)}")
println(f"  Downregulated: {len(down)}")

# Top 10 upregulated genes
println("\nTop 10 upregulated:")
let top10 = up |> head(10)
for g in top10 {
  println(f"  {g.gene}: log2FC={g.log2FoldChange:.2}, padj={g.padj:.2e}")
}

Pipeline Composition with Functions

Extract reusable pipeline steps into functions. Since BioLang is pipe-first, any function that takes data as its first argument works naturally in a pipe chain: 

# Define reusable pipeline steps as functions
fn qc_reads(path) {
  let reads = read_fastq(path) |> collect()
  let passed = reads |> filter(|r| mean_phred(r.quality) >= 25)
  {total: len(reads), passed: len(passed), reads: passed}
}

fn align_reads(sample_id, r1, r2, ref) {
  let bam = sample_id + ".sorted.bam"
  shell(f"bwa-mem2 mem -t 16 {ref} {r1} {r2} | samtools sort -@ 8 -o {bam}")
  shell(f"samtools index {bam}")
  bam
}

fn call_variants(bam, ref) {
  let vcf = replace(bam, ".bam", ".vcf.gz")
  shell(f"gatk HaplotypeCaller -R {ref} -I {bam} -O {vcf}")
  vcf
}

# Compose into a full pipeline
let ref = "GRCh38.fa"
let qc = qc_reads("sample_R1.fq.gz")
println(f"QC: {qc.passed}/{qc.total} reads passed")

let bam = align_reads("sample", "sample_R1.fq.gz", "sample_R2.fq.gz", ref)
let vcf = call_variants(bam, ref)

let variants = read_vcf(vcf) |> collect()
println(f"Called {len(variants)} variants")
