Streams

Streams are lazy iterators that process data one element at a time without loading everything into memory. In bioinformatics, where datasets routinely exceed available RAM, streams are essential for processing FASTQ files with billions of reads, BAM files spanning hundreds of gigabytes, and VCF files with millions of variants.

Streams vs. Lists

Lists are eager — all elements exist in memory simultaneously. Streams are lazy — elements are computed on demand and discarded after use: 

# Eager: loads entire file into memory (dangerous for large files)
let all_reads = read_fastq("huge.fq.gz")  # List[Read] — may OOM

# Lazy: processes one read at a time (constant memory)
let read_stream = stream_fastq("huge.fq.gz")  # Stream[Read] — safe
let filtered = read_stream
  |> filter(|r| mean_phred(r.quality) >= 30)
  |> take(1_000_000)
  |> collect()   # Only now are reads materialized

Creating Streams

From Files

# Stream biological file formats
let reads = stream_fastq("sample.fq.gz")
let alignments = stream_bam("aligned.bam")
let variants = stream_vcf("calls.vcf.gz")
let sequences = stream_fasta("reference.fa")
let intervals = stream_bed("regions.bed")
let features = stream_gff("annotations.gff3")

# Stream generic text lines
let lines = stream_lines("log.txt")

From Ranges and Generators

# Infinite range stream
let naturals = stream(1..)

# Stream from a generator function
let random_seqs = stream_generate(|| random_dna(100))

# Stream from a list (makes it lazy)
let lazy = [1, 2, 3, 4, 5] |> to_stream()

# Repeat a value
let zeros = stream_repeat(0)          # Infinite stream of zeros
let headers = stream_repeat("NA", 10) # 10 copies of "NA"

Custom Generators

# yield-based generator
let fibonacci = stream_fn(|| {
  let a = 0
  let b = 1
  loop {
    yield a
    let next = a + b
    a = b
    b = next
  }
})

# Generate sliding windows over a chromosome
let windows = stream_fn(|| {
  let pos = 0
  let step = 1000
  let size = 5000
  while pos + size <= chrom_length {
    yield interval(chrom, pos, pos + size)
    pos = pos + step
  }
})

Stream Operations

Streams support the same functional operations as lists, but all operations are lazy until a terminal operation is called:

Lazy (Intermediate) Operations

let pipeline = stream_fastq("reads.fq.gz")
  |> filter(|r| mean_phred(r.quality) >= 20)      # Lazy
  |> map(|r| trim_adapters(r))            # Lazy
  |> filter(|r| r.length >= 50)    # Lazy
  |> take(1_000_000)                      # Lazy — limits total
  |> drop(100)                            # Lazy — drops first 100

# Nothing has been read from disk yet!
# The pipeline is just a description of operations.

Terminal (Consuming) Operations

# collect — materialize into a List
let reads = pipeline |> collect()

# for_each — process each element for side effects
pipeline |> for_each(|r| write_fastq_record(out, r))

# count — count elements
let n = pipeline |> count()

# reduce — fold all elements
let total_bases = pipeline |> map(|r| r.length) |> reduce(|a, b| a + b)

# first / last — get single elements
let first_read = pipeline |> first()
let last_read = pipeline |> last()

# any / all — short-circuit boolean tests
let has_long = pipeline |> any(|r| r.length > 10_000)
let all_hq = pipeline |> all(|r| mean_phred(r.quality) >= 30)

# to_table — collect into a Table
let summary = pipeline |> to_table(columns = ["name", "length", "quality"])

Stream Consumption

Streams are consumed exactly once. After a terminal operation, the stream is exhausted and cannot be reused: 

let s = stream_fastq("data.fq")

let count = s |> count()      # Consumes the stream
# let data = s |> collect()   # Error: stream already consumed

# To process a stream multiple times, create it again:
let s1 = stream_fastq("data.fq")
let count = s1 |> count()

let s2 = stream_fastq("data.fq")
let data = s2 |> collect()

# Or use tee to split a stream into two
let (for_count, for_stats) = stream_fastq("data.fq") |> tee()
let count = for_count |> count()
let stats = for_stats |> map(|r| mean_phred(r.quality)) |> collect()

Backpressure

BioLang streams implement natural backpressure. The producer only generates the next element when the consumer is ready. This prevents memory buildup in multi-stage pipelines: 

# This pipeline uses constant memory regardless of input size.
# Each stage processes one element at a time:
stream_bam("whole_genome.bam")          # Read one record
  |> filter(|r| r.mapq >= 30)           # Check quality
  |> filter(|r| !r.is_duplicate)        # Check duplicate flag
  |> filter(|r| r.is_proper_pair)       # Check pairing
  |> map(|r| extract_insert_size(r))    # Compute insert size
  |> write_stream("insert_sizes.txt")   # Write one line

# The BAM file is never fully loaded — one record flows through at a time

File Streaming

Streaming I/O is the primary use case for streams in bioinformatics. BioLang automatically handles compressed formats: 

# Stream from compressed FASTQ, write compressed output
stream_fastq("input.fq.gz")
  |> filter(|r| mean_phred(r.quality) >= 30)
  |> write_fastq("output.fq.gz")   # Auto-detects .gz extension

# Stream BAM, compute coverage, write BED
stream_bam("aligned.bam")
  |> filter(|r| r.mapq >= 20)
  |> coverage(bin_size = 1000)
  |> filter(|r| r.depth >= 10)
  |> write_bed("covered_regions.bed")

# Stream VCF, annotate, write filtered VCF
stream_vcf("raw.vcf.gz")
  |> filter(|v| v.qual >= 30.0)
  |> annotate_stream("clinvar.vcf.gz")
  |> filter(|v| v.annotation.clinical_significance == "Pathogenic")
  |> write_vcf("pathogenic.vcf.gz")

Chunked Processing

Process streams in batches for operations that benefit from batch processing: 

# Process in chunks of 10,000
stream_fastq("reads.fq.gz")
  |> chunk(10_000)
  |> for_each(|batch| {
    let stats = compute_batch_stats(batch)
    print(f"Batch: {stats.avg_quality} avg quality, {len(batch)} reads")
  })

# Parallel batch processing
stream_fasta("proteins.fa")
  |> chunk(1000)
  |> parallel_map(|batch| {
    batch |> map(|seq| predict_structure(seq))
  })
  |> flatten()
  |> write_results("predictions.json")

Merging Streams

# Concatenate streams sequentially
let all_reads = stream_concat([
  stream_fastq("lane1.fq.gz"),
  stream_fastq("lane2.fq.gz"),
  stream_fastq("lane3.fq.gz")
])

# Interleave paired-end reads
let interleaved = stream_interleave(
  stream_fastq("R1.fq.gz"),
  stream_fastq("R2.fq.gz")
)

# Merge sorted streams (maintains sort order)
let merged = stream_merge_sorted(
  stream_vcf("chr1.vcf"),
  stream_vcf("chr2.vcf"),
  by = |v| (v.chrom, v.pos)
)

Stream Statistics

# Compute statistics in a single pass
let stats = stream_fastq("sample.fq.gz")
  |> stream_stats(|r| mean_phred(r.quality))

print(f"Count: {stats.count}")
print(f"Mean:  {stats.mean}")
print(f"Std:   {stats.std}")
print(f"Min:   {stats.min}")
print(f"Max:   {stats.max}")

# Histogram in one pass
let hist = stream_fastq("sample.fq.gz")
  |> map(|r| r.length)
  |> stream_histogram(bins = 50)

Progress Tracking

# Add progress reporting to any stream
stream_bam("large.bam")
  |> progress(every = 1_000_000, label = "Processing reads")
  |> filter(|r| r.mapq >= 30)
  |> count()

# Output:
# [Processing reads] 1,000,000 records...
# [Processing reads] 2,000,000 records...
# [Processing reads] 3,000,000 records...
# Total: 2,847,293
