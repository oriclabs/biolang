Error Handling

BioLang provides try/catch for exception-style error handling, the error() function for throwing errors, ?? for nil coalescing, and ?. for safe navigation. These tools cover everything from simple defaults to robust multi-stage pipeline error recovery.

Throwing Errors

Use error("message") to throw an error. The error propagates up the call stack until it is caught by a try/catch block: 

# Functions that can fail throw errors
fn parse_vcf_line(line: String) {
  let fields = line |> split("\t")
  if len(fields) < 8 {
    error("VCF line has fewer than 8 fields")
  }

  let chrom = fields[0]
  let pos = int(fields[1])
  let qual = float(fields[5])

  Variant {
    chrom: chrom,
    pos: pos,
    ref_allele: fields[3],
    alt_allele: fields[4],
    qual: qual
  }
}

Try / Catch

Wrap fallible operations in try/catch blocks. The caught value e is a plain string containing the error message: 

# Basic try/catch
try {
  let data = csv("input.csv")
  let results = analyze(data)
  write_csv(results, "output.csv")
  print("Analysis complete")
} catch e {
  print(f"Pipeline failed: {e}")
}

# try/catch for file operations
try {
  let variants = read_vcf("variants.vcf")
  let annotated = annotate(variants, "clinvar.vcf")
  write_vcf(annotated, "annotated.vcf")
} catch e {
  print(f"Error: {e}")
}

Nil Coalescing: ??

The ?? operator returns the left-hand value if it is not nil, otherwise returns the right-hand default: 

# Provide defaults for nil values
let gene = find_gene("BRCA1", db)
let g = gene ?? default_gene

# Chain ?? for fallback sequences
let ref_path = env("REF_GENOME")
  ?? config["reference"]
  ?? "/refs/default/GRCh38.fa"

# Common with map lookups
let score = scores["BRCA1"] ?? 0.0
let name = record["gene_name"] ?? "unknown"

Safe Navigation: ?.

The ?. operator accesses a field only if the receiver is not nil. If the receiver is nil, the entire expression evaluates to nil without an error: 

# Safe navigation on potentially nil values
let gene_name = result?.gene?.symbol
let score = annotation?.impact?.score ?? 0.0

# Useful in pipe chains with optional data
let pathway = find_gene("TP53", db)
  ?.pathway
  ?? "unknown"

Error Propagation Pattern

Since BioLang does not have a postfix ? operator, use try/catch to propagate errors up the call stack: 

# Chain fallible operations with try/catch
fn load_and_filter(path: String) {
  let text = read_text(path)
  let table = csv(text)
  table |> filter(|r| r.score > 0.5)
}

# Errors from read_text or csv propagate automatically.
# Catch them at the call site:
try {
  let data = load_and_filter("samples.csv")
  print(f"Loaded {len(data)} rows")
} catch e {
  print(f"Failed to load: {e}")
}

# Multi-step pipeline
fn process_sample(path: String) {
  let data = csv(path)
  let validated = validate_schema(data)
  normalize(validated)
}

Finding Values

Functions like find return nil when no match is found. Use ?? to provide defaults: 

# find returns nil if no match
let gene = find_gene("BRCA1", db)

# Provide a default
let g = gene ?? fetch_from_ncbi("BRCA1")

# Check for nil explicitly
if gene != nil {
  print(f"Found {gene.symbol} on {gene.chrom}")
  analyze(gene)
} else {
  print("Gene not found in database")
}

Try as an Expression

try can be used as an expression. If the block throws, the catch branch provides a fallback value: 

# try/catch as an expression
let data = try { csv("data.csv") } catch _ { empty_table() }

# Useful in map for fault-tolerant batch processing
let results = file_paths
  |> map(|path| try { csv(path) } catch _ { nil })
  |> filter(|r| r != nil)

Validation with error()

# Validate inputs and throw on failure
fn validate_vcf(table: Table) {
  if !("CHROM" in table.columns) {
    error("Missing required column: CHROM")
  }
  if !("POS" in table.columns) {
    error("Missing required column: POS")
  }
  if table.num_rows == 0 {
    error("VCF table is empty")
  }
  table
}

# Usage
try {
  let data = csv("variants.csv")
  let validated = validate_vcf(data)
  process(validated)
} catch e {
  print(f"Validation failed: {e}")
}

Retry Logic

# Retry with exponential backoff
fn fetch_with_retry(url: String, max_attempts = 3) {
  let attempt = 0
  let last_error = nil

  while attempt < max_attempts {
    try {
      return fetch(url)
    } catch e {
      last_error = e
      attempt = attempt + 1
      if attempt < max_attempts {
        sleep(pow(2, attempt) * 1000)
      }
    }
  }

  error(f"Failed after {max_attempts} attempts: {last_error}")
}

# Usage
let seq = fetch_with_retry("https://api.ncbi.nlm.nih.gov/seq/NM_007294")

Retry

The retry expression re-attempts a block up to N times with an optional delay — essential for flaky network calls to bio databases: 

# Retry NCBI fetch 3 times with 2-second delay
let gene = retry(3, delay: 2000) {
    ncbi_gene("TP53")
}

# Retry with increasing delay (manual exponential backoff)
let result = retry(5, delay: 1000) {
    ensembl_vep("rs699")
}

Collecting Results

# Process all, collect successes and failures separately
let successes = []
let failures = []

for s in samples {
  try {
    let result = process(s)
    successes = push(successes, {id: s.id, data: result})
  } catch e {
    failures = push(failures, {id: s.id, error: e})
  }
}

print(f"Succeeded: {len(successes)}, Failed: {len(failures)}")
for f in failures {
  print(f"  {f.id}: {f.error}")
}

Assertions

# assert — throws an error if condition is false
assert(len(reads) > 0, "No reads found in input file")
assert(len(reference) > 0, "Reference sequence is empty")

# assert with equality checks
assert(len(columns) == expected_cols, "Column count mismatch")
assert(sample_a != sample_b, "Samples must be different")
